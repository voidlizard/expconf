The saturation arithmetic component supports arithmetic, relational, and equality operations on integer literals (without suffixes) ranging from 0 to CHAOS_PP_LIMIT_MAG.

Saturation occurs when an operation attempts to decrease a 0 value or increase a CHAOS_PP_LIMIT_MAG value. In both cases, the decrease or increase becomes an identity--i.e. an attempt to decrease 0, such as 

CHAOS_PP_DEC(0)

results in 0, and an attempt to increase CHAOS_PP_LIMIT_MAG, such as 

CHAOS_PP_INC(CHAOS_PP_LIMIT_MAG)

results in CHAOS_PP_LIMIT_MAG.

The CHAOS_PP_IN_RANGE predicate can be used to test whether an integer literal is in range (i.e. is greater than or equal to 0 and less than or equal to CHAOS_PP_LIMIT_MAG). 

CHAOS_PP_IN_RANGE(23)      // 1
CHAOS_PP_IN_RANGE(1000000) // 0

It should be reinforced that the saturation arithmetic facilities operate on integer literals (without suffixes), not integral constant expressions such as 1 + 2.

Arithmetic Operators

The following table summarizes the arithmetic operations and their equivalents in C or C++:

CHAOS_PP_ADD(x, y) x + y > CHAOS_PP_LIMIT_MAG ? CHAOS_PP_LIMIT_MAG : x + y
CHAOS_PP_DEC(x) x - 1 < 0 ? 0 : x - 1
CHAOS_PP_DIV(x, y) x / y
CHAOS_PP_INC(x) x + 1 > CHAOS_PP_LIMIT_MAG ? CHAOS_PP_LIMIT_MAG : x + 1
CHAOS_PP_MUL(x, y) x * y > CHAOS_PP_LIMIT_MAG ? CHAOS_PP_LIMIT_MAG : x * y
CHAOS_PP_MOD(x, y) x % y
CHAOS_PP_SUB(x, y) x - y < 0 ? 0 : x - y

Equality & Relational Operators

The following table summarizes the equality and relational operations and their equivalents in C or C++.

CHAOS_PP_EQUAL(x, y) x == y
CHAOS_PP_GREATER(x, y) x > y
CHAOS_PP_GREATER_EQUAL(x, y) x >= y
CHAOS_PP_LESS(x, y) x < y
CHAOS_PP_LESS_EQUAL(x, y) x <= y
CHAOS_PP_NOT_EQUAL(x, y) x != y
