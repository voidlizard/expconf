A string is a data structure consisting of a series of space-separated letters, numerals, or underscores. For example, 

a b c

is a string containing three elements, a, b, and c.

The string-specific primitives are located in the chaos/preprocessor/string subdirectory.

Restricted Domain

Strings, unlike other data structures, have a restricted domain of possible element values. Each element must be a single letter (from a to z or from A to Z), a numeral (from 0 to 9), or an underscore (_). Any other character is invalid, and any operation that inserts an invalid character invalidates the string. The invocation of an algorithm on an invalid string is undefined and will likely cause catastrophic failure.

Note that it is safe to use the core string operations defined in <chaos/preprocessor/string/core.h> with invalid strings provided that the invalid element is never the first element of the string. For example, 

CHAOS_PP_STRING_HEAD(a xyz b) // a
CHAOS_PP_STRING_TAIL(a xyz b) // xyz b

CHAOS_PP_STRING_HEAD(xyz b)   // invalid

This allowance makes it possible to reuse the string data type in the creation of, for example, and extensible string data type. However, such use requires extreme care.

Using Strings as Generic Data Structures

Strings can be used as generic data structures if the the <chaos/preprocessor/string/core.h> header is included. For example, 

CHAOS_PP_CONCAT((CHAOS_PP_STRING) C h a o s) // Chaos

Nil Strings

In C (prior to C99) and C++, there is no such thing as a nil string. However, the special value CHAOS_PP_STRING_NIL() can be used to indicate a nil string to the core string operations defined in <chaos/preprocessor/string/core.h>--thus allowing the generic algorithms to operate on nil strings in a temporal fashion. The other string-specific primitives, on the other hand, do not recognize CHAOS_PP_STRING_NIL(). Therefore, a "would-be" nil string result from a string-specific primitive is undefined. The result of generic algorithm that yields a nil string is well-defined provided that it is used only as a generic data structure--meaning that it can be passed to other generic algorithms, etc.--or used by the core operations in <chaos/preprocessor/string/core.h> after the type prefix is stripped. For example, 

CHAOS_PP_APPEND(
    CHAOS_PP_DROP(3, (CHAOS_PP_STRING) a b c),
    (CHAOS_PP_STRING) x y
)
// (CHAOS_PP_STRING) x y

In C99, nil strings are well-defined as "nothing", which is accepted as the representation of nil by all string primitives.
