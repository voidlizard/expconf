The repetition/enum_from_to_trailing.h header defines an algorithm that enumerates a user-specified target for each number in a specified range.

Usage

#include <chaos/preprocessor/repetition/enum_from_to_trailing.h>

Contents

CHAOS_PP_ENUM_FROM_TO_TRAILING

The CHAOS_PP_ENUM_FROM_TO_TRAILING macro enumerates its third argument for each number in the range specified by its first two arguments.

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_ENUM_FROM_TO_TRAILING(from, to, macro))
from

The lower bound of the range of numbers for which macro is to be repeated. Valid values range from 0 to to.

to

The upper bound of the range of numbers for which macro is to be repeated. Valid values range from from to CHAOS_PP_LIMIT_MAG.

macro

The target that is to be repeated. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, n)

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_ENUM_FROM_TO_TRAILING(from, to, macro, ...))
from

The lower bound of the range of numbers for which macro is to be repeated. Valid values range from 0 to to.

to

The upper bound of the range of numbers for which macro is to be repeated. Valid values range from from to CHAOS_PP_LIMIT_MAG.

macro

The target that is to be repeated. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, n, data†)

† The data pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the auxiliary data, __VA_ARGS__.

[...]

Auxiliary data to be passed through the algorithm to the target.

Usage - C/C++ Specific 

CHAOS_PP_EXPR(CHAOS_PP_ENUM_FROM_TO_TRAILING(from, to, macro, data))
from

The lower bound of the range of numbers for which macro is to be repeated. Valid values range from 0 to to.

to

The upper bound of the range of numbers for which macro is to be repeated. Valid values range from from to CHAOS_PP_LIMIT_MAG.

macro

The target that is to be repeated. This target must be a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, n, data)
data

Auxiliary data to be passed through the algorithm to the target.

Remarks

This macro behaves identically to CHAOS_PP_ENUM_FROM_TO except that it inserts a leading comma if any repetitions exist. This is illustrated by the following pseudo-code: 

, macro(s, from, data), macro(s, from + 1, data), ... macro(s, to - 1, data)

This macro deduces the current recursion state and invokes CHAOS_PP_ENUM_FROM_TO_TRAILING_S. If the current recursion state is already known, direct use of CHAOS_PP_ENUM_FROM_TO_TRAILING_S is more efficient.

This macro may not be used if bypass semantics are in effect.

CHAOS_PP_ENUM_FROM_TO_TRAILING_ID

The CHAOS_PP_ENUM_FROM_TO_TRAILING_ID macro indirectly expands to CHAOS_PP_ENUM_FROM_TO_TRAILING.

Usage 

CHAOS_PP_ENUM_FROM_TO_TRAILING_ID()

Remarks

This macro effectively allows CHAOS_PP_ENUM_FROM_TO_TRAILING to pass through itself without becoming disabled.

CHAOS_PP_ENUM_FROM_TO_TRAILING_

Usage - C99 Specific 

CHAOS_PP_ENUM_FROM_TO_TRAILING_

Remarks

CHAOS_PP_LAMBDA(CHAOS_PP_ENUM_FROM_TO_TRAILING_ID)()
