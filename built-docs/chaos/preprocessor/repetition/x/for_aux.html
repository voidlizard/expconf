The repetition/x/for_aux.h header defines a predicated exponential algorithm which can be parametrically resumed.

Usage

#include <chaos/preprocessor/repetition/x/for_aux.h>

Contents

  • CHAOS_PP_FOR_AUX_X

CHAOS_PP_FOR_AUX_X

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_FOR_AUX_X(size, pred, op, macro, data, ...))
size

The size of the buffer of valid recursion steps maintained by the algorithm.

pred

The predicate that determines whether the algorithm should continue. This predicate must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

pred(s, state†)

† The state pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the initial state, __VA_ARGS__.

op

The operation used to iterate the state. This operation must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

op(s, state†)

† The state pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the initial state, __VA_ARGS__.

macro

The target to be repeated. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, state†)

† The state pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the initial state, __VA_ARGS__.

[data]

Auxiliary data to be passed through the algorithm to the predicate, operation, and target. This argument may be either a parenthetic expression of any arity or a non-parenthetic expression.

[...]

The initial state.

Remarks

CHAOS_PP_overview

This macro behaves identically to CHAOS_PP_FOR except that the invocations of the target are not trampolined, the algorithm makes exponential use of the recursion backend, and the algorithm itself yields a trampolined deferred expression whenever the number of recursion steps is exhausted. This allows for parametric resumption.

This algorithm maintains a buffer of size recursion steps available at all times. This guarantees that the target has at least size recursion steps available whenever it is invoked.

The exponential structure of the X-suffixed algorithms yields a total of α recursion steps, where p is the number of parametric resumptions plus the initial entry, sω is CHAOS_PP_LIMIT_EXPR, s is the current recursion state at the beginning of the algorithm (i.e. CHAOS_PP_STATE()), sδ is the buffer size, and z is the exponential base. Currently, z is fixed at two in the CHAOS_PP_FOR_X algorithm (though it could easily be increased), which reduces the equation to β. Ignoring the complexity of pred, op, and macro (each of which are invoked once for each variation of the state), each iteration of the state requires only a single recursion step. This implies that the algorithm is capable of repeating the target γ times.

This macro deduces the current recursion state and invokes CHAOS_PP_FOR_AUX_X_S. If the current recursion state is already known, direct use of CHAOS_PP_FOR_AUX_X_S is more efficient.

This macro may not be used if bypass semantics are in effect.

CHAOS_PP_FOR_AUX_X_ID

The CHAOS_PP_FOR_AUX_X_ID macro indirectly expands to CHAOS_PP_FOR_AUX_X.

Usage 

CHAOS_PP_FOR_AUX_X_ID()

Remarks

This macro effectively allows CHAOS_PP_FOR_AUX_X to pass through itself without becoming disabled.

Remarks

CHAOS_PP_LAMBDA(CHAOS_PP_FOR_AUX_X_ID)()
