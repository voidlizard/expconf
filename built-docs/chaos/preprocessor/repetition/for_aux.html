The repetition/for_aux.h header defines a predicated repetition algorithm.

Usage

#include <chaos/preprocessor/repetition/for_aux.h>

CHAOS_PP_FOR_AUX

The CHAOS_PP_FOR_AUX macro is a predicated repetition construct.

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_FOR_AUX(pred, op, macro, data, ...))
pred

The predicate that determines whether the algorithm should continue. This predicate must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

pred(s, data†, state‡)

† If the auxiliary data is parenthetic, the data pseudo-parameter must be the number of parameters that corresponds to the number of sub-elements in the auxiliary data. Otherwise, it must be a single parameter.

‡ The state pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the initial state, __VA_ARGS__.

op

The operation used to iterate the state. This operation must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

op(s, data†, state‡)

† If the auxiliary data is parenthetic, the data pseudo-parameter must be the number of parameters that corresponds to the number of sub-elements in the auxiliary data. Otherwise, it must be a single parameter.

‡ The state pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the initial state, __VA_ARGS__.

macro

The target to be repeated. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, data†, state‡)

† If the auxiliary data is parenthetic, the data pseudo-parameter must be the number of parameters that corresponds to the number of sub-elements in the auxiliary data. Otherwise, it must be a single parameter.

‡ The state pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the initial state, __VA_ARGS__.

[data]

Auxiliary data to be passed through the algorithm to the predicate, operation, and target. This argument may be either a parenthetic expression of any arity or a non-parenthetic expression.

[...]

The initial state.

Remarks

The CHAOS_PP_FOR_AUX macro itself expands to a deferred expression in terms of CHAOS_PP_NEXT(s). This deferred expression, when invoked by CHAOS_PP_EXPR, expands to the result of repeating the target while iterating the state until the predicate expands to 0. This is illustrated by the following pseudo-code (where D is data, M is macro, and O is op): 

M(s, D, state) M(s, D, O(s, D, state)) ... M(s, D, O(s, D, ... O(s, D, state) ...))

The predicate, operation, and target are all invoked with the current recursion state, the auxiliary data (data), and the current state (state or __VA_ARGS__). The predicate must expands to 0, 1, or a deferred expression in terms of CHAOS_PP_NEXT(s) that expands to either 0 or 1. If the predicate ultimately expands to 0 the algorithm terminates. Otherwise, the target is invoked with the current state, the operation operation is applied to the current state to yield a new state, and the algorithm continues. Both the target and the operation may expand to deferred expressions in terms of CHAOS_PP_NEXT(s). Expansion to a deferred expression allows the predicate, operation, and target to recurse on themselves if necessary.

Invocations of the target (but not the predicate and operation) are trampolined such that each invocation is invoked with CHAOS_PP_NEXT(CHAOS_PP_NEXT(s)), where s is the original recursion state at the beginning of the algorithm (i.e. CHAOS_PP_STATE()). This maximizes the utility of the recursion backend.

The auxiliary data, data, is treated differently depending on whether it is parenthetic. If data is non-parenthetic, it is passed directly to the operation as a single argument. In C99, if data is a parenthetic expression of any arity, its sub-elements are passed to the predicate, operation, and target as individual arguments. In C (prior to C99) and C++, if data is a unary parenthetic expression, its sub-element is passed to the operation as a single argument (a parenthetic expression of any other arity is prohibited).

Ignoring the complexity of pred, op, and macro (each of which is invoked once for each variation of the state), each iteration of the state requires only a single recursion step.

This macro deduces the current recursion state and invokes CHAOS_PP_FOR_AUX_S. If the current recursion state is already known, direct use of CHAOS_PP_FOR_AUX_S is more efficient.

This macro may not be used if bypass semantics are in effect.

CHAOS_PP_FOR_AUX_ID

The CHAOS_PP_FOR_AUX_ID macro indirectly expands to CHAOS_PP_FOR_AUX.

Usage 

CHAOS_PP_FOR_AUX_ID()

Remarks

This macro effectively allows CHAOS_PP_FOR_AUX to pass through itself without becoming disabled.

CHAOS_PP_FOR_AUX_

The CHAOS_PP_FOR_AUX_ macro expands to a λ expression representing CHAOS_PP_FOR_AUX.

Usage - C99 Specific 

CHAOS_PP_FOR_AUX_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_FOR_AUX_ID)()
