The repetition/parametric/repeat_shifted.h header defines an algorithm that repeats a user-specified target a specified number times which can be parametrically resumed.

Usage

#include <chaos/preprocessor/repetition/parametric/repeat_shifted.h>

Contents

  • CHAOS_PP_REPEAT_SHIFTED_PARAMETRIC

CHAOS_PP_REPEAT_SHIFTED_PARAMETRIC

The CHAOS_PP_REPEAT_SHIFTED_PARAMETRIC macro repeats its second argument one less time than the number specified by its first argument.

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_REPEAT_SHIFTED_PARAMETRIC(count, macro))
count

One greater than the number of times that macro is to be repeated. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

macro

The target that is to be repeated. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, n)

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_REPEAT_SHIFTED_PARAMETRIC(count, macro, ...))
count

One greater than the number of times that macro is to be repeated. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

macro

The target that is to be repeated. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, n, data†)

† The data pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the auxiliary data, __VA_ARGS__.

[...]

Auxiliary data to be passed through the algorithm to the target.

Usage - C/C++ Specific 

CHAOS_PP_EXPR(CHAOS_PP_REPEAT_SHIFTED_PARAMETRIC(count, macro, data))
count

One greater than the number of times that macro is to be repeated. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

macro

The target that is to be repeated. This target must be a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, n, data)
data

Auxiliary data to be passed through the algorithm to the target.

Remarks

This macro behaves identically to CHAOS_PP_REPEAT_SHIFTED except that the algorithm itself yields a trampolined deferred expression whenever the number of recursion steps is exhausted. This allows for parametric resumption.

Ignoring the complexity of macro (which is invoked exactly count times), the efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Θ(n), where n is count. These bounds preserve all low-order terms, which implies that the algorithm can repeat the target min(p∙(sω-s-1), m) times, where p is the number of parametric resumptions plus the initial entry, sω is CHAOS_PP_LIMIT_EXPR, s is the original recursion state at the beginning of the algorithm (i.e. CHAOS_PP_STATE()), and m is CHAOS_PP_LIMIT_MAG.

This macro deduces the current recursion state and invokes CHAOS_PP_REPEAT_SHIFTED_PARAMETRIC_S. If the current recursion state is already known, direct use of CHAOS_PP_REPEAT_SHIFTED_PARAMETRIC_S is more efficient.

This macro may not be used if bypass semantics are in effect.

CHAOS_PP_REPEAT_SHIFTED_PARAMETRIC_ID

The CHAOS_PP_REPEAT_SHIFTED_PARAMETRIC_ID macro indirectly expands to CHAOS_PP_REPEAT_SHIFTED_PARAMETRIC.

Usage 

CHAOS_PP_REPEAT_SHIFTED_PARAMETRIC_ID()

Remarks

This macro effectively allows CHAOS_PP_REPEAT_SHIFTED_PARAMETRIC to pass through itself without becoming disabled.

CHAOS_PP_REPEAT_SHIFTED_PARAMETRIC_

The CHAOS_PP_REPEAT_SHIFTED_PARAMETRIC_ macro expands to a λ expression representing CHAOS_PP_REPEAT_SHIFTED_PARAMETRIC.

Usage - C99 Specific 

CHAOS_PP_REPEAT_SHIFTED_PARAMETRIC_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_REPEAT_SHIFTED_PARAMETRIC_ID)()
