The repetition/enum_params.h header defines an algorithm that enumerates a sequence of preprocessing tokens a specified number of times.

Usage

#include <chaos/preprocessor/repetition/enum_params.h>

Contents

  • CHAOS_PP_ENUM_PARAMS

CHAOS_PP_ENUM_PARAMS

The CHAOS_PP_ENUM_PARAMS macro enumerates its second argument the number of times specified by its first argument.

Usage - C99 Specific 

CHAOS_PP_ENUM_PARAMS(count, ...)
count

The number of times that __VA_ARGS__ is to be repeated. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

[...]

The sequence of preprocessing tokens to be repeated.

Usage - C/C++ Specific 

CHAOS_PP_ENUM_PARAMS(count, param)
count

The number of times that param is to be repeated. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

param

The sequence of preprocessing tokens to be repeated.

Remarks

This macro expands to the result of enumerating param or __VA_ARGS__ with a comma count times such that the rightmost preprocessing token of each repetition is concatenated to the position of the current repetition. This is illustrated by the following pseudo-code (where N is count - 1): 

param ## 0, param ## 1, ... param ## N

The rightmost preprocessing token of param or __VA_ARGS__ must be either pasteable to a number or be a binary parenthetic expression such as 

(a, b)

If param or __VA_ARGS__ is a binary parenthetic expression, this macro extracts the two elements and concatenates the position of the current repetition to each of them. This is illustrated by the following pseudo-code (where N is again count - 1): 

a ## 0 b ## 0, a ## 1 b ## 1, ... a ## N b ## N

In C99, the requirement that a parenthetic expression must be binary is relaxed. It can be any arity provided that there are at least two elements. 

(a, ...)

In that circumstance, the current repetition number is concatenated to both the first of the two elements and to the all the rest of the elements as a single unit.

This secondary mechanism is intended to make it easy to define parameter lists, such as the following: 

T0 p0, T1 p1, ... T2 p2

The efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Î˜(n+1), where n is count.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_ENUM_PARAMS_BYPASS must be used instead.

Sample Code

#include <chaos/preprocessor/facilities/intercept.h>
#include <chaos/preprocessor/punctuation/comma.h>
#include <chaos/preprocessor/repetition/enum_params.h>

CHAOS_PP_ENUM_PARAMS(3, class T)
// class T0, class T1, class T2

CHAOS_PP_ENUM_PARAMS(3, (class T, = U))
// class T0 = U0, class T1 = U1, class T2 = U2

CHAOS_PP_ENUM_PARAMS(3, (class T, = void CHAOS_PP_INTERCEPT))
// class T0 = void, class T1 = void, class T2 = void

CHAOS_PP_ENUM_PARAMS_ID

The CHAOS_PP_ENUM_PARAMS_ID macro indirectly expands to CHAOS_PP_ENUM_PARAMS.

Usage 

CHAOS_PP_ENUM_PARAMS_ID()

Remarks

This macro effectively allows CHAOS_PP_ENUM_PARAMS to pass through itself without becoming disabled.

CHAOS_PP_ENUM_PARAMS_BYPASS

The CHAOS_PP_ENUM_PARAMS_BYPASS macro behaves identically to CHAOS_PP_ENUM_PARAMS except that it is parametized by the bypass recursion state.

Usage - C99 Specific 

CHAOS_PP_ENUM_PARAMS_BYPASS(s, count, ...)
s

The current bypass recursion state.

count

The number of times that __VA_ARGS__ is to be repeated. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

[...]

The sequence of preprocessing tokens to be repeated.

Usage - C/C++ Specific 

CHAOS_PP_ENUM_PARAMS_BYPASS(s, count, param)
s

The current bypass recursion state.

count

The number of times that param is to be repeated. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

param

The sequence of preprocessing tokens to be repeated.

Remarks

If bypass semantics are already in effect, this macro must be used instead of CHAOS_PP_ENUM_PARAMS.
