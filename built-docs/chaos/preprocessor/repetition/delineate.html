The repetition/delineate.h header defines an algorithm that delineates a user-specified target a specified number of times.

Usage

#include <chaos/preprocessor/repetition/delineate.h>

Contents

  • CHAOS_PP_DELINEATE

CHAOS_PP_DELINEATE

The CHAOS_PP_DELINEATE macro delineates its third argument the number of times specified by its first argument.

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_DELINEATE(count, sep, macro))
count

The number of times that macro is to be repeated. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

sep

The delineator to be situated between each repetition of macro. This delineator must be a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

sep()
macro

The target that is to be repeated. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, n)

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_DELINEATE(count, sep, macro, ...))
count

The number of times that macro is to be repeated. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

sep

The delineator to be situated between each repetition of macro. This delineator must be a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

sep()
macro

The target that is to be repeated. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, n, data†)

† The data pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the auxiliary data, __VA_ARGS__.

[...]

Auxiliary data to be passed through the algorithm to the target.

Usage - C/C++ Specific 

CHAOS_PP_EXPR(CHAOS_PP_DELINEATE(count, sep, macro, data))
count

sep

sep()
macro

The target that is to be repeated. This target must be a macro name that refers to a macro with a signature compatible with the following pseudo-signature:

data

Auxiliary data to be passed through the algorithm to the target.

Remarks

The CHAOS_PP_DELINEATE macro itself expands to a deferred expression in terms of CHAOS_PP_NEXT(s). This deferred expression, when invoked by CHAOS_PP_EXPR, expands to the result of delineating the target with sep count times. This is illustrated by the following pseudo-code: 

macro(s, 0, data) sep() macro(s, 1, data) sep() ... macro(s, count - 1, data)

The target is invoked repeatedly (from 0 to count - 1) with the current recursion state (s), the current repetition (n), and, if it exists, the auxiliary data (data or __VA_ARGS__). This target may expand to a deferred expression in terms of CHAOS_PP_NEXT(s). This allows the target to recurse on itself if necessary.

In C99, it is possible to omit the auxiliary data argument altogether. This is different than passing "nothing" as the auxiliary data. 

CHAOS_PP_EXPR(
    CHAOS_PP_DELINEATE(10, SEP, MACRO,) // #1
    CHAOS_PP_DELINEATE(10, SEP, MACRO)  // #2
)

The first line passes "nothing" as the auxiliary data, which causes MACRO to be invoked as (e.g.)... 

MACRO(s, 0,)

The second line omits the auxiliary data, which causes MACRO to be invoked as (e.g.)... 

MACRO(s, 0)

The fundamental difference is that, in the first line, the auxiliary data is empty (which might have a valid meaning as data), but, in the second line, there is no auxiliary data. Hence, the algorithm is overloaded on a true-optional parameter.

Invocations of the target are trampolined such that each invocation is invoked with CHAOS_PP_NEXT(s), where s is the original recursion state at the beginning of the algorithm (i.e. CHAOS_PP_STATE()). This maximizes the utility of the recursion backend.

Ignoring the complexity of macro (which is invoked exactly count times), the efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Θ(n+1), where n is count.

An invocation of the delineator, sep, is placed between each invocation of the target. This is a nullary invocation that is not intended to be complex--it is prohibited from using the recursion backend. Typically, the sep argument is CHAOS_PP_EMPTY (which makes the function of this macro identical to CHAOS_PP_REPEAT), CHAOS_PP_COMMA (which makes the function of this macro identical to CHAOS_PP_ENUM), or a use of CHAOS_PP_IDENTITY for nearly everything else. It is also possible, though uncommon, to use CHAOS_PP_LPAREN, CHAOS_PP_RPAREN, or some other nullary macro.

This macro deduces the current recursion state and invokes CHAOS_PP_DELINEATE_S. If the current recursion state is already known, direct use of CHAOS_PP_DELINEATE_S is more efficient.

This macro may not be used if bypass semantics are in effect.

Sample Code

#include <chaos/preprocessor/cat.h>
#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/facilities/empty.h>
#include <chaos/preprocessor/facilities/identity.h>
#include <chaos/preprocessor/lambda/ops.h>
#include <chaos/preprocessor/punctuation/comma.h>
#include <chaos/preprocessor/recursion/expr.h>
#include <chaos/preprocessor/repetition/delineate.h>

#define MACRO(s, n, var) var ## n

CHAOS_PP_EXPR(CHAOS_PP_DELINEATE(
    3, CHAOS_PP_EMPTY, MACRO, x
))
// x0 x1 x2

CHAOS_PP_EXPR(CHAOS_PP_DELINEATE(
    3, CHAOS_PP_COMMA, MACRO, y
))
// y0, y1, y2

CHAOS_PP_EXPR(CHAOS_PP_DELINEATE(
    3, CHAOS_PP_IDENTITY(+), MACRO, z
))
// z0 + z1 + z2

#if CHAOS_PP_VARIADICS

CHAOS_PP_EXPR(CHAOS_PP_DELINEATE(
    3, CHAOS_PP_COMMA,
    CHAOS_PP_PRIMITIVE_CAT_(class T, CHAOS_PP_ARG(1))
))
// class T0, class T2, class T3

#endif

CHAOS_PP_DELINEATE_ID

The CHAOS_PP_DELINEATE_ID macro indirectly expands to CHAOS_PP_DELINEATE.

Usage 

CHAOS_PP_DELINEATE_ID()

Remarks

This macro effectively allows CHAOS_PP_DELINEATE to pass through itself without becoming disabled.

CHAOS_PP_DELINEATE_

The CHAOS_PP_DELINEATE_ macro expands to a λ expression representing CHAOS_PP_DELINEATE.

Usage - C99 Specific 

CHAOS_PP_DELINEATE_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_DELINEATE_ID)()
