The repetition/for.h header defines a predicated repetition algorithm.

Usage

#include <chaos/preprocessor/repetition/for.h>

CHAOS_PP_FOR

The CHAOS_PP_FOR macro is a predicated repetition construct.

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_FOR(pred, op, macro, ...))
pred

The predicate to be used to determine whether the algorithm should continue. This predicate must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

pred(s, state†)

† The state pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the initial state, __VA_ARGS__.

op

The operation to be used to iterate the state. This operation must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

op(s, state†)

† The state pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the initial state, __VA_ARGS__.

macro

The target to be repeated. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, state†)

† The state pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the initial state, __VA_ARGS__.

[...]

The initial state.

Usage - C/C++ Specific 

CHAOS_PP_EXPR(CHAOS_PP_FOR(pred, op, macro, state))

op

The operation to be used to iterate the state. This operation must be a macro name that refers to a macro with a signature compatible with the following pseudo-signature:

macro

The target to be repeated. This target must be a macro name that refers to a macro with a signature compatible with the following pseudo-signature:

Remarks

The CHAOS_PP_FOR macro itself expands to a deferred expression in terms of CHAOS_PP_NEXT(s). This deferred expression, when invoked by CHAOS_PP_EXPR, expands to the result of repeating the target while iterating the state until the predicate expands to 0. This is illustrated by the following pseudo-code: 

macro(s, state)  macro(s, op(s, state)) ... macro(s, op(s, ... op(s, state) ... ))

The predicate, operation, and target are all invoked with the current recursion state (s) and the current state (state or __VA_ARGS__). The predicate must expand to 0, 1, or a deferred expression in terms of CHAOS_PP_NEXT(s) that expands to either 0 or 1. If the predicate ultimately expands to 0, the algorithm terminates. Otherwise, the target is invoked with the current state, the operation is applied to the current state to yield a new state, and the algorithm continues. Both the target and the operation may expand to deferred expressions in terms of CHAOS_PP_NEXT(s). Expansion to a deferred expression allows the predicate, operation, and target to recurse on themselves if necessary.

Invocations of the target (but not the predicate and operation) are trampolined such that each invocation is invoked with CHAOS_PP_NEXT(CHAOS_PP_NEXT(s)), where s is the original recursion state at the beginning of the algorithm (i.e. CHAOS_PP_STATE()). This maximizes the utility of the recursion backend.

Ignoring the complexity of pred, op, and macro (each of which is invoked once for each variation of the state), each iteration of the state requires only a single recursion step.

This macro deduces the current recursion state and invokes CHAOS_PP_FOR_S. If the current recursion state is already known, direct use of CHAOS_PP_FOR_S is more efficient.

This macro may not be used if bypass semantics are in effect.

Sample Code

#include <chaos/preprocessor/arithmetic/dec.h>
#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/lambda/ops.h>
#include <chaos/preprocessor/logical/bool.h>
#include <chaos/preprocessor/recursion/expr.h>
#include <chaos/preprocessor/repetition/for.h>

#define PRED(s, x) CHAOS_PP_BOOL(x)
#define OP(s, x) CHAOS_PP_DEC(x)
#define MACRO(s, x) { x }

CHAOS_PP_EXPR(CHAOS_PP_FOR(
    PRED, OP, MACRO, 3
))
// { 3 } { 2 } { 1 }

#if CHAOS_PP_VARIADICS

??=include CHAOS_PP_PLACEHOLDERS(1)

CHAOS_PP_EXPR(CHAOS_PP_FOR(
    CHAOS_PP_BOOL_(_1), CHAOS_PP_DEC_(_1),
    CHAOS_PP_LAMBDA({) _1 },
    3
))
// { 3 } { 2 } { 1 }

??=include CHAOS_PP_PLACEHOLDERS(0)

#endif

CHAOS_PP_FOR_ID

The CHAOS_PP_FOR_ID macro indirectly expands to CHAOS_PP_FOR.

Usage 

CHAOS_PP_FOR_ID()

Remarks

This macro effectively allows CHAOS_PP_FOR to pass through itself without becoming disabled.

CHAOS_PP_FOR_

The CHAOS_PP_FOR_ macro expands to a λ expression representing CHAOS_PP_FOR.

Usage - C99 Specific 

CHAOS_PP_FOR_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_FOR_ID)()
