The lambda/trampoline.h header defines a macro used to abstract the difference between the trampolined invocation of a macro and a λ expression.

Usage

#include <chaos/preprocessor/lambda/trampoline.h>

Contents

  • CHAOS_PP_TRAMPOLINE

CHAOS_PP_TRAMPOLINE

The CHAOS_PP_TRAMPOLINE macro abstracts the difference between a trampolined invocation of a macro and a λ expression.

Usage - C99 Specific 

CHAOS_PP_TRAMPOLINE(x)()(s, x, ...)
x

The macro or λ expression to be invoked. If this argument is a macro, it must be a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, args†)

† The args pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in __VA_ARGS__ after intermediate expansion.

s

The recursion state in terms of which the resulting obstructed expression is to be generated.

[...]

The intermediate argument or arguments that are to comprise the trailing arguments passed to x.

Usage - C/C++ Specific 

CHAOS_PP_TRAMPOLINE(x)()(s, x, im)
x

The macro to be invoked. This argument must be a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, args†)

† The args pseudo-parameter must be the number of parameters that corresponds to the number of arguments in im after intermediate expansion.

s

The recursion state in terms of which the resulting obstructed expression is to be generated.

im

The intermediate argument that is to comprise the trailing arguments passed to x.

Remarks

This macro expands to an obstructed expression in terms of s that contains an invocation of x (regardless of whether x is a macro name or a λ expression). It also allows this obstructed expression, when fully invoked, to expand to a deferred expression in terms of CHAOS_PP_NEXT(s).

Normal usage consists of invoking CHAOS_PP_TRAMPOLINE without the trailing invocations, passing that result around, and finishing the invocation at another time. This avoids repeatedly detecting the difference between a macro name and a λ expression. 

#define MACRO(s, x) MACRO_I(s, x, CHAOS_PP_TRAMPOLINE(x))
#define MACRO_I(s, x, _x) _x()(s, x, ~)

An algorithm trampolines the invocation of a macro (or λ expression) if the entire algorithm is unrolled prior the actual invocation of that macro. Each macro is invoked with the same recursion state, thus minimizing recursion depth and maximizing the utility of the recursion backend.

This macro may not be used when bypass semantics are in effect.

Sample Code

#include <chaos/preprocessor/arithmetic/dec.h>
#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/control/if.h>
#include <chaos/preprocessor/lambda/ops.h>
#include <chaos/preprocessor/lambda/trampoline.h>
#include <chaos/preprocessor/punctuation/comma.h>
#include <chaos/preprocessor/punctuation/comma_if.h>
#include <chaos/preprocessor/recursion/basic.h>
#include <chaos/preprocessor/recursion/expr.h>
#include <chaos/preprocessor/tuple/eat.h>

#if CHAOS_PP_VARIADICS

#define REPEAT(count, macro, ...) \
    REPEAT_S(CHAOS_PP_STATE(), count, macro, __VA_ARGS__) \
    /**/
#define REPEAT_S(s, count, macro, ...) \
    CHAOS_PP_EXPR_S(s)(REPEAT_I( \
        CHAOS_PP_NEXT(s), CHAOS_PP_NEXT(s), count, \
        macro, CHAOS_PP_TRAMPOLINE(macro), (__VA_ARGS__) \
    )) \
    /**/

#else /* CHAOS_PP_VARIADICS */

#define REPEAT(count, macro, data) \
    REPEAT_S(CHAOS_PP_STATE(), count, macro, data) \
    /**/
#define REPEAT_S(s, count, macro, data) \
    CHAOS_PP_EXPR_S(s)(REPEAT_I( \
        CHAOS_PP_NEXT(s), CHAOS_PP_NEXT(s), count, \
        macro, CHAOS_PP_TRAMPOLINE(macro), (data) \
    )) \
    /**/

#endif /* CHAOS_PP_VARIADICS */

#define REPEAT_INDIRECT() REPEAT_I
#define REPEAT_I(s, o, count, macro, _m, pd) \
    CHAOS_PP_IF(count)( \
        REPEAT_II, CHAOS_PP_TUPLE_EAT(7) \
    )( \
        CHAOS_PP_OBSTRUCT(), s, o, \
        CHAOS_PP_DEC(count), macro, _m, pd \
    ) \
    /**/
#define REPEAT_II(_, s, o, count, macro, _m, pd) \
    CHAOS_PP_EXPR_S(s) _(REPEAT_INDIRECT _()( \
        CHAOS_PP_NEXT(s), o, count, macro, _m, pd \
    )) \
    _m()(o, macro, count CHAOS_PP_COMMA() CHAOS_PP_REM pd) \
    /**/

#define MACRO(s, n, id) CHAOS_PP_COMMA_IF(n) id ## n

CHAOS_PP_EXPR(REPEAT(3, MACRO, class T))
// class T0, class T1, class T2

#if CHAOS_PP_VARIADICS

??=include CHAOS_PP_PLACEHOLDERS(1)

CHAOS_PP_EXPR(REPEAT(
    3,
    CHAOS_PP_COMMA_IF_(_1) CHAOS_PP_PRIMITIVE_CAT_(class T, _1),
))
// class T0, class T1, class T2

??=include CHAOS_PP_PLACEHOLDERS(0)

#endif

CHAOS_PP_TRAMPOLINE_ID

The CHAOS_PP_TRAMPOLINE_ID macro indirectly expands to CHAOS_PP_TRAMPOLINE.

Usage 

CHAOS_PP_TRAMPOLINE_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_TRAMPOLINE_

The CHAOS_PP_TRAMPOLINE_ macro expands to a λ expression representing CHAOS_PP_TRAMPOLINE.

Usage - C99 Specific 

CHAOS_PP_TRAMPOLINE_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_TRAMPOLINE)
