The lambda/curry.h header defines a mechanism that curries the λ placeholders in a λ expression.

Usage

#include <chaos/preprocessor/lambda/curry.h>

Contents

  • CHAOS_PP_CURRY

CHAOS_PP_CURRY

The CHAOS_PP_CURRY macro curries the λ placeholders in a λ expression.

Usage - C99 Specific 

CHAOS_PP_CURRY(expr)
[expr]

The λ expression whose λ placeholders are to be curried.

Usage - C99 Specific 

CHAOS_PP_CURRY(expr, ...)
[expr]

The λ expression whose λ placeholders are to be curried.

[...]

The arguments used to replace λ placeholders in expr.

Remarks

This macro expands to a λ expression where the λ placeholders in expr have been replaced by arguments from __VA_ARGS__ or curried. All uses of the CHAOS_PP_SELF operator are also replaced.

As many λ placeholders as there are arguments in __VA_ARGS__ are replaced by those arguments, and the remaining λ placeholders are shifted down. For example, if E is a λ expression in terms of _0, _1, and _2, then... 

CHAOS_PP_CURRY(E, X)

...results in a λ expression with _0 replaced by X, _1 replaced by _0, and _2 replaced by _1. Further, if two arguments are applied 

CHAOS_PP_CURRY(E, X, Y)

...the resulting λ expression has _0 replaced by X, _1 replaced by Y, and _2 replaced by _0.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_CURRY_BYPASS must be used instead.

Sample Code

#include <chaos/preprocessor/cat.h>
#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/lambda/curry.h>
#include <chaos/preprocessor/lambda/invoke.h>
#include <chaos/preprocessor/lambda/ops.h>
#include <chaos/preprocessor/recursion/expr.h>

#if CHAOS_PP_VARIADICS

??=include CHAOS_PP_PLACEHOLDERS(1)

CHAOS_PP_INVOKE(
    CHAOS_PP_CAT_(class T, _0) CHAOS_PP_LAMBDA(=) _1,
    x, y
)
// CHAOS_PP_CAT_ID()(class T, x) = y

CHAOS_PP_INVOKE(
    CHAOS_PP_CURRY(
        CHAOS_PP_CURRY(
            CHAOS_PP_CAT_(class T, _0) CHAOS_PP_LAMBDA(=) _1,
            x
        ),
        y
    )
)
// CHAOS_PP_CAT_ID()(class T, x) = y

??=include CHAOS_PP_PLACEHOLDERS(0)

#endif

CHAOS_PP_CURRY_ID

The CHAOS_PP_CURRY_ID macro indirectly expands to CHAOS_PP_CURRY.

Usage - C99 Specific 

CHAOS_PP_CURRY_ID()

Remarks

This macro effectively allows CHAOS_PP_CURRY to pass through itself without becoming disabled.

CHAOS_PP_CURRY_

The CHAOS_PP_CURRY_ macro expands to a λ expression representing CHAOS_PP_CURRY.

Usage - C99 Specific 

CHAOS_PP_CURRY_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_CURRY_ID)()

CHAOS_PP_CURRY_BYPASS

The CHAOS_PP_CURRY_BYPASS macro behaves identically to CHAOS_PP_CURRY except that it is parametized by the bypass recursion state.

Usage - C99 Specific 

CHAOS_PP_CURRY_BYPASS(s, expr)
s

The current bypass recursion state.

[expr]

The λ expression whose λ placeholders are to be curried.

Usage - C99 Specific 

CHAOS_PP_CURRY_BYPASS(s, expr, ...)
s

The current bypass recursion state.

[expr]

The λ expression whose λ placeholders are to be curried.

[...]

The arguments used to replace λ placeholders in expr.

Remarks

If bypass semantics are already in effect, this macro must be used instead of CHAOS_PP_CURRY.
