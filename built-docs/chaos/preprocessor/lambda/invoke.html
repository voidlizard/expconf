The lambda/invoke.h header defines a macro that invokes a λ expression.

Usage

#include <chaos/preprocessor/lambda/invoke.h>

Contents

  • CHAOS_PP_INVOKE

CHAOS_PP_INVOKE

The CHAOS_PP_INVOKE macro invokes a λ expression.

Usage - C99 Specific 

CHAOS_PP_INVOKE(expr)
[expr]

The λ expression to be invoked.

Usage - C99 Specific 

CHAOS_PP_INVOKE(expr, ...)
[expr]

The λ expression to be invoked.

[...]

The arguments used to replace λ placeholders in expr.

Remarks

This macro produces the deferred expression represented by expr.

The __VA_ARGS__ arguments are used to replace λ placeholders in expr. The first of these corresponds to the zero-th λ placeholder, the second to the first, and so on.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_INVOKE_BYPASS must be used instead.

Sample Code

#include <chaos/preprocessor/cat.h>
#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/lambda/invoke.h>
#include <chaos/preprocessor/lambda/ops.h>
#include <chaos/preprocessor/recursion/expr.h>

#if CHAOS_PP_VARIADICS

??=include CHAOS_PP_PLACEHOLDERS(1)

CHAOS_PP_EXPR(CHAOS_PP_INVOKE(
    CHAOS_PP_CAT_(_0, CHAOS_PP_PRIMITIVE_CAT_(_1, _2)),
    x, y, z
))
// xyz

??=include CHAOS_PP_PLACEHOLDERS(0)

#endif

CHAOS_PP_INVOKE_ID

The CHAOS_PP_INVOKE_ID macro indirectly expands to CHAOS_PP_INVOKE.

Usage - C99 Specific 

CHAOS_PP_INVOKE_ID()

Remarks

This macro effectively allows CHAOS_PP_INVOKE to pass through itself without becoming disabled.

CHAOS_PP_INVOKE_

The CHAOS_PP_INVOKE_ macro expands to a λ expression representing CHAOS_PP_INVOKE.

Usage - C99 Specific 

CHAOS_PP_INVOKE_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_INVOKE_ID)()

CHAOS_PP_INVOKE_BYPASS

The CHAOS_PP_INVOKE_BYPASS macro behaves identically to CHAOS_PP_INVOKE except that it is parametized by the bypass recursion state.

Usage - C99 Specific 

CHAOS_PP_INVOKE_BYPASS(s, expr)
s

The current bypass recursion state.

[expr]

The λ expression to be invoked.

Usage - C99 Specific 

CHAOS_PP_INVOKE_BYPASS(s, expr, ...)
s

The current bypass recursion state.

[expr]

The λ expression to be invoked.

[...]

The arguments used to replace λ placeholders in expr.

Remarks

If bypass semantics are already in effect, this macro must be used instead of CHAOS_PP_INVOKE.

CHAOS_PP_INVOKE_BYPASS_ID

The CHAOS_PP_INVOKE_BYPASS_ID macro indirectly expands to CHAOS_PP_INVOKE_BYPASS.

Usage - C99 Specific 

CHAOS_PP_INVOKE_BYPASS_ID()

Remarks

This macro effectively allows CHAOS_PP_INVOKE_BYPASS to pass through itself without becoming disabled.

CHAOS_PP_INVOKE_BYPASS_

The CHAOS_PP_INVOKE_BYPASS_ macro expands to a λ expression representing CHAOS_PP_INVOKE_BYPASS.

Usage - C99 Specific 

CHAOS_PP_INVOKE_BYPASS_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_INVOKE_BYPASS_ID)()
