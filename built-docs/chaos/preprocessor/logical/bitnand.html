CHAOS_PP_BITNAND

The CHAOS_PP_BITNAND macro expands to the logical NAND of its operands.

Usage 

CHAOS_PP_BITNAND(x)(y)
x

The left operand. Valid values are 0 and 1.

y

The right operand. Valid values are 0 and 1.

Remarks

If x and y are both 1, this macro expands to 0. Otherwise, it expands to 1.

This macro does not perform Boolean conversions on its operands. If this conversion is desirable, CHAOS_PP_NAND may be used instead.

This macro is lazy. If x is 0, y is not evaluated.

See Also

  • CHAOS_PP_NAND

Sample Code

#include <chaos/preprocessor/debug/failure.h>
#include <chaos/preprocessor/logical/bitnand.h>

CHAOS_PP_BITNAND(1)(1) // 0
CHAOS_PP_BITNAND(1)(0) // 1

CHAOS_PP_BITNAND(0)(CHAOS_PP_FAILURE()) // 1

CHAOS_PP_BITNAND_ID

The CHAOS_PP_BITNAND_ID macro indirectly expands to CHAOS_PP_BITNAND.

Usage 

CHAOS_PP_BITNAND_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_BITNAND_

The CHAOS_PP_BITNAND_ macro expands to a λ expression representing CHAOS_PP_BITNAND.

Usage - C99 Specific 

CHAOS_PP_BITNAND_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_BITNAND)
