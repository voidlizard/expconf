CHAOS_PP_BITXNOR

The CHAOS_PP_BITXNOR macro expands to the logical XNOR of its operands.

Usage 

CHAOS_PP_BITXNOR(x)(y)
x

The left operand. Valid values are 0 and 1.

y

The right operand. Valid values are 0 and 1.

Remarks

If either x or y is 1 exclusively, this macro expands to 0. Otherwise, it expands to 1.

This macro does not perform Boolean conversions on its operands. If this conversion is desirable, CHAOS_PP_XNOR may be used instead.

See Also

  • CHAOS_PP_XNOR

Sample Code

#include <chaos/preprocessor/debug/failure.h>
#include <chaos/preprocessor/logical/bitxnor.h>

CHAOS_PP_BITXNOR(0)(1) // 0
CHAOS_PP_BITXNOR(1)(1) // 1

CHAOS_PP_BITXNOR_ID

The CHAOS_PP_BITXNOR_ID macro indirectly expands to CHAOS_PP_BITXNOR.

Usage 

CHAOS_PP_BITXNOR_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_BITXNOR_

The CHAOS_PP_BITXNOR_ macro expands to a λ expression representing CHAOS_PP_BITXNOR.

Usage - C99 Specific 

CHAOS_PP_BITXNOR_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_BITXNOR)
