The recursion/rail.h header defines a macro used to delay the invocation of another macro through an arbitrary number of scans.

Usage

#include <chaos/preprocessor/recursion/rail.h>

Contents

CHAOS_PP_RAIL

The CHAOS_PP_RAIL macro creates a rail of its argument.

Usage 

CHAOS_PP_RAIL(macro)
macro

The macro whose invocation is to be delayed.

Remarks

This macro delays the invocation of macro until the context introduced by CHAOS_PP_WALL is found.

Normal usage includes the invocation of this macro followed by the argument list to macro. For example, 

#define MACRO(a, b, c) // ...
CHAOS_PP_RAIL(MACRO)(~, ~, ~)

If and when the CHAOS_PP_WALL context is found, this macro expands to a deferred expression that contains an invocation of macro against the trailing arguments.

Sample Code

#include <chaos/preprocessor/punctuation/comma.h>
#include <chaos/preprocessor/punctuation/paren.h>
#include <chaos/preprocessor/recursion/expr.h>
#include <chaos/preprocessor/recursion/rail.h>

#define A(x) B(x)
#define B(x) C(x)
#define C(x) x

A(CHAOS_PP_COMMA()) // error

CHAOS_PP_EXPR(CHAOS_PP_WALL(
    A(
        CHAOS_PP_RAIL(CHAOS_PP_RPAREN)()
        CHAOS_PP_RAIL(CHAOS_PP_COMMA)()
        CHAOS_PP_RAIL(CHAOS_PP_LPAREN)()
    )
))
// ) , (

CHAOS_PP_UNSAFE_RAIL

The CHAOS_PP_UNSAFE_RAIL macro creates a rail of its argument.

Usage 

CHAOS_PP_UNSAFE_RAIL(macro)
macro

The macro whose invocation is to be delayed.

Remarks

This macro delays the invocation of macro until the context introduced by CHAOS_PP_WALL is found.

Normal usage includes the invocation of this macro followed by the argument list to macro. For example, 

#define MACRO(a, b, c) // ...
CHAOS_PP_UNSAFE_RAIL(MACRO)(~, ~, ~)

If and when the CHAOS_PP_WALL context is found, this macro expands to an invocation of macro against the trailing arguments. This is similar in function to CHAOS_PP_RAIL, but it does not yield a deferred expression. The direct invocation makes CHAOS_PP_UNSAFE_RAIL less robust because macro is expanded in the CHAOS_PP_WALL context (which implies that no rails can result from the invocation of macro). The benefit of CHAOS_PP_UNSAFE_RAIL is that no outer CHAOS_PP_EXPR is required, which is syntactically cleaner.

CHAOS_PP_WALL

The CHAOS_PP_WALL macro causes rails to expand.

Usage - C99 Specific 

CHAOS_PP_WALL(...)
[...]

The argument whose rails are to be expanded.

Usage - C/C++ Specific 

CHAOS_PP_WALL(x)
x

The argument whose rails are to be expanded.

Remarks

This macro is an identity macro that expands to x or __VA_ARGS__ except that all rails created by CHAOS_PP_RAIL that are contained in x or __VA_ARGS__ are expanded to deferred expressions. Consequently, normal usage includes a wrapping CHAOS_PP_EXPR or similar to evaluate the resulting deferred expressions if any. For example, 

CHAOS_PP_EXPR(CHAOS_PP_WALL(...))
