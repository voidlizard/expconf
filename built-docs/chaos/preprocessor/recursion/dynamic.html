The recursion/dynamic.h header defines a macro that utilizes a sequence to build a parametric application of the recursion backend.

Usage

#include <chaos/preprocessor/recursion/dynamic.h>

Contents

  • CHAOS_PP_DYNAMIC

CHAOS_PP_DYNAMIC

The CHAOS_PP_DYNAMIC macro utilizes its first argument to build a parametric application of the recursion backend around a macro invocation.

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_DYNAMIC(seq, macro, args))
[seq]

The sequence to be used to generate the parametric application.

macro

The macro to be invoked inside the parametric application.

args

The argument list to be applied to macro.

Usage - C/C++ Specific 

CHAOS_PP_EXPR(CHAOS_PP_DYNAMIC(seq, macro, args))
seq

The sequence to be used to generate the parametric application.

macro

The macro to be invoked inside the parametric application.

args

The argument list to be applied to macro.

Remarks

The CHAOS_PP_DYNAMIC macro itself expands to a deferred expression in terms of CHAOS_PP_NEXT(s). This deferred expression contains as many parametric applications of CHAOS_PP_EXPR_S(s)(...) surrounding the application of args to macro as there are elements in seq. For example, if CHAOS_PP_NEXT(s) is 2, the expression 

CHAOS_PP_DYNAMIC((1)(2)(3), MACRO, (...))

results in the deferred expression 

CHAOS_PP_EXPR_S(2)(CHAOS_PP_EXPR_S(2)(CHAOS_PP_EXPR_S(2)(
    MACRO(...)
)))

This deferred expression, when invoked by CHAOS_PP_EXPR, expands to the result of MACRO(...) scanned three additional times. I.e. the expression MACRO(...) is scanned one time more than there are elements in seq.

The primary utility of this macro is to provide recursion steps for an algorithm whose recursion requirements are linear (or better) and easily computed as a sequence from its input--while guaranting constant usage of the recursion backend. This is often the case with algorithms that operate on sequences.

Note that the number of parametric applications can be doubled simply by doubling seq. This might be done, for example, by an algorithm that requires two recursion steps for each element in a sequence. 

#define ALGO(seq) \
    CHAOS_PP_DYNAMIC(seq seq, ALGO_I, (seq)) \
    /**/
#define ALGO_I(seq) // ...

A fixed number of additional steps can be added simply by adding sequence elements, such as seq(~)(~), as the contents of each element are irrelevent.

Because the recursion steps needed by an algorithm are supplied via the external parametric applications, an algorithm need not internally cascade the recursion backend (which is the usual case in recursive algorithms). Instead, it can just generate a deferred expression whenever it needs to recurse.

This macro deduces the current recursion state and invokes CHAOS_PP_DYNAMIC_S. If the current recursion state is already known, direct use of CHAOS_PP_DYNAMIC_S is more efficient.

This macro may not be used if bypass semantics are in effect.

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/control/iif.h>
#include <chaos/preprocessor/recursion/basic.h>
#include <chaos/preprocessor/recursion/dynamic.h>
#include <chaos/preprocessor/recursion/expr.h>
#include <chaos/preprocessor/seq/core.h>
#include <chaos/preprocessor/tuple/eat.h>

#define FOR_EACH(macro, seq, data) \
    CHAOS_PP_DYNAMIC(seq, FOR_EACH_A, (macro, seq, data)) \
    /**/

#define FOR_EACH_INDIRECT() FOR_EACH_A
#define FOR_EACH_A(macro, seq, data) \
    CHAOS_PP_IIF(CHAOS_PP_SEQ_IS_CONS(seq))( \
        FOR_EACH_B, CHAOS_PP_TUPLE_EAT(4) \
    )(CHAOS_PP_OBSTRUCT(), macro, seq, data) \
    /**/
#define FOR_EACH_B(_, macro, seq, data) \
    macro _(CHAOS_PP_SEQ_HEAD(seq), data) \
    FOR_EACH_INDIRECT _()(macro, CHAOS_PP_SEQ_TAIL(seq), data) \
    /**/

#define M(x, _) < x >

CHAOS_PP_EXPR(FOR_EACH(
    M, (a)(b)(c)(d)(e)(f)(g)(h)(i)(j), ~
))
// < a > < b > < c > < d > < e > < f > < g > < h > < i > < j >

CHAOS_PP_DYNAMIC_ID

The CHAOS_PP_DYNAMIC_ID macro indirectly expands to CHAOS_PP_DYNAMIC.

Usage 

CHAOS_PP_DYNAMIC_ID()

Remarks

This macro effectively allows CHAOS_PP_DYNAMIC to pass through itself without becoming disabled.

CHAOS_PP_DYNAMIC_

The CHAOS_PP_DYNAMIC_ macro expands to a λ expression representing CHAOS_PP_DYNAMIC.

Usage - C99 Specific 

CHAOS_PP_DYNAMIC_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_DYNAMIC_ID)()

CHAOS_PP_DYNAMIC_S

The CHAOS_PP_DYNAMIC_S macro behaves identically to CHAOS_PP_DYNAMIC except that it is parametized by the recursion state.

Usage - C99 Specific 

CHAOS_PP_EXPR_S(s)(CHAOS_PP_DYNAMIC_S(s, seq, macro, args))
s

The current recursion state.

[seq]

The sequence to be used to generate the parametric application.

macro

The macro to be invoked inside the parametric application.

args

The argument list to be applied to macro.

Usage - C++ Specific 

CHAOS_PP_EXPR_S(s)(CHAOS_PP_DYNAMIC_S(s, seq, macro, args))
s

The current recursion state.

seq

The sequence to be used to generate the parametric application.

macro

The macro to be invoked inside the parametric application.

args

The argument list to be applied to macro.

Remarks

This macro may not be used if bypass semantics are in effect.
