CHAOS_PP_ASSERT

The CHAOS_PP_ASSERT macro intentionally causes an error if its argument is zero.

Usage 

CHAOS_PP_ASSERT(cond)
cond

The condition used to trigger the assertion. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

Remarks

If cond evaluates to non-zero, this macro expands to nothing. Otherwise, it intentionally causes an error.

See Also

  • CHAOS_PP_ASSERT_MSG
  • CHAOS_PP_FAILURE

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/debug/assert.h>

CHAOS_PP_ASSERT(CHAOS_PP_VARIADICS)

CHAOS_PP_ASSERT_ID

The CHAOS_PP_ASSERT_ID macro indirectly expands to CHAOS_PP_ASSERT.

Usage 

CHAOS_PP_ASSERT_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_ASSERT_

The CHAOS_PP_ASSERT_ macro expands to a λ expression representing CHAOS_PP_ASSERT.

Usage - C99 Specific 

CHAOS_PP_ASSERT_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_ASSERT)

CHAOS_PP_ASSERT_MSG

The CHAOS_PP_ASSERT_MSG macro expands to a user-specified message if its first argument is 0.

Usage - C99 Specific 

CHAOS_PP_ASSERT_MSG(cond, ...)
cond

The condition used to trigger the assertion. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

[...]

The resulting message if cond is 0.

Usage - C/C++ Specific 

CHAOS_PP_ASSERT_MSG(cond, msg)
cond

The condition used to trigger the assertion. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

msg

The resulting message if cond is 0.

Remarks

If cond evaluates to non-zero, this macro expands to nothing. Otherwise, it expands to msg or __VA_ARGS__.

See Also

  • CHAOS_PP_ASSERT
  • CHAOS_PP_FAILURE

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/debug/assert.h>

CHAOS_PP_ASSERT_MSG(CHAOS_PP_VARIADICS, "Variadic macros are required";)
