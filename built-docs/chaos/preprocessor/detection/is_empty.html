The detection/is_empty.h header defines macros that detect emptiness.

Usage

#include <chaos/preprocessor/detection/is_empty.h>

Contents

  • CHAOS_PP_IS_EMPTY
  • CHAOS_PP_IS_EMPTY_NON_FUNCTION

CHAOS_PP_IS_EMPTY

The CHAOS_PP_IS_EMPTY macro detects whether its argument is defined as a configuration flag with an empty replacement list.

Usage 

CHAOS_PP_IS_EMPTY(id)
id

The identifier to be tested.

Remarks

This macro detects whether its argument is defined with an empty replacement list as opposed to defined as 0, defined as 1, or undefined.

This macro is not a general purpose emptiness detection macro. The id argument must expand to 0, expand to 1, expand to nothing, or not expand at all. Any other input is undefined.

See Also

  • CHAOS_PP_IS_FLAG
  • CHAOS_PP_IS_ONE

Sample Code

#include <chaos/preprocessor/detection/is_empty.h>

#define A
#define B 0
#define C 1

CHAOS_PP_IS_EMPTY(A) // 1
CHAOS_PP_IS_EMPTY(B) // 0
CHAOS_PP_IS_EMPTY(C) // 0
CHAOS_PP_IS_EMPTY(D) // 0

CHAOS_PP_IS_EMPTY_ID

The CHAOS_PP_IS_EMPTY_ID macro indirectly expands to CHAOS_PP_IS_EMPTY.

Usage 

CHAOS_PP_IS_EMPTY_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_IS_EMPTY_

The CHAOS_PP_IS_EMPTY_ macro expands to a λ expression representing CHAOS_PP_IS_EMPTY.

Usage - C99 Specific 

CHAOS_PP_IS_EMPTY_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_IS_EMPTY)

CHAOS_PP_IS_EMPTY_NON_FUNCTION

The CHAOS_PP_IS_EMPTY_NON_FUNCTION macro is a general purpose, but limited, emptiness-detection primitive.

Usage - C99 Specific 

CHAOS_PP_IS_EMPTY_NON_FUNCTION(...)
[...]

The expression to be tested.

Remarks

If __VA_ARGS__ is empty, this macro expands to 1. Otherwise, it expands to 0.

Though this macro is a general purpose macro (unlike CHAOS_PP_IS_EMPTY), it is not a complete solution to emptiness detection. Specifically, __VA_ARGS__ may not terminate with the name of a function-like macro: 

#define FUNC() /* ... */

CHAOS_PP_IS_EMPTY_NON_FUNCTION(FUNC) // invalid

(Full-fledged detection of emptiness is not possible without well-defined token-pasting semantics when token-pasting does not yield a single preprocessing token.)

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/detection/is_empty.h>

#if CHAOS_PP_VARIADICS

#define AB

CHAOS_PP_IS_EMPTY_NON_FUNCTION(AB) // 1
CHAOS_PP_IS_EMPTY_NON_FUNCTION(&&) // 0

#endif
