The arithmetic/add.h header defines an addition operator.

Usage

#include <chaos/preprocessor/arithmetic/add.h>

Contents

  • CHAOS_PP_ADD

CHAOS_PP_ADD

The CHAOS_PP_ADD macro computes the sum of two values.

Usage 

CHAOS_PP_ADD(x, y)
x

The first addend. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

y

The second addend. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

Remarks

This macro expands to the sum of x and y. If the result is greater than CHAOS_PP_LIMIT_MAG, it is saturated to CHAOS_PP_LIMIT_MAG.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_ADD_BYPASS must be used instead.

See Also

  • CHAOS_PP_ADD_MACHINE
  • CHAOS_PP_ARBITRARY_ADD

Sample Code

#include <chaos/preprocessor/arithmetic/add.h>

CHAOS_PP_ADD(11, 12) // 23

CHAOS_PP_ADD_ID

The CHAOS_PP_ADD_ID macro indirectly expands to CHAOS_PP_ADD.

Usage 

CHAOS_PP_ADD_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_ADD_

The CHAOS_PP_ADD_ macro expands to a λ expression representing CHAOS_PP_ADD.

Usage - C99 Specific 

CHAOS_PP_ADD_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_ADD)

CHAOS_PP_ADD_BYPASS

The CHAOS_PP_ADD_BYPASS macro behaves identically to CHAOS_PP_ADD except that it is parametized by the bypass recursion state.

Usage 

CHAOS_PP_ADD_BYPASS(s, x, y)
s

The current bypass recursion state.

x

The first addend. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

y

The second addend. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

Remarks

If bypass semantics are already in effect, this macro must be used instead of CHAOS_PP_ADD.

CHAOS_PP_ADD_BYPASS_ID

The CHAOS_PP_ADD_BYPASS_ID macro indirectly expands to CHAOS_PP_ADD_BYPASS.
