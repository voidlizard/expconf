The arithmetic/in_range.h header defines a predicate that determines if an integer literal is in range.

Usage

#include <chaos/preprocessor/arithmetic/in_range.h>

Contents

  • CHAOS_PP_IN_RANGE

CHAOS_PP_IN_RANGE

The CHAOS_PP_IN_RANGE macro is a predicate that tests whether its argument is greater than or equal to 0 and less than or equal to CHAOS_PP_LIMIT_MAG.

Usage 

CHAOS_PP_IN_RANGE(x)
x

The integer literal to be tested.

Remarks

If x is the greater than or equal to 0 and less than or equal to CHAOS_PP_LIMIT_MAG, this macro expands to 1. Otherwise, it expands to 0.

Sample Code

#include <chaos/preprocessor/arithmetic/in_range.h>

CHAOS_PP_IN_RANGE(123)     // 1
CHAOS_PP_IN_RANGE(1000000) // 0

CHAOS_PP_IN_RANGE_ID

The CHAOS_PP_IN_RANGE_ID macro indirectly expands to CHAOS_PP_IN_RANGE.

Usage 

CHAOS_PP_IN_RANGE_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_IN_RANGE_

The CHAOS_PP_IN_RANGE_ macro expands to a λ expression representing CHAOS_PP_IN_RANGE.

Usage - C99 Specific 

CHAOS_PP_IN_RANGE_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_IN_RANGE)
