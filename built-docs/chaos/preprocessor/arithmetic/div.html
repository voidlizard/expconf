Usage

#include <chaos/preprocessor/arithmetic/div.h>

CHAOS_PP_DIV

The CHAOS_PP_DIV macro computes the quotient produced by dividing two values.

Usage 

CHAOS_PP_DIV(x, y)
x

The dividend. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

y

The divisor. Valid values range from 1 to CHAOS_PP_LIMIT_MAG.

Remarks

This macro expands to the quotient produced by dividing x by y. If y is 0, this macro intentionally produces an error.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_DIV_BYPASS must be used instead.

Sample Code

CHAOS_PP_DIV(10, 3) // 3

CHAOS_PP_DIV_ID

The CHAOS_PP_DIV_ID macro indirectly expands to CHAOS_PP_DIV.

Usage 

CHAOS_PP_DIV_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_DIV_

The CHAOS_PP_DIV_ macro expands to a λ expression representing CHAOS_PP_DIV.

Usage - C99 Specific 

CHAOS_PP_DIV_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_DIV)

CHAOS_PP_DIV_BYPASS

The CHAOS_PP_DIV_BYPASS macro behaves identically to CHAOS_PP_DIV except that it is parametized by the bypass recursion state.

Usage 

CHAOS_PP_DIV_BYPASS(s, x, y)
s

The current bypass recursion state.

x

The dividend. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

y

The divisor. Valid values range from 1 to CHAOS_PP_LIMIT_MAG.

Remarks

If bypass semantics are already in effect, this macro must be used instead of CHAOS_PP_DIV.

CHAOS_PP_DIV_BYPASS_ID

The CHAOS_PP_DIV_BYPASS_ID macro indirectly expands to CHAOS_PP_DIV_BYPASS.

Remarks

CHAOS_PP_DIV_BYPASS_

Usage - C99 Specific
