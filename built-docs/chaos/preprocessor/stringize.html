The stringize.h header defines stringizing macros.

Usage

#include <chaos/preprocessor/stringize.h>

Contents

  • CHAOS_PP_PRIMITIVE_STRINGIZE

CHAOS_PP_STRINGIZE

The CHAOS_PP_STRINGIZE macro stringizes its argument after it has been expanded.

Usage - C99 Specific 

CHAOS_PP_STRINGIZE(...)
[...]

The argument to be stringized.

Usage - C/C++ Specific 

CHAOS_PP_STRINGIZE(x)
x

The argument to be stringized.

Remarks

This macro stringizes its argument with the stringizing operator. 

#__VA_ARGS__ // C99
#x           // C and C++

This macro allows its argument to expand on input. This macro's sibling, CHAOS_PP_PRIMITIVE_STRINGIZE, does not provide this delay.

Both leading and trailing whitespace are removed from the argument when it is stringized, and internal whitespace separations are each condensed to a single space character. 

CHAOS_PP_STRINGIZE( abc ) // "abc"
CHAOS_PP_STRINGIZE(1   2) // "1 2"

In C99, if __VA_ARGS__ expands to a placemarker, this macro expands to an empty string literal. 

CHAOS_PP_STRINGIZE() // ""

Sample Code

#define A text

CHAOS_PP_STRINGIZE(A) // "text"

CHAOS_PP_STRINGIZE_ID

The CHAOS_PP_STRINGIZE_ID macro indirectly expands to CHAOS_PP_STRINGIZE.

Usage 

CHAOS_PP_STRINGIZE_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_STRINGIZE_

The CHAOS_PP_STRINGIZE_ macro expands to a λ expression representing CHAOS_PP_STRINGIZE.

Usage - C99 Specific 

CHAOS_PP_STRINGIZE_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_STRINGIZE)

CHAOS_PP_PRIMITIVE_STRINGIZE

The CHAOS_PP_PRIMITIVE_STRINGIZE macro stringizes its argument before it is expanded.

Usage - C99 Specific 

CHAOS_PP_PRIMITIVE_STRINGIZE(...)
[...]

The argument to be stringized.

Usage - C/C++ Specific 

CHAOS_PP_PRIMITIVE_STRINGIZE(x)
x

The argument to be stringized.

Remarks

This macro behaves identically to CHAOS_PP_STRINGIZE except that it does not allow its argument to expand before it is stringized.
