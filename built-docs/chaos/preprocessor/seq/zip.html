The seq/zip.h header defines a macro that encapsulates each element of a sequence.

Usage

#include <chaos/preprocessor/seq/zip.h>

Contents

CHAOS_PP_SEQ_ZIP

The CHAOS_PP_SEQ_ZIP macro encapsulates each element of its argument.

Usage - C99 Specific 

CHAOS_PP_SEQ_ZIP(seq)
[seq]

The sequence whose elements are to be encapsulated.

Remarks

This macro expands to a sequence containing each element of seq enclosed in parentheses. This is illustrated by the following pseudo-code: 

((seq[0]))((seq[1])) ... ((seq[length(seq) - 1]))

This macro is used primarily by algorithms that operate specifically on variadic sequences. Normally, such algorithms "zip" the entire sequence on which they operate and invoke the corresponding non-variadic sequence algorithm. (The purpose of doing such a thing is to prevent an argument clash with the state or auxiliary data arguments of various algorithms.)

CHAOS_PP_SEQ_ZIP_ID

The CHAOS_PP_SEQ_ZIP_ID macro indirectly expands to CHAOS_PP_SEQ_ZIP.

Usage - C99 Specific 

CHAOS_PP_SEQ_ZIP_ID()

Remarks

This macro effectively allows CHAOS_PP_SEQ_ZIP to pass through itself without becoming disabled.

CHAOS_PP_SEQ_ZIP_

The CHAOS_PP_SEQ_ZIP_ macro expands to a λ expression representing CHAOS_PP_SEQ_ZIP.

Usage - C99 Specific 

CHAOS_PP_SEQ_ZIP_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_SEQ_ZIP_ID)()

CHAOS_PP_SEQ_UNZIP

The CHAOS_PP_SEQ_UNZIP macro restores a sequence previously encoded with CHAOS_PP_SEQ_ZIP.

Usage - C99 Specific 

CHAOS_PP_SEQ_UNZIP(seq)
[seq]

The sequence to be restored.

Remarks

This macro removes the parentheses from each element of seq. Consequently, each element of seq must actually be parenthesized. 

CHAOS_PP_SEQ_UNZIP(
    ((a, b))((x, y, z))((p))
)
// (a, b)(x, y, z)(p)

This macro is used primarily by some algorithms that operate specifically on variadic sequences. Such algorithms "zip" the entire sequence on which they operate and invoke the corresponding non-variadic sequence algorithm. The result of that invocation is subsequently "unzipped" to yield the final result. (An example of such an algorithm is CHAOS_PP_VARIADIC_SEQ_FILTER.)
