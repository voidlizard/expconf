The algorithm/enumerate.h header defines an algorithm that enumerates the elements of a generic data structure.

Usage

#include <chaos/preprocessor/algorithm/enumerate.h>

Contents

CHAOS_PP_ENUMERATE

The CHAOS_PP_ENUMERATE macro enumerates the elements of its argument.

Usage 

CHAOS_PP_ENUMERATE(g)
g

The generic data structure whose elements are to be enumerated.

Remarks

This macro expands to a comma-separated list containing the elements of g.

The efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Θ(n+1), where n is the number of elements in g.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_ENUMERATE_BYPASS must be used instead.

CHAOS_PP_ENUMERATE_ID

The CHAOS_PP_ENUMERATE_ID macro indirectly expands to CHAOS_PP_ENUMERATE.

Usage 

CHAOS_PP_ENUMERATE_ID()

Remarks

This macro effectively allows CHAOS_PP_ENUMERATE to pass through itself without becoming disabled.

CHAOS_PP_ENUMERATE_

The CHAOS_PP_ENUMERATE_ macro expands to a λ expression representing CHAOS_PP_ENUMERATE.

Usage - C99 Specific 

CHAOS_PP_ENUMERATE_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_ENUMERATE_ID)()

CHAOS_PP_ENUMERATE_BYPASS

The CHAOS_PP_ENUMERATE_BYPASS macro behaves identically to CHAOS_PP_ENUMERATE except that it is parametized by the bypass recursion state.

Usage 

CHAOS_PP_ENUMERATE_BYPASS(s, g)
s

The current bypass recursion state.

g

The generic data structure whose elements are to be enumerated.

Remarks

If bypass semantics are already in effect, this macro must be used instead of CHAOS_PP_ENUMERATE.
