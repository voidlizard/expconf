The algorithm/take.h header defines an algorithm that extracts a user-specified number of elements from the beginning of a generic data structure.

Usage

#include <chaos/preprocessor/algorithm/take.h>

Contents

  • CHAOS_PP_TAKE

CHAOS_PP_TAKE

The CHAOS_PP_TAKE macro extracts the number of elements specified by its first argument from its second argument.

Usage 

CHAOS_PP_TAKE(n, g)
n

The number of elements to be extracted. Valid values range from 0 to CHAOS_PP_SIZE(g).

g

The generic data structure from which elements are to be extracted.

Remarks

This macro expands to a generic data structure of the same type as g containing the first n elements of g.

The efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Θ(n+1), where n is the argument n.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_TAKE_BYPASS must be used instead.

Sample Code

#include <chaos/preprocessor/algorithm/take.h>
#include <chaos/preprocessor/array/core.h>

CHAOS_PP_TAKE(2, (CHAOS_PP_ARRAY) (5, (a, b, c, d, e)))
// (CHAOS_PP_ARRAY) (2, (a, b))

CHAOS_PP_TAKE_ID

The CHAOS_PP_TAKE_ID macro indirectly expands to CHAOS_PP_TAKE.

Usage 

CHAOS_PP_TAKE_ID()

Remarks

This macro effectively allows CHAOS_PP_TAKE to pass through itself without becoming disabled.

CHAOS_PP_TAKE_

The CHAOS_PP_TAKE_ macro expands to a λ expression representing CHAOS_PP_TAKE.

Usage - C99 Specific 

CHAOS_PP_TAKE_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_TAKE_ID)()

CHAOS_PP_TAKE_BYPASS

The CHAOS_PP_TAKE_BYPASS macro behaves identically to CHAOS_PP_TAKE except that it is parametized by the bypass recursion state.

Usage 

CHAOS_PP_TAKE_BYPASS(s, n, g)
s

The current bypass recursion state.

n

The number of elements to be extracted. Valid values range from 0 to CHAOS_PP_SIZE(g).

g

The generic data structure from which elements are to be extracted.

Remarks

If bypass semantics are already in effect, this macro must be used instead of CHAOS_PP_TAKE.

CHAOS_PP_TAKE_BYPASS_ID

The CHAOS_PP_TAKE_BYPASS_ID macro indirectly expands to CHAOS_PP_TAKE_BYPASS.

Usage 

CHAOS_PP_TAKE_BYPASS_ID()

Remarks

CHAOS_PP_TAKE_BYPASS_

Usage - C99 Specific
