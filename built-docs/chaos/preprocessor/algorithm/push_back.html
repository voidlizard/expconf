The algorithm/push_back.h header defines an algorithm that appends an element to the end of a generic data structure.

Usage

#include <chaos/preprocessor/algorithm/push_back.h>

Contents

  • CHAOS_PP_PUSH_BACK

CHAOS_PP_PUSH_BACK

The CHAOS_PP_PUSH_BACK macro appends its second argument to the tail of its first argument.

Usage - C99 Specific 

CHAOS_PP_PUSH_BACK(g, ...)
g

The generic data structure to which the element is to be appended.

[...]

The element to be appended to g.

Usage - C/C++ Specific 

CHAOS_PP_PUSH_BACK(g, x)
g

The generic data structure to which the element is to be appended.

x

The element to be appended to g.

Remarks

This macro expands to a generic data structure of the same type as g that contains the elements of g with the element (x or __VA_ARGS__) appended to the tail.

The efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Θ(n+1), where n is the number of elements in g.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_PUSH_BACK_BYPASS must be used instead.

Sample Code

#include <chaos/preprocessor/algorithm/push_back.h>
#include <chaos/preprocessor/list/core.h>

CHAOS_PP_PUSH_BACK((CHAOS_PP_LIST) (a, (b, ...)), c)
// (CHAOS_PP_LIST) (a, (b, (c, ...)))

CHAOS_PP_PUSH_BACK_ID

The CHAOS_PP_PUSH_BACK_ID macro indirectly expands to CHAOS_PP_PUSH_BACK.

Usage 

CHAOS_PP_PUSH_BACK_ID()

Remarks

This macro effectively allows CHAOS_PP_PUSH_BACK to pass through itself without becoming disabled.

CHAOS_PP_PUSH_BACK_

The CHAOS_PP_PUSH_BACK_ macro expands to a λ expression representing CHAOS_PP_PUSH_BACK.

Usage - C99 Specific 

CHAOS_PP_PUSH_BACK_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_PUSH_BACK_ID)()

CHAOS_PP_PUSH_BACK_BYPASS

The CHAOS_PP_PUSH_BACK_BYPASS macro behaves identically to CHAOS_PP_PUSH_BACK except that it is parametized by the bypass recursion state.

Usage - C99 Specific 

CHAOS_PP_PUSH_BACK_BYPASS(s, g, ...)
s

The current bypass recursion state.

g

The generic data structure to which the element is to be appended.

[...]

The element to be appended to g.

Usage - C/C++ Specific 

CHAOS_PP_PUSH_BACK_BYPASS(s, g, x)
s

The current bypass recursion state.

g

The generic data structure to which the element is to be appended.

x

The element to be appended to g.

Remarks

If bypass semantics are already in effect, this macro must be used instead of CHAOS_PP_PUSH_BACK.

CHAOS_PP_PUSH_BACK_BYPASS_ID

The CHAOS_PP_PUSH_BACK_BYPASS_ID macro indirectly expands to CHAOS_PP_PUSH_BACK_BYPASS.

Usage 

CHAOS_PP_PUSH_BACK_BYPASS_ID()

Remarks

This macro effectively allows CHAOS_PP_PUSH_BACK_BYPASS to pass through itself without becoming disabled.

CHAOS_PP_PUSH_BACK_BYPASS_

The CHAOS_PP_PUSH_BACK_BYPASS_ macro expands to a λ expression representing CHAOS_PP_PUSH_BACK_BYPASS.

Usage - C99 Specific 

CHAOS_PP_PUSH_BACK_BYPASS_

Remarks
