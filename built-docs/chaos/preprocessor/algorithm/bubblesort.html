The algorithm/bubblesort.h header defines an algorithm that sorts a generic data structure.

Usage

#include <chaos/preprocessor/algorithm/bubblesort.h>

Contents

  • CHAOS_PP_BUBBLESORT

CHAOS_PP_BUBBLESORT

The CHAOS_PP_BUBBLESORT macro sorts its second argument according to the predicate specified by its first argument.

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_BUBBLESORT(pred, g))
pred

The predicate that determines the ordering between one element and another. This predicate must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

pred(s, x†, y†)

† The pseudo-parameters x and y must each be the number of parameters that corresponds to the number of sub-elements in each element of g.

g

The generic data structure to be sorted.

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_BUBBLESORT(pred, g, ...))
pred

The predicate that determines the ordering between one element and another. This predicate must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

pred(s, x†, y†, data‡)

† The pseudo-parameters x and y must each be the number of parameters that corresponds to the number of sub-elements in each element of g.

‡ The data pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the auxiliary data, __VA_ARGS__.

g

The generic data structure to be sorted.

[...]

Auxiliary data to be passed through the algorithm to the predicate.

Usage - C/C++ Specific 

CHAOS_PP_EXPR(CHAOS_PP_BUBBLESORT(pred, g, data))
pred

The predicate that determines the ordering between one element and another. This predicate must be a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

pred(s, x, y, data)

† The pseudo-parameters x and y must each be the number of parameters that corresponds to the number of sub-elements in each element of g.

‡ The data pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the auxiliary data, __VA_ARGS__.

g

The generic data structure to be sorted.

Auxiliary data to be passed through the algorithm to the predicate.

Remarks

The CHAOS_PP_BUBBLESORT macro itself expands to a deferred expression in terms of CHAOS_PP_NEXT(s). This deferred expression, when invoked by CHAOS_PP_EXPR, expands to a generic data structure of the same type as g that contains the elements of g sorted according to pred.

The predicate is invoked with the current recursion state (s), two elements of g (x and y), and, if it exists, the auxiliary data (data or __VA_ARGS__). This invocation must expand to 0, 1, or a deferred expression in terms of CHAOS_PP_NEXT(s) that expands to either 0 or 1. Expansion to a deferred expression allows the predicate to recurse on itself if necessary.

If the predicate results in 1, x is sorted to a position before y. Otherwise, y is sorted to a position before x. If two are equivalent according to pred, their sorted position relative to each other is unspecified. That is, the algorithm does not guarantee stability.

In C99, it is possible to omit the auxiliary data argument altogether. This is different than passing "nothing" as the auxiliary data. 

#define D (CHAOS_PP_TUPLE) (a, b, c)

CHAOS_PP_EXPR(
    CHAOS_PP_BUBBLESORT(PRED, D,) // #1
    CHAOS_PP_BUBBLESORT(PRED, D)  // #2
)

The first line passes "nothing" as the auxiliary data, which causes PRED to be invoked as (e.g.)... 

PRED(s, a, b,)

The second line omits the auxiliary data, which causes PRED to be invoked as (e.g.)... 

PRED(s, a, b)

The fundamental difference is that, in the first line, the auxiliary data is empty (which might have a valid meaning as data), but, in the second line, there is no auxiliary data. Hence, the algorithm is overloaded on a true-optional parameter.

This macro deduces the current recursion state and invokes CHAOS_PP_BUBBLESORT_S. If the current recursion state is already known, direct use of CHAOS_PP_BUBBLESORT_S is more efficient.

This macro may not be used if bypass semantics are in effect.

Sample Code

#include <chaos/preprocessor/algorithm/bubblesort.h>
#include <chaos/preprocessor/comparison/less.h>
#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/lambda/ops.h>
#include <chaos/preprocessor/recursion/expr.h>
#include <chaos/preprocessor/seq/core.h>

#define PRED(s, x, y, _) CHAOS_PP_LESS(x, y)

CHAOS_PP_EXPR(CHAOS_PP_BUBBLESORT(
    PRED, (CHAOS_PP_SEQ) (3)(5)(1)(4)(2), ~
))
// (CHAOS_PP_SEQ) (1)(2)(3)(4)(5)

#if CHAOS_PP_VARIADICS

CHAOS_PP_EXPR(CHAOS_PP_BUBBLESORT(
    CHAOS_PP_LESS_(CHAOS_PP_ARG(1), CHAOS_PP_ARG(2)),
    (CHAOS_PP_SEQ) (3)(5)(1)(4)(2)
))
// (CHAOS_PP_SEQ) (1)(2)(3)(4)(5)

#endif

CHAOS_PP_BUBBLESORT_ID

The CHAOS_PP_BUBBLESORT_ID macro indirectly expands to CHAOS_PP_BUBBLESORT.

Usage 

CHAOS_PP_BUBBLESORT_ID()

Remarks

This macro effectively allows CHAOS_PP_BUBBLESORT to pass through itself without becoming disabled.

CHAOS_PP_BUBBLESORT_

The CHAOS_PP_BUBBLESORT_ macro expands to a λ expression representing CHAOS_PP_BUBBLESORT.

Usage - C99 Specific 

CHAOS_PP_BUBBLESORT_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_BUBBLESORT_ID)()
