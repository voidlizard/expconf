The algorithm/x/filter.h header defines an exponential algorithm that filters elements from a generic data structure according to a user-specified criteria which can be parametrically resumed.

Usage

#include <chaos/preprocessor/algorithm/x/filter.h>

Contents

  • CHAOS_PP_FILTER_X

CHAOS_PP_FILTER_X

The CHAOS_PP_FILTER_X macro filters elements from its third argument according to the criteria specified by its second argument.

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_FILTER_X(size, pred, g))
size

The size of the buffer of valid recursion steps maintained by the algorithm.

pred

The predicate that determines whether an element is to remain in the resulting generic data structure. This predicate must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

pred(s, elem†)

† If the type of g is non-variadic, elem must be either ... or the number of parameters that corresponds to the number of sub-elements in each element of g. Otherwise, elem must be ... or a single parameter.

g

The generic data structure to be filtered.

Usage - C99 Specific

Remarks

This macro behaves identically to CHAOS_PP_FILTER except that the invocations of the predicate are not trampolined, the algorithm makes exponential use of the recursion backend, and the algorithm itself yields a trampolined deferred expression whenever the number of recursion steps is exhausted. This allows for parametric resumption.

This algorithm maintains a buffer of size recursion steps available at all times. This guarantees that the predicate has at least size recursion steps available whenever it is invoked.

The exponential structure of the X-suffixed algorithms yields a total of α recursion steps, where p is the number of parametric resumptions plus the initial entry, sω is CHAOS_PP_LIMIT_EXPR, s is the current recursion state at the beginning of the algorithm (i.e. CHAOS_PP_STATE()), sδ is the buffer size, and z is the exponential base. Currently, z is fixed at two in the CHAOS_PP_FILTER_X algorithm (though it could easily be increased), which reduces the equation to β. Ignoring the complexity of pred (which is invoked once for each element in g), the efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper bound of O(2n+1), where n is the number of elements in g, occurs when all of the elements of g are preserved (i.e. when pred always results in 1). The asymptotic lower bound of Ω(n+1) occurs when all of the elements in g are removed (i.e. when pred always results in 0). These bounds preserve all low-order terms, which implies that the algorithm can process at least γ elements even if the worst case scenario is achieved and δ elements only if the best case scenario is achieved.

This macro deduces the current recursion state and invokes CHAOS_PP_FILTER_X_S. If the current recursion state is already known, direct use of CHAOS_PP_FILTER_X_S is more efficient.

This macro may not be used if bypass semantics are in effect.

CHAOS_PP_FILTER_X_ID

The CHAOS_PP_FILTER_X_ID macro indirectly expands to CHAOS_PP_FILTER_X.

Usage 

CHAOS_PP_FILTER_X_ID()

Remarks

This macro effectively allows CHAOS_PP_FILTER_X to pass through itself without becoming disabled.

CHAOS_PP_FILTER_X_

Usage - C99 Specific 

CHAOS_PP_FILTER_X_

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_FILTER_X_ID)()
