The algorithm/append.h header defines an algorithm that appends one generic data structure to another.

Usage

#include <chaos/preprocessor/algorithm/append.h>

Contents

  • CHAOS_PP_APPEND

CHAOS_PP_APPEND

The CHAOS_PP_APPEND macro appends its second argument to its first argument.

Usage 

CHAOS_PP_APPEND(g1, g2)
g1

The generic data structure to which g2 is to be appended.

g2

The generic data structure to be appended to g1.

Remarks

This macro appends the elements contained in g2 to the tail of g1, resulting in a generic data structure of the same type as g1.

The efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper bound of O(n+m+2), where n is the number of elements in g1 and m is the number of elements in g2, occurs when the type of g2 differs from the type of g1. The asymptotic lower bound of Ω(n+1) occurs when g1 and g2 share the same type.

The efficiency of the algorithm's use of the recursion backend can also be described in terms of the total depth of recursion steps required to complete the algorithm. The asymptotic upper bound of O(max(n,m)+1) occurs when the type of g2 differs from the type of g1. The asymptotic lower bound of Ω(n+1) occurs when g1 and g2 share the same type.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_APPEND_BYPASS must be used instead.

See Also

  • CHAOS_PP_PREPEND
  • CHAOS_PP_PUSH_BACK

Sample Code

#include <chaos/preprocessor/algorithm/append.h>
#include <chaos/preprocessor/list/core.h>
#include <chaos/preprocessor/seq/core.h>

CHAOS_PP_APPEND(
    (CHAOS_PP_SEQ) (a)(b)(c),
    (CHAOS_PP_LIST) (x, (y, (z, ...)))
)
// (CHAOS_PP_SEQ) (a)(b)(c)(x)(y)(z)

CHAOS_PP_APPEND_ID

The CHAOS_PP_APPEND_ID macro indirectly expands to CHAOS_PP_APPEND.

Usage 

CHAOS_PP_APPEND_ID()

Remarks

This macro effectively allows CHAOS_PP_APPEND to pass through itself without becoming disabled.

CHAOS_PP_APPEND_

The CHAOS_PP_APPEND_ macro expands to a λ expression representing CHAOS_PP_APPEND.

Usage - C99 Specific 

CHAOS_PP_APPEND_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_APPEND_ID)()

CHAOS_PP_APPEND_BYPASS

The CHAOS_PP_APPEND_BYPASS macro behaves identically to CHAOS_PP_APPEND except that it is parametized by the bypass recursion state.

Usage 

CHAOS_PP_APPEND_BYPASS(s, g1, g2)
s

The current bypass recursion state.

g1

The generic data structure to which g2 is to be appended.

g2

The generic data structure to be appended to g1.

Remarks

If bypass semantics are already in effect, this macro must be used instead of CHAOS_PP_APPEND.

CHAOS_PP_APPEND_BYPASS_ID

The CHAOS_PP_APPEND_BYPASS_ID macro indirectly expands to CHAOS_PP_APPEND_BYPASS.

Usage 

CHAOS_PP_APPEND_BYPASS_ID()

Remarks

This macro effectively allows CHAOS_PP_APPEND_BYPASS to pass through itself without becoming disabled.

CHAOS_PP_APPEND_BYPASS_

The CHAOS_PP_APPEND_BYPASS_ macro expands to a λ expression representing CHAOS_PP_APPEND_BYPASS.

Usage - C99 Specific 

CHAOS_PP_APPEND_BYPASS_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_APPEND_BYPASS_ID)()
