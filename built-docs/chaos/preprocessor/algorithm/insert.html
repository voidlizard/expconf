The algorithm/insert.h header defines an algorithm that inserts an element into a generic data structure.

Usage

#include <chaos/preprocessor/algorithm/insert.h>

Contents

  • CHAOS_PP_INSERT

CHAOS_PP_INSERT

The CHAOS_PP_INSERT macro inserts its third argument into its second argument before the index specified by its first argument.

Usage - C99 Specific 

CHAOS_PP_INSERT(i, g, ...)
i

The position at which the element is to be inserted. Valid values range from 0 to CHAOS_PP_SIZE(g).

g

The generic data structure into which the element is to be inserted.

[...]

The element to be inserted.

Usage - C/C++ Specific 

CHAOS_PP_INSERT(i, g, x)
i

The position at which the element is to be inserted. Valid values range from 0 to CHAOS_PP_SIZE(g)

g

The generic data structure into which the element is to be inserted.

x

The element to be inserted.

Remarks

This macro expands to a generic data structure of the same type as g with the elements of g with x (or __VA_ARGS__) inserted at position i.

The efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Θ(n+1), where n is i.

Note that, in C99, this algorithm makes no attempt to verify that __VA_ARGS__ can be inserted into g. Thus it is possible to produce invalid data structures. For example, 

CHAOS_PP_INSERT(0, (CHAOS_PP_STRING) a b c, +) // invalid

The above results in 

(CHAOS_PP_STRING) + a b c

which is an invalid string.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_INSERT_BYPASS must be used instead.

See Also

  • CHAOS_PP_SEQ_INSERT
  • CHAOS_PP_TUPLE_INSERT

Sample Code

#include <chaos/preprocessor/algorithm/insert.h>
#include <chaos/preprocessor/string/core.h>

CHAOS_PP_INSERT(2, (CHAOS_PP_STRING) a b c, _)
// (CHAOS_PP_STRING) a b _ c

CHAOS_PP_INSERT_ID

The CHAOS_PP_INSERT_ID macro indirectly expands to CHAOS_PP_INSERT.

Usage 

CHAOS_PP_INSERT_ID()

Remarks

This macro effectively allows CHAOS_PP_INSERT to pass through itself without becoming disabled.

CHAOS_PP_INSERT_

The CHAOS_PP_INSERT_ macro expands to a λ expression representing CHAOS_PP_INSERT.

Usage - C99 Specific 

CHAOS_PP_INSERT_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_INSERT_ID)()

CHAOS_PP_INSERT_BYPASS

The CHAOS_PP_INSERT_BYPASS macro behaves identically to CHAOS_PP_INSERT except that it is parametized by the bypass recursion state.

Usage - C99 Specific 

CHAOS_PP_INSERT_BYPASS(s, i, g, ...)
s

The current bypass recursion state.

i

The position at which the element is to be inserted. Valid values range from 0 to CHAOS_PP_SIZE(g).

g

The generic data structure into which the element is to be inserted.

[...]

The element to be inserted.

Usage - C/C++ Specific 

CHAOS_PP_INSERT_BYPASS(s, i, g, x)
s

The current bypass recursion state.

i

The position at which the element is to be inserted. Valid values range from 0 to CHAOS_PP_SIZE(g)

g

The generic data structure into which the element is to be inserted.

x

The element to be inserted.

Remarks

If bypass semantics are already in effect, this macro must be used instead of CHAOS_PP_INSERT.

CHAOS_PP_INSERT_BYPASS_ID

The CHAOS_PP_INSERT_BYPASS_ID macro indirectly expands to CHAOS_PP_INSERT_BYPASS.

CHAOS_PP_INSERT_BYPASS_ID()
