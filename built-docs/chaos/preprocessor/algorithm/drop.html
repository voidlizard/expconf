The algorithm/drop.h header defines an algorithm that yields the remainder of a generic data structure after removing a user-specified number of elements.

Usage

#include <chaos/preprocessor/algorithm/drop.h>

Contents

  • CHAOS_PP_DROP

CHAOS_PP_DROP

The CHAOS_PP_DROP macro removes the number of elements specified by its first argument from the head of its second argument.

Usage 

CHAOS_PP_DROP(n, g)
n

The number of elements to be removed. Valid values range from 0 to CHAOS_PP_SIZE(g).

g

The generic data structure from which elements are to be removed.

Remarks

This macro expands to a generic data structure of the same type as g containing all but the first n elements of g.

The efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Θ(n+1), where n is n.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_DROP_BYPASS must be used instead.

Sample Code

#include <chaos/preprocessor/algorithm/drop.h>
#include <chaos/preprocessor/list/core.h>

CHAOS_PP_DROP(
    3,
    (CHAOS_PP_LIST) (a, (b, (c, (d, (e, ...)))))
)
// (CHAOS_PP_LIST) (d, (e, ...))

CHAOS_PP_DROP_ID

The CHAOS_PP_DROP_ID macro indirectly expands to CHAOS_PP_DROP.

Usage 

CHAOS_PP_DROP_ID()

Remarks

This macro effectively allows CHAOS_PP_DROP to pass through itself without becoming disabled.

CHAOS_PP_DROP_

The CHAOS_PP_DROP_ macro expands to a λ expression representing CHAOS_PP_DROP.

Usage - C99 Specific 

CHAOS_PP_DROP_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_DROP_ID)()

CHAOS_PP_DROP_BYPASS

The CHAOS_PP_DROP_BYPASS macro behaves identically to CHAOS_PP_DROP except that it is parametized by the bypass recursion state.

Usage 

CHAOS_PP_DROP_BYPASS(s, n, g)
s

The current bypass recursion state.

n

The number of elements to be removed. Valid values range from 0 to CHAOS_PP_SIZE(g).

g

The generic data structure from which elements are to be removed.

Remarks

If bypass semantics are already in effect, this macro must be used instead of CHAOS_PP_DROP.

CHAOS_PP_DROP_BYPASS_ID

The CHAOS_PP_DROP_BYPASS_ID macro indirectly expands to CHAOS_PP_DROP_BYPASS.

Usage 

CHAOS_PP_DROP_BYPASS_ID()

Remarks

This macro effectively allows CHAOS_PP_DROP_BYPASS to pass through itself without becoming disabled.

CHAOS_PP_DROP_BYPASS_

The CHAOS_PP_DROP_BYPASS_ macro expands to a λ expression representing CHAOS_PP_DROP_BYPASS.

Usage - C99 Specific 

CHAOS_PP_DROP_BYPASS_

Remarks

This macro is a shorthand form of the following λ binding:
