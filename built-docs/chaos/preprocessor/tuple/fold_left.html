The tuple/fold_left.h header defines an algorithm that accumulates the elements of a tuple from the first element to the last element.

Usage

#include <chaos/preprocessor/tuple/fold_left.h>

Contents

  • CHAOS_PP_TUPLE_FOLD_LEFT

CHAOS_PP_TUPLE_FOLD_LEFT

The CHAOS_PP_TUPLE_FOLD_LEFT macro accumulates the elements of its second argument with the operation specified by its first argument.

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_TUPLE_FOLD_LEFT(op, tuple, ...))
op

The operation that is to be used to accumulate the elements of tuple. This operation must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

op(s, elem, state†)

† The state pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the accumulation state, __VA_ARGS__.

[tuple]

The tuple whose elements are to be accumulated.

[...]

The initial state of the accumulation.

Remarks

The CHAOS_PP_TUPLE_FOLD_LEFT macro itself expands to a deferred expression in terms of CHAOS_PP_NEXT(s). This deferred expression, when invoked by CHAOS_PP_EXPR, expands to the result of accumulating the elements of tuple, from left to right, with the user-specified operation. This is illustrated by the following pseudo-code: 

op(s, tuple[length(tuple) - 1], ... op(s, tuple[1], op(s, tuple[0], state)) ... )

An invocation of the operation with the current recursion state (s), a particular element (elem), and the current state (__VA_ARGS__) expands to a new state used in the accumulation of subsequent elements (or as the final result of the algorithm if no further elements exist). The operation may also expand to a deferred expression in terms of CHAOS_PP_NEXT(s) that expands to a new state. Expansion to a deferred expression allows the operation to recurse on itself if necessary.

Invocations of the operation are trampolined such that each invocation is invoked with CHAOS_PP_NEXT(CHAOS_PP_NEXT(s)), where s is the original recursion state at the beginning of the algorithm (i.e. CHAOS_PP_STATE()). This maximizes the utility of the recursion backend.

Ignoring the complexity of op (which is invoked once for each element in tuple), the efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Θα, where n is the number of elements in tuple.

Ignoring the complexity of op once again, the efficiency of the algorithm's use of the recursion backend can be described in terms of the total depth of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Θβ, where n is the number of elements in tuple.

This macro operates on the nil tuple representation used by the core tuple operations defined in <chaos/preprocessor/tuple/core.h>. Specifically, () is a one-element tuple, not a zero-element tuple.

This macro deduces the current recursion state and invokes CHAOS_PP_TUPLE_FOLD_LEFT_S. If the current recursion state is already known, direct use of CHAOS_PP_TUPLE_FOLD_LEFT_S is more efficient.

This macro may not be used if bypass semantics are in effect.

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/recursion/expr.h>
#include <chaos/preprocessor/tuple/fold_left.h>

#if CHAOS_PP_VARIADICS

#define OP(s, elem, r) r ## elem

CHAOS_PP_EXPR(CHAOS_PP_TUPLE_FOLD_LEFT(
    OP,
    (a, b, c, d, e, f, g, h, i, j, k, l, m, n, o, p, q, r, s, t, u, v, w, x, y, z),
))
// abcdefghijklmnopqrstuvwxyz

#endif

CHAOS_PP_TUPLE_FOLD_LEFT_ID

The CHAOS_PP_TUPLE_FOLD_LEFT_ID macro indirectly expands to CHAOS_PP_TUPLE_FOLD_LEFT.

Usage - C99 Specific 

CHAOS_PP_TUPLE_FOLD_LEFT_ID()

Remarks

This macro effectively allows CHAOS_PP_TUPLE_FOLD_LEFT to pass through itself without becoming disabled.

CHAOS_PP_TUPLE_FOLD_LEFT_

The CHAOS_PP_TUPLE_FOLD_LEFT_ macro expands to a λ expression representing CHAOS_PP_TUPLE_FOLD_LEFT.

Usage - C99 Specific 

CHAOS_PP_TUPLE_FOLD_LEFT_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_TUPLE_FOLD_LEFT_ID)()

CHAOS_PP_TUPLE_FOLD_LEFT_S

The CHAOS_PP_TUPLE_FOLD_LEFT_S macro behaves identically to CHAOS_PP_TUPLE_FOLD_LEFT except that it is parametized by the recursion state.

Usage - C99 Specific 

CHAOS_PP_EXPR_S(s)(CHAOS_PP_TUPLE_FOLD_LEFT_S(s, op, tuple, ...))
s

The current recursion state.

op

The operation that is to be used to accumulate the elements of tuple. This operation must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

op(s, elem, state†)

† The state pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the accumulation state, __VA_ARGS__.

[tuple]

The tuple whose elements are to be accumulated.

[...]

The initial state of the accumulation.

Remarks

This macro may not be used if bypass semantics are in effect.
