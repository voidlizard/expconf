The tuple/take.h header defines an algorithm that extracts a user-specified number of elements from the beginning of a tuple.

Usage

#include <chaos/preprocessor/tuple/take.h>

Contents

  • CHAOS_PP_TUPLE_TAKE

CHAOS_PP_TUPLE_TAKE

The CHAOS_PP_TUPLE_TAKE macro extracts the number of elements specified by its first argument from its second argument.

Usage - C99 Specific 

CHAOS_PP_TUPLE_TAKE(n, tuple)
n

The number of elements to be extracted. Valid values range from 0 to CHAOS_PP_TUPLE_SIZE(tuple).

[tuple]

The tuple from which elements are to be extracted.

Remarks

This macro expands to a tuple containing the first n elements of tuple.

The efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Θ, where n is n.

This macro operates on the nil tuple representation used by the core tuple operations defined in <chaos/preprocessor/tuple/core.h>. Specifically, () is a one-element tuple, not a zero-element tuple.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_TUPLE_TAKE_BYPASS must be used instead.

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/tuple/take.h>

#if CHAOS_PP_VARIADICS

CHAOS_PP_TUPLE_TAKE(
    12,
    (a, b, c, d, e, f, g, h, i, j, k, l, m, n, o, p, q, r, s, t, u, v, w, x, y, z)
)
// (a, b, c, d, e, f, g, h, i, j, k, l)

#endif

CHAOS_PP_TUPLE_TAKE_ID

The CHAOS_PP_TUPLE_TAKE_ID macro indirectly expands to CHAOS_PP_TUPLE_TAKE.

Usage - C99 Specific 

CHAOS_PP_TUPLE_TAKE_ID()

Remarks

This macro effectively allows CHAOS_PP_TUPLE_TAKE to pass through itself without becoming disabled.

CHAOS_PP_TUPLE_TAKE_

The CHAOS_PP_TUPLE_TAKE_ macro expands to a λ expression representing CHAOS_PP_TUPLE_TAKE.

Usage - C99 Specific 

CHAOS_PP_TUPLE_TAKE_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_TUPLE_TAKE_ID)()

CHAOS_PP_TUPLE_TAKE_BYPASS

The CHAOS_PP_TUPLE_TAKE_BYPASS macro behaves identically to CHAOS_PP_TUPLE_TAKE except that it is parametized by the bypass recursion state.

Usage - C99 Specific 

CHAOS_PP_TUPLE_TAKE_BYPASS(s, n, tuple)
s

The current bypass recursion state.

n

The number of elements to be extracted. Valid values range from 0 to CHAOS_PP_TUPLE_SIZE(tuple).

[tuple]

The tuple from which elements are to be extracted.

Remarks

If bypass semantics are already in effect, this macro must be used instead of CHAOS_PP_TUPLE_TAKE.
