The tuple/replace.h header defines an algorithm that replaces an element in a tuple.

Usage

#include <chaos/preprocessor/tuple/replace.h>

Contents

  • CHAOS_PP_TUPLE_REPLACE

CHAOS_PP_TUPLE_REPLACE

The CHAOS_PP_TUPLE_REPLACE macro replaces the element at the index specified by its first argument in its second argument with the element specified by its third argument.

Usage - C99 Specific 

CHAOS_PP_TUPLE_REPLACE(i, tuple, x)
i

The zero-based index of the element to be replaced. Valid values range from 0 to CHAOS_PP_TUPLE_SIZE(tuple) - 1.

tuple

The tuple in which an element is to be replaced.

[x]

The element used to replace the element at index i in tuple.

Remarks

This macro expands to a tuple containing the elements of tuple except with the i-th element replaced by x.

The efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Θ, where n is i.

This macro operates on the nil tuple representation used by the core tuple operations defined in <chaos/preprocessor/tuple/core.h>. Specifically, () is a one-element tuple, not a zero-element tuple.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_TUPLE_REPLACE_BYPASS must be used instead.

See Also

  • CHAOS_PP_REPLACE
  • CHAOS_PP_SEQ_REPLACE

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/tuple/replace.h>

#if CHAOS_PP_VARIADICS

CHAOS_PP_TUPLE_REPLACE(5, (a, b, c, d, e, f, g), +) // (a, b, c, d, e, +, g)

#endif

CHAOS_PP_TUPLE_REPLACE_ID

The CHAOS_PP_TUPLE_REPLACE_ID macro indirectly expands to CHAOS_PP_TUPLE_REPLACE.

Usage - C99 Specific 

CHAOS_PP_TUPLE_REPLACE_ID()

Remarks

This macro effectively allows CHAOS_PP_TUPLE_REPLACE to pass through itself without becoming disabled.

CHAOS_PP_TUPLE_REPLACE_

The CHAOS_PP_TUPLE_REPLACE_ macro expands to a λ expression representing CHAOS_PP_TUPLE_REPLACE.

Usage - C99 Specific 

CHAOS_PP_TUPLE_REPLACE_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_TUPLE_REPLACE_ID)()

CHAOS_PP_TUPLE_REPLACE_BYPASS

The CHAOS_PP_TUPLE_REPLACE_BYPASS macro behaves identically to CHAOS_PP_TUPLE_REPLACE except that it is parametized by the bypass recursion state.

Usage - C99 Specific 

CHAOS_PP_TUPLE_REPLACE_BYPASS(s, i, tuple, x)
s

The current bypass recursion state.

i

The zero-based index of the element to be replaced. Valid values range from 0 to CHAOS_PP_TUPLE_SIZE(tuple) - 1.

tuple

The tuple in which an element is to be replaced.

[x]

The element used to replace the element at index i in tuple.

Remarks

If bypass semantics are already in effect, this macro must be used instead of CHAOS_PP_TUPLE_REPLACE.

CHAOS_PP_TUPLE_REPLACE_BYPASS_ID

The CHAOS_PP_TUPLE_REPLACE_BYPASS_ID macro indirectly expands to CHAOS_PP_TUPLE_REPLACE_BYPASS.

Usage - C99 Specific 

CHAOS_PP_TUPLE_REPLACE_BYPASS_ID()

Remarks

This macro effectively allows CHAOS_PP_TUPLE_REPLACE_BYPASS to pass through itself without becoming disabled.

CHAOS_PP_TUPLE_REPLACE_BYPASS_

The CHAOS_PP_TUPLE_REPLACE_BYPASS_ macro expands to a λ expression representing CHAOS_PP_TUPLE_REPLACE_BYPASS.

Usage - C99 Specific 

CHAOS_PP_TUPLE_REPLACE_BYPASS_

Remarks
