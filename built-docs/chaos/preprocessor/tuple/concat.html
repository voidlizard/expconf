The tuple/concat.h header defines an algorithm that concatenates the elements of a tuple.

Usage

#include <chaos/preprocessor/tuple/concat.h>

Contents

  • CHAOS_PP_TUPLE_CONCAT

CHAOS_PP_TUPLE_CONCAT

The CHAOS_PP_TUPLE_CONCAT macro concatenates the elements of its argument.

Usage - C99 Specific 

CHAOS_PP_TUPLE_CONCAT(tuple)
[tuple]

The tuple whose elements are to be concatenated.

Remarks

This macro performs an ordered concatenation of the elements of tuple beginning with the first element. This is illustrated by the following pseudo-code: 

( ... ((tuple[0] ## tuple[1]) ## tuple[2]) ... ## tuple[length(tuple) - 1])

The efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Θ, where n is the number of elements in tuple.

This macro operates on the nil tuple representation used by the core tuple operations defined in <chaos/preprocessor/tuple/core.h>. Specifically, () is a one-element tuple, not a zero-element tuple.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_TUPLE_CONCAT_BYPASS must be used instead.

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/tuple/concat.h>

#if CHAOS_PP_VARIADICS

CHAOS_PP_TUPLE_CONCAT(
    (a, b, c, d, e, f, g, h, i, j, k, l, m, n, o, p, q, r, s, t, u, v, w, x, y, z)
)
// abcdefghijklmnopqrstuvwxyz

#endif

CHAOS_PP_TUPLE_CONCAT_ID

The CHAOS_PP_TUPLE_CONCAT_ID macro indirectly expands to CHAOS_PP_TUPLE_CONCAT.

Usage - C99 Specific 

CHAOS_PP_TUPLE_CONCAT_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_TUPLE_CONCAT_

The CHAOS_PP_TUPLE_CONCAT_ macro expands to a λ expression representing CHAOS_PP_TUPLE_CONCAT.

Usage - C99 Specific 

CHAOS_PP_TUPLE_CONCAT_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_TUPLE_CONCAT)

CHAOS_PP_TUPLE_CONCAT_BYPASS

The CHAOS_PP_TUPLE_CONCAT_BYPASS macro behaves identically to CHAOS_PP_TUPLE_CONCAT except that it is parametized by the bypass recursion state.

Usage - C99 Specific 

CHAOS_PP_TUPLE_CONCAT_BYPASS(s, tuple)
s

The current bypass recursion state.

[tuple]

The tuple whose elements are to be concatenated.

Remarks

If bypass semantics are already in effect, this macro must be used instead of CHAOS_PP_TUPLE_CONCAT.
