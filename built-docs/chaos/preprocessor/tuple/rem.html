CHAOS_PP_TUPLE_REM

The CHAOS_PP_TUPLE_REM macro removes the parentheses from a tuple of the specified size.

Usage - C99 Specific 

CHAOS_PP_TUPLE_REM(size) tuple
[size]

This argument exists only for compatibility. It is otherwise ignored.

tuple

The tuple from which parentheses are to be removed.

Usage - C/C++ Specific 

CHAOS_PP_TUPLE_REM(size) tuple
size

The size of the tuple from which parentheses are to be removed. Valid values range from 0 to CHAOS_PP_LIMIT_TUPLE.

tuple

The tuple from which parentheses are to be removed.

Remarks

The CHAOS_PP_TUPLE_REM macro itself expands to a macro capable of being invoked against a tuple of arity size. This macro, when invoked against tuple, expands to the contents of the tuple. 

CHAOS_PP_TUPLE_REM(3)(a, b, c) // a, b, c

Note that in C99, the CHAOS_PP_TUPLE_REM macro expands to a macro capable of removing the parentheses from a tuple of any arity. The size parameter is ignored.

Sample Code

#include <chaos/preprocessor/facilities/expand.h>
#include <chaos/preprocessor/recursion/basic.h>
#include <chaos/preprocessor/tuple/rem.h>

#define A(x, y) \
    CHAOS_PP_EXPAND( \
        CHAOS_PP_DEFER(B)( \
            CHAOS_PP_TUPLE_REM(2) x, \
            CHAOS_PP_TUPLE_REM(2) y \
        ) \
    ) \
    /**/
#define B(a, b, c, d) a + b + c + d

A((1, 2), (3, 4)) // 1 + 2 + 3 + 4

CHAOS_PP_TUPLE_REM_CTOR

The CHAOS_PP_TUPLE_REM_CTOR macro removes the parentheses from a tuple of the specified size.

Usage - C99 Specific 

CHAOS_PP_TUPLE_REM_CTOR(size, tuple)
[size]

This argument exists only for compatibility. It is otherwise ignored.

tuple

The tuple from which parentheses are to be removed.

Usage - C/C++ Specific 

CHAOS_PP_TUPLE_REM_CTOR(size, tuple)
size

The size of the tuple from which parentheses are to be removed. Valid values range from 0 to CHAOS_PP_LIMIT_TUPLE.

tuple

The tuple from which parentheses are to be removed.

Remarks

This macro expands to the contents of tuple. 

CHAOS_PP_TUPLE_REM_CTOR(3, (a, b, c)) // a, b, c

Note that in C99, the CHAOS_PP_TUPLE_REM_CTOR macro is capable of removing the parenthese from a tuple of any size. Thus, the size parameter is ignored.

The primary purpose of this macro is remove the parentheses from a tuple that is the result of a macro invocation. In such a circumstance, use of CHAOS_PP_TUPLE_REM will not expand the against the tuple without the manual introduction of another scan: 

#define A (a, b, c)

CHAOS_PP_EXPAND(CHAOS_PP_TUPLE_REM(3) A) // a, b, c

This macro aids in this situation by allowing the tuple argument to expand prior to parentheses removal: 

CHAOS_PP_TUPLE_REM_CTOR(3, A) // a, b, c

CHAOS_PP_REM

The CHAOS_PP_REM macro removes the parentheses from a unary tuple in C++ (and C prior to C99) or from a tuple of any arity in C99.

Usage - C99 Specific 

CHAOS_PP_REM tuple
tuple

The tuple from which parentheses are to be removed.

Usage - C/C++ Specific 

CHAOS_PP_REM tuple
tuple

The unary tuple from which parentheses are to be removed.

Remarks

This macro is used primarily in algorithms as a generalization between the allowance of variadic data or state in C99 and the rigidity of unary data or state in C++ (and C prior to C99) such that the implementation (other than the interface) can be shared in both C99 and C/C++ configurations.

Sample Code

#include <chaos/preprocessor/arithmetic/dec.h>
#include <chaos/preprocessor/cat.h>
#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/control/when.h>
#include <chaos/preprocessor/facilities/binary.h>
#include <chaos/preprocessor/punctuation/comma_if.h>
#include <chaos/preprocessor/recursion/basic.h>
#include <chaos/preprocessor/recursion/expr.h>
#include <chaos/preprocessor/tuple/rem.h>

#if CHAOS_PP_VARIADICS

#define REPEAT(count, macro, ...) \
    REPEAT_S(CHAOS_PP_STATE(), count, macro, __VA_ARGS__) \
    /**/
#define REPEAT_S(s, count, macro, ...) \
    REPEAT_I( \
        CHAOS_PP_OBSTRUCT(), CHAOS_PP_NEXT(s), \
        count, macro, (__VA_ARGS__) \
    ) \
    /**/

#else

#define REPEAT(count, macro, data) \
    REPEAT_S(CHAOS_PP_STATE(), count, macro, data) \
    /**/
#define REPEAT_S(s, count, macro, data) \
    REPEAT_I( \
        CHAOS_PP_OBSTRUCT(), CHAOS_PP_NEXT(s), \
        count, macro, (data) \
    ) \
    /**/

#endif

#define REPEAT_INDIRECT() REPEAT_I
#define REPEAT_I(_, s, count, macro, pd) \
    CHAOS_PP_WHEN _(count)( \
        CHAOS_PP_EXPR_S(s) _(REPEAT_INDIRECT _()( \
            CHAOS_PP_OBSTRUCT _(), CHAOS_PP_NEXT(s), \
            CHAOS_PP_DEC(count), macro, pd \
        )) \
        macro _(s, CHAOS_PP_DEC(count), CHAOS_PP_REM pd) \
    ) \
    /**/

#if CHAOS_PP_VARIADICS

#define MACRO(s, n, x, y) CHAOS_PP_COMMA_IF(n) class x ## n = y

CHAOS_PP_EXPR(REPEAT(
    3, MACRO, T, U
))
// class T0 = U, class T1 = U, class T2 = U

#else

#define MACRO(s, n, xy) \
    CHAOS_PP_COMMA_IF(n) \
    class CHAOS_PP_CAT(CHAOS_PP_BINARY(0) xy, n) \
        = CHAOS_PP_BINARY(1) xy \
    /**/

CHAOS_PP_EXPR(REPEAT(
    3, MACRO, (T, U)
))
// class T0 = U, class T1 = U, class T2 = U

#endif
