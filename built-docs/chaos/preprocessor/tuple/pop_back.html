The tuple/pop_back.h header defines an algorithm that removes the last element from a tuple.

Usage

#include <chaos/preprocessor/tuple/pop_back.h>

Contents

  • CHAOS_PP_TUPLE_POP_BACK

CHAOS_PP_TUPLE_POP_BACK

The CHAOS_PP_TUPLE_POP_BACK macro removes the last element from its argument.

Usage - C99 Specific 

CHAOS_PP_TUPLE_POP_BACK(tuple)
tuple

The non-nil tuple whose last element is to be removed.

Remarks

This macro expands to a tuple that contains all of the elements of tuple except the last.

The efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Θ, where n is the number of elements in tuple.

This macro operates on the nil tuple representation used by the core tuple operations defined in <chaos/preprocessor/tuple/core.h>. Specifically, () is a one-element tuple, not a zero-element tuple.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_TUPLE_POP_BACK_BYPASS must be used instead.

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/tuple/pop_back.h>

#if CHAOS_PP_VARIADICS

CHAOS_PP_TUPLE_POP_BACK((a, b, c)) // (b, c)
CHAOS_PP_TUPLE_POP_BACK((a))       //

#endif

CHAOS_PP_TUPLE_POP_BACK_ID

The CHAOS_PP_TUPLE_POP_BACK_ID macro indirectly expands to CHAOS_PP_TUPLE_POP_BACK.

Usage - C99 Specific 

CHAOS_PP_TUPLE_POP_BACK_ID()

Remarks

This macro effectively allows CHAOS_PP_TUPLE_POP_BACK to pass through itself without becoming disabled.

CHAOS_PP_TUPLE_POP_BACK_

The CHAOS_PP_TUPLE_POP_BACK_ macro expands to a λ expression representing CHAOS_PP_TUPLE_POP_BACK.

Usage - C99 Specific 

CHAOS_PP_TUPLE_POP_BACK_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_TUPLE_POP_BACK_ID)()

CHAOS_PP_TUPLE_POP_BACK_BYPASS

The CHAOS_PP_TUPLE_POP_BACK_BYPASS macro behaves identically to CHAOS_PP_TUPLE_POP_BACK except that it is parametized by the bypass recursion state.

Usage - C99 Specific 

CHAOS_PP_TUPLE_POP_BACK_BYPASS(s, tuple)
s

The current bypass recursion state.

tuple

The non-nil tuple whose last element is to be removed.

Remarks

If bypass semantics are already in effect, this macro must be used instead of CHAOS_PP_TUPLE_POP_BACK.
