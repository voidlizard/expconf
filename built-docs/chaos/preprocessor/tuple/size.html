CHAOS_PP_TUPLE_QUICK_SIZE

The CHAOS_PP_TUPLE_QUICK_SIZE macro counts the elements in its argument.

Usage - C99 Specific 

CHAOS_PP_TUPLE_QUICK_SIZE(tuple)
[tuple]

The tuple whose elements are to be counted.

Remarks

This macro expands to the number of elements in tuple. If tuple contains more than CHAOS_PP_LIMIT_TUPLE elements, the result is saturated to CHAOS_PP_LIMIT_TUPLE.

Unlike CHAOS_PP_TUPLE_SIZE, CHAOS_PP_TUPLE_QUICK_SIZE is a constant-time operation at the price of a much lower limit on countable elements.

This macro operates on the nil tuple representation used by the core tuple operations defined in <chaos/preprocessor/tuple/core.h>. Specifically, () is a one-element tuple, not a zero-element tuple.

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/tuple/size.h>

#if CHAOS_PP_VARIADICS

CHAOS_PP_TUPLE_QUICK_SIZE((a, b, c))       // 3
CHAOS_PP_TUPLE_QUICK_SIZE((a, b, c, d, e)) // 5

#endif

CHAOS_PP_TUPLE_SIZE

The CHAOS_PP_TUPLE_SIZE macro counts the elements in its argument.

Usage - C99 Specific 

CHAOS_PP_TUPLE_SIZE(tuple)
[tuple]

The tuple whose elements are to be counted.

Remarks

This macro expands to the number of elements in tuple. If tuple contains more than CHAOS_PP_LIMIT_MAG elements, the result is saturated to CHAOS_PP_LIMIT_MAG.

The efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Θ, where n is the number of elements in tuple.

This macro operates on the nil tuple representation used by the core tuple operations defined in <chaos/preprocessor/tuple/core.h>. Specifically, () is a one-element tuple, not a zero-element tuple.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_TUPLE_SIZE_BYPASS must be used instead.

CHAOS_PP_TUPLE_SIZE_BYPASS

The CHAOS_PP_TUPLE_SIZE_BYPASS macro behaves identically to CHAOS_PP_TUPLE_SIZE except that it is parametized by the bypass recursion state.

Usage - C99 Specific 

CHAOS_PP_TUPLE_SIZE_BYPASS(s, tuple)
s

The current bypass recursion state.

[tuple]

The tuple whose elements are to be counted.

Remarks

If bypass semantics are already in effect, this macro must be used instead of CHAOS_PP_TUPLE_SIZE.

CHAOS_PP_VARIADIC_QUICK_SIZE

The CHAOS_PP_VARIADIC_QUICK_SIZE macro counts the its arguments.

Usage - C99 Specific 

CHAOS_PP_VARIADIC_QUICK_SIZE(...)
[...]

The arguments to be counted.

Remarks

This macro expands to the number of arguments it is passed. If __VA_ARGS__ contains more that CHAOS_PP_LIMIT_TUPLE arguments, the result is saturated to CHAOS_PP_LIMIT_TUPLE.
