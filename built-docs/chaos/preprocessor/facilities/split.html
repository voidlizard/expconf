The facilities/split.h header defines a macro that extracts an element from a binary intermediate.

Usage

#include <chaos/preprocessor/facilities/split.h>

Contents

  • CHAOS_PP_SPLIT

CHAOS_PP_SPLIT

The CHAOS_PP_SPLIT macro extracts an element from a binary intermediate.

Usage - C99 Specific 

CHAOS_PP_SPLIT(i, ...)
i

The index of the element to be extracted. Valid values are 0 and 1.

[...]

The intermediate from which an element is to be extracted. This intermediate must expand to at least two elements.

Usage - C/C++ Specific 

CHAOS_PP_SPLIT(i, im)
i

The index of the element to be extracted. Valid values are 0 and 1.

im

The intermediate from which an element is to be extracted. This intermediate must expand to two elements.

Remarks

If i is 0, this macro expands to the first element of im or __VA_ARGS__. Otherwise, it expands to the second element of im or the elements after the first in __VA_ARGS__.

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/facilities/split.h>

#define A() p, q

CHAOS_PP_SPLIT(0, A()) // p
CHAOS_PP_SPLIT(1, A()) // q

#if CHAOS_PP_VARIADICS

#define B() x, y, z

CHAOS_PP_SPLIT(0, B()) // x
CHAOS_PP_SPLIT(1, B()) // y, z

#endif

CHAOS_PP_SPLIT_ID

The CHAOS_PP_SPLIT_ID macro indirectly expands to CHAOS_PP_SPLIT.

Usage 

CHAOS_PP_SPLIT_ID()

Remarks

This macro effectively allows CHAOS_PP_SPLIT to pass through itself without becoming disabled.

CHAOS_PP_SPLIT_

The CHAOS_PP_SPLIT_ macro expands to a λ expression representing CHAOS_PP_SPLIT.

Usage - C99 Specific 

CHAOS_PP_SPLIT_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_SPLIT_ID)()
