The facilities/overload.h header defines macros used to facilitate overloading of macros.

Usage

#include <chaos/preprocessor/facilities/overload.h>

Contents

  • CHAOS_PP_OVERLOAD
  • CHAOS_PP_QUICK_OVERLOAD

CHAOS_PP_QUICK_OVERLOAD

The CHAOS_PP_QUICK_OVERLOAD macro facilitates overloading of macros on the number of arguments.

Usage - C99 Specific 

CHAOS_PP_QUICK_OVERLOAD(pre, ...)(...)
pre

The prefix of a set of overloaded macros.

[...]

The arguments to be passed to an overloaded macro.

Remarks

This macro counts the number of arguments in __VA_ARGS__ and expands to an expression of the following form: 

pre ## count(__VA_ARGS__)

If the number of arguments is greater than CHAOS_PP_LIMIT_TUPLE, the result is saturated to CHAOS_PP_LIMIT_TUPLE.

The number of arguments is never zero--even in situations such as the following: 

CHAOS_PP_QUICK_OVERLOAD(XYZ_,)()

The above is interpreted as a single empty argument.

Unlike CHAOS_PP_OVERLOAD, CHAOS_PP_QUICK_OVERLOAD is a constant-time operation at the price of a much lower limit on countable elements.

See Also

  • CHAOS_PP_OVERLOAD

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/facilities/overload.h>

#define MACRO_1(a) -a
#define MACRO_2(a, b) a - b
#define MACRO_3(a, b, c) a - b - c

#if CHAOS_PP_VARIADICS

#define MACRO(...) \
    CHAOS_PP_QUICK_OVERLOAD(MACRO_, __VA_ARGS__)(__VA_ARGS__) \
    /**/

MACRO(1)       // -1
MACRO(1, 2)    // 1 - 2
MACRO(1, 2, 3) // 1 - 2 - 3

#endif

CHAOS_PP_OVERLOAD

The CHAOS_PP_OVERLOAD macro facilitates overloading of macros on the number of arguments.

Usage - C99 Specific 

CHAOS_PP_OVERLOAD(pre, ...)(...)
pre

The prefix of a set of overloaded macros.

[...]

The arguments to be passed to an overloaded macro.

Remarks

This macro counts the number of arguments in __VA_ARGS__ and expands to an expression of the following form: 

pre ## count(__VA_ARGS__)

The number of arguments is never zero--even in situations such as the following: 

CHAOS_PP_OVERLOAD(XYZ_,)()

The above is interpreted as a single empty argument.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_OVERLOAD_BYPASS must be used instead.

See Also

  • CHAOS_PP_QUICK_OVERLOAD

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/facilities/overload.h>

#define MACRO_1(a) a
#define MACRO_2(a, b) a + b
#define MACRO_3(a, b, c) a + b + c

#if CHAOS_PP_VARIADICS

#define MACRO(...) \
    CHAOS_PP_OVERLOAD(MACRO_, __VA_ARGS__)(__VA_ARGS__) \
    /**/

MACRO(1)       // 1
MACRO(1, 2)    // 1 + 2
MACRO(1, 2, 3) // 1 + 2 + 3

#endif

CHAOS_PP_OVERLOAD_BYPASS

The CHAOS_PP_OVERLOAD_BYPASS macro behaves identically to CHAOS_PP_OVERLOAD except that it is parametized by the bypass recursion state.

Usage - C99 Specific 

CHAOS_PP_OVERLOAD_BYPASS(s, pre, ...)(...)
s

The current bypass recursion state.

pre

The prefix of a set of overloaded macros.

[...]

The arguments to be passed to an overloaded macro.

Remarks

If bypass semantics are already in effect, this macro must be used instead of CHAOS_PP_OVERLOAD.
