CHAOS_PP_IS_OPTIONAL

The CHAOS_PP_IS_OPTIONAL macro tests whether a true-optional argument is unused.

Usage - C99 Specific 

CHAOS_PP_IS_OPTIONAL(...)
[...]

The argument to be tested.

Remarks

If __VA_ARGS__ contains only a single argument, this macro expands to 1. Otherwise, it expands to 0.

See Also

  • CHAOS_PP_EXPOSE
  • CHAOS_PP_NON_OPTIONAL
  • CHAOS_PP_OPTIONAL
  • CHAOS_PP_PACK_OPTIONAL

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/facilities/optional.h>

#if CHAOS_PP_VARIADICS

#define MACRO(...) CHAOS_PP_IS_OPTIONAL(__VA_ARGS__)

MACRO(a)    // 1
MACRO(a, b) // 0

#endif

CHAOS_PP_OPTIONAL

The CHAOS_PP_OPTIONAL macro extracts a true-optional argument.

Usage - C99 Specific 

CHAOS_PP_OPTIONAL(...)
[...]

The argument from which a true-optional argument is to be extracted.

Remarks

If __VA_ARGS__ contains more than a single argument, this macro expands to the arguments following the first. Otherwise, it expands to nothing.

See Also

  • CHAOS_PP_EXPOSE
  • CHAOS_PP_IS_OPTIONAL
  • CHAOS_PP_NON_OPTIONAL
  • CHAOS_PP_PACK_OPTIONAL

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/facilities/optional.h>

#if CHAOS_PP_VARIADICS

#define MACRO(...) \
    MACRO_I( \
        CHAOS_PP_NON_OPTIONAL(__VA_ARGS__), \
        CHAOS_PP_OPTIONAL(__VA_ARGS__) \
    ) \
    /**/
#define MACRO_I(a, ...) (a)-(__VA_ARGS__)

MACRO(1)    // (1)-()
MACRO(1, 2) // (1)-(2)

#endif

CHAOS_PP_NON_OPTIONAL

The CHAOS_PP_NON_OPTIONAL macro extracts the non-optional argument to which a true-optional argument may be attached.

Usage - C99 Specific

[...]

CHAOS_PP_PACK_OPTIONAL

The CHAOS_PP_PACK_OPTIONAL argument is used to preserve a true-optional argument (or lack thereof) in subsequent invocations.

Usage - C99 Specific 

CHAOS_PP_PACK_OPTIONAL(...)
[...]

The argument whose true-optional argument is to be packed.

Remarks

If __VA_ARGS__ contains more than a single argument, this macro expands to an expression of the form... 

(, args)

...where args is a placeholder for all the arguments in __VA_ARGS__ except the first. Otherwise, this macro expands to nullary parentheses.

The result of this macro is subsequently unpacked by CHAOS_PP_EXPOSE in a deferred context context to propogate a true-optional argument.

Sample Code

#include <chaos/preprocessor/arithmetic/dec.h>
#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/control/when.h>
#include <chaos/preprocessor/facilities/optional.h>
#include <chaos/preprocessor/recursion/basic.h>
#include <chaos/preprocessor/recursion/expr.h>

#if CHAOS_PP_VARIADICS

#define REPEAT(count, ...) \
    REPEAT_S(CHAOS_PP_STATE(), count, __VA_ARGS__) \
    /**/
#define REPEAT_S(s, count, ...) \
    REPEAT_I( \
        CHAOS_PP_OBSTRUCT(), CHAOS_PP_NEXT(s), count, \
        CHAOS_PP_NON_OPTIONAL(__VA_ARGS__), \
        CHAOS_PP_PACK_OPTIONAL(__VA_ARGS__) \
    ) \
    /**/
#define REPEAT_INDIRECT() REPEAT_I
#define REPEAT_I(_, s, count, macro, data) \
    CHAOS_PP_WHEN _(count)( \
        CHAOS_PP_EXPR_S(s) _(REPEAT_INDIRECT _()( \
            CHAOS_PP_OBSTRUCT _(), CHAOS_PP_NEXT(s), \
            CHAOS_PP_DEC(count), macro, data \
        )) \
        macro _(s, CHAOS_PP_DEC(count) CHAOS_PP_EXPOSE(data)) \
    ) \
    /**/

#define M1(s, n) < n >
#define M2(s, n, data) < n, data >

CHAOS_PP_EXPR(REPEAT(3, M1))    // < 0 > < 1 > < 2 >
CHAOS_PP_EXPR(REPEAT(3, M2, ~)) // < 0, ~ > < 1, ~ > < 2, ~ >

#endif

CHAOS_PP_EXPOSE

The CHAOS_PP_EXPOSE macro unpacks an argument previously packed with CHAOS_PP_PACK_OPTIONAL.

Usage 

CHAOS_PP_EXPOSE(pack)
pack

The parenthetic expression to be unpacked.

Remarks

In C99, this macro expands to the contents of the parenthetic expression pack (which may or may not contain a leading comma). 

#define A ()
#define B (, data)

CHAOS_PP_EXPOSE(A) //
CHAOS_PP_EXPOSE(B) // , data

In C and C++, pack must be a unary parenthetic expression, and this macro expands to the element contained by the unary parenthetic expression plus a leading comma. 

#define A (data)
CHAOS_PP_EXPOSE(A) // , data

This macro is used to propogate true-optional arguments in deferred contexts.
