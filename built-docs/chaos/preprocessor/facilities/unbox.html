The facilities/unbox.h header defines a macro used to abstract empty arguments.

Usage

#include <chaos/preprocessor/facilities/unbox.h>

Contents

  • CHAOS_PP_UNBOX

CHAOS_PP_UNBOX

The CHAOS_PP_UNBOX macro is used to abstract empty arguments.

Usage - C99 Specific 

CHAOS_PP_UNBOX(...)
[...]

The argument to be unboxed.

Usage - C/C++ Specific 

CHAOS_PP_UNBOX(x)
x

The argument to be unboxed.

Remarks

If x is a unary parenthetic expression, this macro expands to the contents of that unary parenthetic expression. In C99, if __VA_ARGS__ is a parenthetic expression of any arity, this macro expands to the contents of that parenthetic expression. Otherwise, this macro expands to nothing.

In C and C++, x must be either a unary parenthetic expression or a non-parenthetic expression. Any other input is invalid.

This macro is used to simulate empty arguments--primarily in C (prior to C99) and C++.

See Also

  • CHAOS_PP_EMPTY
  • CHAOS_PP_IDENTITY

Sample Code

#include <chaos/preprocessor/facilities/unbox.h>

#define MACRO(cv) CHAOS_PP_UNBOX(cv)

MACRO((const))    // const
MACRO((volatile)) // volatile
MACRO(.)          //

CHAOS_PP_UNBOX_ID

The CHAOS_PP_UNBOX_ID macro indirectly expands to CHAOS_PP_UNBOX.

Usage 

CHAOS_PP_UNBOX_ID()

Remarks

This macro effectively allows CHAOS_PP_UNBOX to pass through itself without becoming disabled.

CHAOS_PP_UNBOX_

The CHAOS_PP_UNBOX_ macro expands to a λ expression representing CHAOS_PP_UNBOX.

Usage - C99 Specific 

CHAOS_PP_UNBOX_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_UNBOX_ID)()
