The facilities/type.h header defines a macro used to non-intrusively encode types that contain open commas.

CHAOS_PP_TYPE

The CHAOS_PP_TYPE macro is a rail used to encode types that contain open commas.

Usage - C99 Specific 

CHAOS_PP_TYPE(...)
[...]

The type containing open commas to be encoded.

Remarks

This macro expands to __VA_ARGS__ when (and only when) the context introduced by the CHAOS_PP_WALL macro is present.

The common use case for this macro is currently theoretical, as C++ does not yet allow variadic arguments.

See Also

  • CHAOS_PP_DECODE
  • CHAOS_PP_ENCODE
  • CHAOS_PP_TYPE_II
  • CHAOS_PP_WALL

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/facilities/type.h>
#include <chaos/preprocessor/recursion/rail.h>

#if CHAOS_PP_VARIADICS

#define MACRO(type) type

MACRO(int) // int

CHAOS_PP_WALL(
    MACRO(CHAOS_PP_TYPE(std::pair<int, int>))
)
// std::pair<int, int>

#endif

CHAOS_PP_TYPE_II

The CHAOS_PP_TYPE_II macro is a rail used to encode types that contain open commas.

Usage 

CHAOS_PP_TYPE_II(CHAOS_PP_BEGIN type CHAOS_PP_END)
[type]

The name of a type containing any number of open commas.

Remarks

This macro, combined with CHAOS_PP_BEGIN and CHAOS_PP_END, expands to a deferred expression that expands to type when (and only when) the context introduced by the CHAOS_PP_WALL macro is present.

This macro is used to pass types that contain open commas--even in when variadics are not enabled--without having to count commas.

See Also

  • CHAOS_PP_DECODE
  • CHAOS_PP_ENCODE
  • CHAOS_PP_TYPE

Sample Code

#include <chaos/preprocessor/facilities/type.h>
#include <chaos/preprocessor/recursion/expr.h>
#include <chaos/preprocessor/recursion/rail.h>

#define A(x) B(x)
#define B(x) C(x)
#define C(x) D(x)
#define D(x) x

CHAOS_PP_EXPR(CHAOS_PP_WALL(
    A(
        CHAOS_PP_TYPE_II(CHAOS_PP_BEGIN std::pair<int, int> CHAOS_PP_END)
    )
))
// std::pair<int, int>
