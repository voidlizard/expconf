The facilities/whitespace.h header defines macros used to remove whitespace from an argument.

Usage

#include <chaos/preprocessor/facilities/whitespace.h>

Contents

  • CHAOS_PP_CLEAN
  • CHAOS_PP_NO_LEADING
  • CHAOS_PP_NO_TRAILING

CHAOS_PP_NO_LEADING

The CHAOS_PP_NO_LEADING macro removes leading whitespace from its argument.

Usage - C99 Specific 

CHAOS_PP_NO_LEADING(...)
[...]

The argument whose leading whitespace is to be removed.

Remarks

This macro removes leading whitespace (if any) from its argument by concatenating a placemarker to the first preprocessing token of __VA_ARGS__.

This macro is only useful for subsequent production of string literals (via the stringizing operator) or header-name preprocessing tokens of the h-char-sequence form.

See Also

  • CHAOS_PP_CLEAN
  • CHAOS_PP_NO_TRAILING

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/facilities/empty.h>
#include <chaos/preprocessor/facilities/whitespace.h>

#if CHAOS_PP_VARIADICS

#define SPACE CHAOS_PP_EMPTY() CHAOS_PP_EMPTY()

#define A(...) __VA_ARGS__
#define B(...) CHAOS_PP_NO_LEADING(__VA_ARGS__)

!A(SPACE X) // ! X
!B(SPACE X) // !X

#endif

CHAOS_PP_NO_LEADING_ID

The CHAOS_PP_NO_LEADING_ID macro indirectly expands to CHAOS_PP_NO_LEADING.

Usage - C99 Specific 

CHAOS_PP_NO_LEADING_ID()

Remarks

This macro effectively allows CHAOS_PP_NO_LEADING to pass through itself without becoming disabled.

CHAOS_PP_NO_LEADING_

The CHAOS_PP_NO_LEADING_ macro expands to a λ expression representing CHAOS_PP_NO_LEADING.

Usage - C99 Specific 

CHAOS_PP_NO_LEADING_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_NO_LEADING_ID)()

CHAOS_PP_CLEAN

The CHAOS_PP_CLEAN macro removes leading and trailing whitespace from its argument.

Usage - C99 Specific 

CHAOS_PP_CLEAN(...)
[...]

The argument whose leading and trailing whitespace is to be removed.

Remarks

This macro removes both leading and trailing whitespace (if any) from its argument by concatenating a placemarker to both the first and the last preprocessing token of __VA_ARGS__.

This macro is only useful for subsequent production of string literals (via the stringizing operator) or header-name preprocessing tokens of the h-char-sequence form.

See Also

  • CHAOS_PP_NO_LEADING
  • CHAOS_PP_NO_TRAILING

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/facilities/empty.h>
#include <chaos/preprocessor/facilities/whitespace.h>

#if CHAOS_PP_VARIADICS

#define SPACE CHAOS_PP_EMPTY() CHAOS_PP_EMPTY()

#define A(...) __VA_ARGS__
#define B(...) CHAOS_PP_CLEAN(__VA_ARGS__)

[A(SPACE X SPACE)] // [ X ]
[B(SPACE X SPACE)] // [X]

#endif
