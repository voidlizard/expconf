The facilities/push.h header defines a macro that splices two arguments.

Usage

#include <chaos/preprocessor/facilities/push.h>

Contents

  • CHAOS_PP_PUSH

CHAOS_PP_PUSH

The CHAOS_PP_PUSH macro splices two arguments by placing them directly adjacent to each other.

Usage - C99 Specific 

CHAOS_PP_PUSH(a, ...)
[a]

The first argument.

[...]

The second argument.

Usage - C/C++ Specific 

CHAOS_PP_PUSH(a, b)
a

The first argument.

b

The second argument.

Remarks

This macro facilitates the placement of two macro arguments directly adjacent to one another which is impossible directly. 

#define A(x, y) CHAOS_PP_STRINGIZE(x y)

A(1,2) // "1 2"

Normally, such whitespace has no semantic effect, but it can during the subsequent production of string literals (via the stringizing operator) or header-name preprocessing tokens of the h-char-sequence form. 

#define B(x, y) CHAOS_PP_STRINGIZE(CHAOS_PP_PUSH(x,y))

B(1,2) // "12"

Note that an argument can include surrounding whitespace: 

#define C(x) [x]

C(y)   // [y]
C( z ) // [ z ]

There is nothing that can be done about this in C (prior to C99) and C++ except to be extremely careful when whitespace matters. In C99, however, this macro strips leading and trailing whitespace.

Also note that this macro does not concatenate the two arguments. For example, 

CHAOS_PP_PUSH(+,+)

does not result in a single ++ preprocessing token. Rather, it results in two adjacent + preprocessing tokens with no intervening whitespace. This occurs because the preprocessor operates on preprocessing tokens, not on their textual representation.

Sample Code

#include <chaos/preprocessor/facilities/push.h>

#define H(a, b) <CHAOS_PP_PUSH(a,b).h>

#include H(+, +) // <++.h>
#include H(+, -) // <+-.h>

CHAOS_PP_PUSH_ID

The CHAOS_PP_PUSH_ID macro indirectly expands to CHAOS_PP_PUSH.

Usage 

CHAOS_PP_PUSH_ID()

Remarks

This macro effectively allows CHAOS_PP_PUSH to pass through itself without becoming disabled.

CHAOS_PP_PUSH_

The CHAOS_PP_PUSH_ macro expands to a λ expression representing CHAOS_PP_PUSH.

Usage - C99 Specific 

CHAOS_PP_PUSH_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_PUSH_ID)()
