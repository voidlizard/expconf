The facilities/encode.h header defines macros used to encode and decode arguments containing commas.

Usage

#include <chaos/preprocessor/facilities/encode.h>

Contents

  • CHAOS_PP_DECODE
  • CHAOS_PP_ENCODE

CHAOS_PP_ENCODE

The CHAOS_PP_ENCODE macro encodes its arguments by wrapping them in parentheses.

Usage - C99 Specific 

CHAOS_PP_ENCODE(...)
[...]

The arguments to be encoded.

Remarks

This macro is used as a complement to CHAOS_PP_DECODE to pass an argument that is logically a single argument but is physically more than one argument.

See Also

  • CHAOS_PP_DECODE
  • CHAOS_PP_TYPE
  • CHAOS_PP_TYPE_II

CHAOS_PP_ENCODE_ID

The CHAOS_PP_ENCODE_ID macro indirectly expands to CHAOS_PP_ENCODE.

Usage - C99 Specific 

CHAOS_PP_ENCODE_ID()

Remarks

This macro effectively allows CHAOS_PP_ENCODE to pass through itself without becoming disabled.

CHAOS_PP_ENCODE_

The CHAOS_PP_ENCODE_ macro expands to a λ expression representing CHAOS_PP_ENCODE.

Usage - C99 Specific 

CHAOS_PP_ENCODE_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_ENCODE_ID)()

CHAOS_PP_DECODE

The CHAOS_PP_DECODE macro decodes an argument.

Usage - C99 Specific 

CHAOS_PP_DECODE(...)
[...]

The argument to be decoded.

Remarks

This macro is used to decode arguments possibly encoded with CHAOS_PP_ENCODE.

If __VA_ARGS__ is a parenthetic expression, this macro expands to the contents of that parenthetic expression. Otherwise, it expands to __VA_ARGS__.

The common use case for this macro is currently theoretical, as C++ does not yet allow variadic arguments.

See Also

  • CHAOS_PP_ENCODE
  • CHAOS_PP_TYPE
  • CHAOS_PP_TYPE_II

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/facilities/encode.h>

#if CHAOS_PP_VARIADICS

#define MACRO(type) CHAOS_PP_DECODE(type)

MACRO(int) // int
MACRO(CHAOS_PP_ENCODE(std::pair<int, int>)) // std::pair<int, int>

#endif
