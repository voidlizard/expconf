CHAOS_PP_EMPTY

The CHAOS_PP_EMPTY macro expands to nothing.

Usage 

CHAOS_PP_EMPTY()

Remarks

This macro is useful in contexts where it is necessary to express emptiness programmatically.

In C99, macro arguments are allowed to be empty. However, this is not the case in C and C++. This macro, combined with CHAOS_PP_IDENTITY, can be used to simulate empty arguments. 

#define MACRO(cv) cv()

MACRO(CHAOS_PP_IDENTITY(const)) // const
MACRO(CHAOS_PP_EMPTY)           //

See Also

  • CHAOS_PP_IDENTITY
  • CHAOS_PP_UNBOX

Sample Code

#include <chaos/preprocessor/facilities/empty.h>
#include <chaos/preprocessor/punctuation/comma.h>
#include <chaos/preprocessor/recursion/expr.h>
#include <chaos/preprocessor/repetition/delineate.h>

#define MACRO(s, n, _) n

CHAOS_PP_EXPR(
    CHAOS_PP_DELINEATE(3, CHAOS_PP_EMPTY, MACRO, ~)
)
// 0 1 2

CHAOS_PP_EXPR(
    CHAOS_PP_DELINEATE(3, CHAOS_PP_COMMA, MACRO, ~)
)
// 0, 1, 2

CHAOS_PP_EMPTY_ID

The CHAOS_PP_EMPTY_ID macro indirectly expands to CHAOS_PP_EMPTY.

Usage 

CHAOS_PP_EMPTY_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_EMPTY_

The CHAOS_PP_EMPTY_ macro expands to a λ expression representing CHAOS_PP_EMPTY.

Usage - C99 Specific 

CHAOS_PP_EMPTY_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_EMPTY)
