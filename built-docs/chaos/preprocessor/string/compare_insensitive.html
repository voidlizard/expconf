The string/compare_insensitive.h header defines an algorithm that compares two strings for case-insensitive equality.

Usage

#include <chaos/preprocessor/string/compare_insensitive.h>

Contents

  • CHAOS_PP_STRING_COMPARE_INSENSITIVE

CHAOS_PP_STRING_COMPARE_INSENSITIVE

The CHAOS_PP_STRING_COMPARE_INSENSITIVE macro compares two strings for case-insensitive equality.

Usage - C99 Specific 

CHAOS_PP_STRING_COMPARE_INSENSITIVE(a, b)
[a]

The first string.

[b]

The second string.

Usage - C/C++ Specific 

CHAOS_PP_STRING_COMPARE_INSENSITIVE(a, b)
a

The first string.

b

The second string.

Remarks

If a and b are both the same string (ignoring case), this macro expands to 1. Otherwise, it expands to 0.

The efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper bound of O(n+1), where n is the number of characters in a and b, occurs when a and a are equal (ignoring case). The asymptotic lower bound of Ω(1) occurs when the first character of a differs from the first character of b.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_STRING_COMPARE_INSENSITIVE_BYPASS must be used instead.

See Also

  • CHAOS_PP_STRING_COMPARE

Sample Code

#include <chaos/preprocessor/string/compare_insensitive.h>

CHAOS_PP_STRING_COMPARE_INSENSITIVE(a b c, A B C) // 1

CHAOS_PP_STRING_COMPARE_INSENSITIVE_

CHAOS_PP_STRING_COMPARE_INSENSITIVE_BYPASS

The CHAOS_PP_STRING_COMPARE_INSENSITIVE_BYPASS macro behaves identically to CHAOS_PP_STRING_COMPARE_INSENSITIVE except that it is parametized by the bypass recursion state.

Usage - C99 Specific 

CHAOS_PP_STRING_COMPARE_INSENSITIVE_BYPASS(s, a, b)
s

The current bypass recursion state.

[a]

The first string.

[b]

The second string.

Usage - C/C++ Specific 

CHAOS_PP_STRING_COMPARE_INSENSITIVE_BYPASS(s, a, b)
s

The current bypass recursion state.

a

The first string.

b

The second string.

Remarks

If bypass semantics are already in effect, this macro must be used instead of CHAOS_PP_STRING_COMPARE_INSENSITIVE.
