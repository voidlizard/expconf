The control/if.h header defines a macro that acts as an "if-else" construct.

Usage

#include <chaos/preprocessor/control/if.h>

Contents

  • CHAOS_PP_IF

CHAOS_PP_IF

The CHAOS_PP_IF macro expands to one of two alternatives depending on its first argument.

Usage - C99 Specific 

CHAOS_PP_IF(cond)(t, ...)
cond

The condition that determines whether the macro expands to t or __VA_ARGS__. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

[t]

The resulting expression if cond is non-zero.

[...]

The resulting expression if cond is 0.

Usage - C/C++ Specific 

CHAOS_PP_IF(cond)(t, f)
cond

The condition that determines whether the macro expands to t or f. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

t

The resulting expression if cond is non-zero.

f

The resulting expression if cond is 0.

Remarks

If cond expands to 0, this macro expands to f or __VA_ARGS__. Otherwise, it expands to t.

This macro performs a Boolean conversion on cond. If this conversion is undesirable, CHAOS_PP_IIF may be used instead.

This macro is lazy. If cond is non-zero, only t is evaluated. Likewise, if cond is 0, only f (or __VA_ARGS__) is evaluated.

Sample Code

#include <chaos/preprocessor/control/if.h>

CHAOS_PP_IF(1)(-, +) // -
CHAOS_PP_IF(0)(-, +) // +

CHAOS_PP_IF_ID

The CHAOS_PP_IF_ID macro indirectly expands to CHAOS_PP_IF.

Usage 

CHAOS_PP_IF_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_IF_

The CHAOS_PP_IF_ macro expands to a λ expression representing CHAOS_PP_IF.

Usage - C99 Specific 

CHAOS_PP_IF_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_IF)
