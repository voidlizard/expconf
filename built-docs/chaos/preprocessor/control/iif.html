The control/iif.h header defines a macro that acts as an "if-else" construct.

Usage

#include <chaos/preprocessor/control/iif.h>

Contents

  • CHAOS_PP_IIF

CHAOS_PP_IIF

The CHAOS_PP_IIF macro expands to one of two alternatives depending on its first argument.

Usage - C99 Specific 

CHAOS_PP_IF(bit)(t, ...)
bit

The condition that determines whether the macro expands to t or __VA_ARGS__. Valid values are 0 and 1.

[t]

The resulting expression if bit is 1.

[...]

The resulting expression if bit is 0.

Usage - C/C++ Specific 

CHAOS_PP_IIF(bit)(t, f)
bit

The condition that determines whether the macro expands to t or f. Valid values are 0 and 1.

t

The resulting expression if bit is 1.

f

The resulting expression if bit is 0.

Remarks

If bit expands to 0, this macro expands to f or __VA_ARGS__. Otherwise, it expands to t.

This macro does not perform a Boolean conversion on bit. If this conversion is desirable, CHAOS_PP_IF may be used instead.

This macro is lazy. If bit is 1, only t is evaluated. Likewise, if bit is 0, only f (or __VA_ARGS__) is evaluated.

Sample Code

#include <chaos/preprocessor/control/iif.h>

CHAOS_PP_IIF(1)(-, +) // -
CHAOS_PP_IIF(0)(-, +) // +

CHAOS_PP_IIF_ID

The CHAOS_PP_IIF_ID macro indirectly expands to CHAOS_PP_IIF.

Usage 

CHAOS_PP_IIF_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_IIF_

The CHAOS_PP_IIF_ macro expands to a λ expression representing CHAOS_PP_IIF.

Usage - C99 Specific 

CHAOS_PP_IIF_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_IIF)
