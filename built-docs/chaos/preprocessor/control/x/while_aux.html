The control/x/while_aux.h header defines an exponential looping construct which can be parametrically resumed.

Usage

#include <chaos/preprocessor/control/x/while_aux.h>

Contents

  • CHAOS_PP_WHILE_AUX_X

CHAOS_PP_WHILE_AUX_X

The CHAOS_PP_WHILE_AUX_X macro iterates a state with a user-specified operation while a user-specified predicate expands to 1.

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_WHILE_AUX_X(size, pred, op, data, ...))
size

The size of the buffer of valid recursion steps maintained by the algorithm.

pred

The predicate that determines whether the loop should continue. This predicate must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

pred(s, data†, state‡)

† If the auxiliary data is parenthetic, the data pseudo-parameter must be the number of parameters that corresponds to the number of sub-elements in the auxiliary data. Otherwise, it must be a single parameter.

‡ The state pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the initial state, __VA_ARGS__.

op

The operation used to iterate the state. This operation must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

op(s, data†, state‡)

† If the auxiliary data is parenthetic, the data pseudo-parameter must be the number of parameters that corresponds to the number of sub-elements in the auxiliary data. Otherwise, it must be a single parameter.

‡ The state pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the initial state, __VA_ARGS__.

[data]

Auxiliary data to be passed through the algorithm to the predicate and the operation. This argument may be either a parenthetic expression of any arity or a non-parenthetic expression.

[...]

The initial state of the loop.

Usage - C/C++ Specific 

CHAOS_PP_EXPR(CHAOS_PP_WHILE_AUX_X(size, pred, op, data, state))
size

The size of the buffer of valid recursion steps maintained by the algorithm.

pred

The predicate that determines whether the loop should continue. This predicate must be a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

pred(s, data, state)
op

The operation used to iterate the state. This operation must be a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

op(s, data, state)
data

Auxiliary data to be passed through the algorithm to the predicate and the operation. This argument may be either a unary parenthetic expression or a non-parenthetic expression.

state

The initial state of the loop.

Remarks

This macro behaves identically to CHAOS_PP_WHILE_AUX except that the algorithm makes exponential use of the recursion backend and the algorithm itself yields a trampolined deferred expression whenever the number of recursion steps is exhausted. This allows for parametric resumption.

This algorithm maintains a buffer of size recursion steps available at all times. This guarantees that the predicate and operation have at least size recursion steps available whenever they are invoked.

This macro deduces the current recursion state and invokes CHAOS_PP_WHILE_AUX_X_S. If the current recursion state is already known, direct use of CHAOS_PP_WHILE_AUX_X_S is more efficient.

This macro may not be used if bypass semantics are in effect.

CHAOS_PP_WHILE_AUX_X_ID

The CHAOS_PP_WHILE_AUX_X_ID macro indirectly expands to CHAOS_PP_WHILE_AUX_X.

Usage 

CHAOS_PP_WHILE_AUX_X_ID()

Remarks

This macro effectively allows CHAOS_PP_WHILE_AUX_X to pass through itself without becoming disabled.

CHAOS_PP_WHILE_AUX_X_

The CHAOS_PP_WHILE_AUX_X_ macro expands to a λ expression representing CHAOS_PP_WHILE_AUX_X.

Usage - C99 Specific 

CHAOS_PP_WHILE_AUX_X_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_WHILE_AUX_X_ID)()
