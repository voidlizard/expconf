The arithmetic/sub.h header defines a subtraction operator.

Usage

#include <chaos/preprocessor/arithmetic/sub.h>

Contents

  • CHAOS_PP_SUB

CHAOS_PP_SUB

The CHAOS_PP_SUB macro computes the difference between two values.

Usage 

CHAOS_PP_SUB(x, y)
x

The minuend. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

y

The subtrahend. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

Remarks

This macro expands to the difference between x and y. If the result is less than 0, it is saturated to 0.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_SUB_BYPASS must be used instead.

See Also

  • CHAOS_PP_ARBITRARY_SUB
  • CHAOS_PP_SUB_MACHINE

Sample Code

#include <chaos/preprocessor/arithmetic/sub.h>

CHAOS_PP_SUB(23, 9) // 14

CHAOS_PP_SUB_ID

The CHAOS_PP_SUB_ID macro indirectly expands to CHAOS_PP_SUB.

Usage 

CHAOS_PP_SUB_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_SUB_

The CHAOS_PP_SUB_ macro expands to a λ expression representing CHAOS_PP_SUB.

Usage - C99 Specific 

CHAOS_PP_SUB_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_SUB)

CHAOS_PP_SUB_BYPASS

The CHAOS_PP_SUB_BYPASS macro behaves identically to CHAOS_PP_SUB except that it is parametized by the bypass recursion state.

Usage 

CHAOS_PP_SUB_BYPASS(s, x, y)
s

The current bypass recursion state.

x

The minuend. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

y

The subtrahend. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

Remarks

If bypass semantics are already in effect, this macro must be used instead of CHAOS_PP_SUB.

CHAOS_PP_SUB_BYPASS_ID

The CHAOS_PP_SUB_BYPASS_ID macro indirectly expands to CHAOS_PP_SUB_BYPASS.

Remarks

CHAOS_PP_SUB_BYPASS_

Usage - C99 Specific 

CHAOS_PP_SUB_BYPASS_

Remarks
