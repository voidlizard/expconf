The arithmetic/mul.h header defines a multiplication operator.

Usage

#include <chaos/preprocessor/arithmetic/mul.h>

Contents

  • CHAOS_PP_MUL

CHAOS_PP_MUL

The CHAOS_PP_MUL macro computes the product of two values.

Usage 

CHAOS_PP_MUL(x, y)
x

The multiplicand. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

y

The multiplier. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

Remarks

This macro expands to the product of x and y. If the result is greater than CHAOS_PP_LIMIT_MAG, it is saturated to CHAOS_PP_LIMIT_MAG.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_MUL_BYPASS must be used instead.

See Also

  • CHAOS_PP_ARBITRARY_MUL
  • CHAOS_PP_MUL_MACHINE

Sample Code

#include <chaos/preprocessor/arithmetic/mul.h>

CHAOS_PP_MUL(5, 7) // 35

CHAOS_PP_MUL_ID

The CHAOS_PP_MUL_ID macro indirectly expands to CHAOS_PP_MUL.

Usage 

CHAOS_PP_MUL_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_MUL_

The CHAOS_PP_MUL_ macro expands to a λ expression representing CHAOS_PP_MUL.

Usage - C99 Specific 

CHAOS_PP_MUL_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_MUL)

CHAOS_PP_MUL_BYPASS

The CHAOS_PP_MUL_BYPASS macro behaves identically to CHAOS_PP_MUL except that it is parametized by the bypass recursion state.

Usage 

CHAOS_PP_MUL_BYPASS(s, x, y)
s

The current bypass recursion state.

x

The multiplicand. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

y

The multiplier. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

Remarks

If bypass semantics are already in effect, this macro must be used instead of CHAOS_PP_MUL.

CHAOS_PP_MUL_BYPASS_ID

The CHAOS_PP_MUL_BYPASS_ID macro indirectly expands to CHAOS_PP_MUL_BYPASS.

Usage

CHAOS_PP_MUL_BYPASS_

Usage - C99 Specific
