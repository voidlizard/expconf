The arithmetic/mod.h header defines a remainder operator.

Usage

#include <chaos/preprocessor/arithmetic/mod.h>

Contents

  • CHAOS_PP_MOD

CHAOS_PP_MOD

The CHAOS_PP_MOD macro computes the remainder produced by dividing two values.

Usage 

CHAOS_PP_MOD(x, y)
x

The base. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

y

The modulus. Valid values range from 1 to CHAOS_PP_LIMIT_MAG.

Remarks

This macro expands to the remainder produced by dividing x by y. If y is 0, this macro intentionally produces an error.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_MOD_BYPASS must be used instead.

Sample Code

#include <chaos/preprocessor/arithmetic/mod.h>

CHAOS_PP_MOD(10, 3) // 1

CHAOS_PP_MOD_ID

The CHAOS_PP_MOD_ID macro indirectly expands to CHAOS_PP_MOD.

Usage 

CHAOS_PP_MOD_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_MOD_

The CHAOS_PP_MOD_ macro expands to a λ expression representing CHAOS_PP_MOD.

Usage - C99 Specific 

CHAOS_PP_MOD_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_MOD)

CHAOS_PP_MOD_BYPASS

The CHAOS_PP_MOD_BYPASS macro behaves identically to CHAOS_PP_MOD except that it is parametized by the bypass recursion state.

Usage 

CHAOS_PP_MOD_BYPASS(s, x, y)
s

The current bypass recursion state.

x

The base. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

y

The modulus. Valid values range from 1 to CHAOS_PP_LIMIT_MAG.

Remarks

If bypass semantics are already in effect, this macro must be used instead of CHAOS_PP_MOD.

CHAOS_PP_MOD_BYPASS_ID

The CHAOS_PP_MOD_BYPASS_ID macro indirectly expands to CHAOS_PP_MOD_BYPASS.

Usage

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_MOD_BYPASS_

Usage - C99 Specific 

CHAOS_PP_MOD_BYPASS_

Remarks
