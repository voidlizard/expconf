The arbitrary/literal.h header defines a macro that produces an arbitrary-precision value from a series of digits.

Usage

#include <chaos/preprocessor/arbitrary/literal.h>

Contents

  • CHAOS_PP_ARBITRARY_LITERAL

CHAOS_PP_ARBITRARY_LITERAL

The CHAOS_PP_ARBITRARY_LITERAL macro converts a series of digits to an arbitrary-precision value.

Usage 

CHAOS_PP_ARBITRARY_LITERAL(literal)
literal

The series of digits to be converted to an arbitrary-precision value. The format of this digit series is similar to a string, but it is restricted to numerals and is optionally prefixed by a parenthesized negation operator.

Remarks

This macro expands to an arbitrary-precision value that represents the series of digits in literal.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_ARBITRARY_LITERAL_BYPASS must be used instead.

Sample Code

#include <chaos/preprocessor/arbitrary/literal.h>

CHAOS_PP_ARBITRARY_LITERAL(1 2 3)  // (1)(2)(3)
CHAOS_PP_ARBITRARY_LITERAL((-)2 0) // ((2)(0))

CHAOS_PP_ARBITRARY_LITERAL_ID

The CHAOS_PP_ARBITRARY_LITERAL_ID macro indirectly expands to CHAOS_PP_ARBITRARY_LITERAL.

Usage 

CHAOS_PP_ARBITRARY_LITERAL_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_ARBITRARY_LITERAL_

The CHAOS_PP_ARBITRARY_LITERAL_ macro expands to a λ expression representing CHAOS_PP_ARBITRARY_LITERAL.

Usage - C99 Specific 

CHAOS_PP_ARBITRARY_LITERAL_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_ARBITRARY_LITERAL)

CHAOS_PP_ARBITRARY_LITERAL_BYPASS

The CHAOS_PP_ARBITRARY_LITERAL_BYPASS macro behaves identically to CHAOS_PP_ARBITRARY_LITERAL except that it is parametized by the bypass recursion state.

Usage 

CHAOS_PP_ARBITRARY_LITERAL_BYPASS(s, literal)
s

The current bypass recursion state.

literal

The series of digits to be converted to an arbitrary-precision value. The format of this digit series is similar to a string, but it is restricted to numerals and is optionally prefixed by a parenthesized negation operator.

Remarks

If bypass semantics are already in effect, this macro must be used instead of CHAOS_PP_ARBITRARY_LITERAL.
