The seq/variadic/transform.h header defines an algorithm that transforms the elements of a variadic sequence with a user-specified operation.

Usage

#include <chaos/preprocessor/seq/variadic/transform.h>

Contents

  • CHAOS_PP_VARIADIC_SEQ_TRANSFORM

CHAOS_PP_VARIADIC_SEQ_TRANSFORM

The CHAOS_PP_VARIADIC_SEQ_TRANSFORM macro transforms each element of its second argument with the operation specified by its first argument.

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_VARIADIC_SEQ_TRANSFORM(op, seq))
op

The operation used to transform each element of seq. This operation must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

op(s, tuple)
seq

The variadic sequence whose elements are to be transformed.

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_VARIADIC_SEQ_TRANSFORM(op, seq, ...))
op

The operation used to transform each element of seq. This operation must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

op(s, tuple, data†)

† The data pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the auxiliary data, __VA_ARGS__.

seq

The variadic sequence whose elements are to be transformed.

[...]

Auxiliary data to be passed through the algorithm to the operation.

Remarks

This macro behaves identically to CHAOS_PP_SEQ_TRANSFORM except that each element is wrapped in parentheses prior to being passed to the operation. For example, instead of the the operation being invoked as... 

op(s, elem, data)

it is invoked as... 

op(s, (elem), data)

This macro deduces the current recursion state and invokes CHAOS_PP_VARIADIC_SEQ_TRANSFORM_S. If the current recursion state is already known, direct use of CHAOS_PP_VARIADIC_SEQ_TRANSFORM_S is more efficient.

This macro may not be used if bypass semantics are in effect.

CHAOS_PP_VARIADIC_SEQ_TRANSFORM_ID

The CHAOS_PP_VARIADIC_SEQ_TRANSFORM_ID macro indirectly expands to CHAOS_PP_VARIADIC_SEQ_TRANSFORM.

Usage - C99 Specific 

CHAOS_PP_VARIADIC_SEQ_TRANSFORM_ID()

Remarks

This macro effectively allows CHAOS_PP_VARIADIC_SEQ_TRANSFORM to pass through itself without becoming disabled.

CHAOS_PP_VARIADIC_SEQ_TRANSFORM_

The CHAOS_PP_VARIADIC_SEQ_TRANSFORM_ macro expands to a λ expression representing CHAOS_PP_VARIADIC_SEQ_TRANSFORM.

Usage - C99 Specific 

CHAOS_PP_VARIADIC_SEQ_TRANSFORM_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_VARIADIC_SEQ_TRANSFORM_ID)()

CHAOS_PP_VARIADIC_SEQ_TRANSFORM_S

The CHAOS_PP_VARIADIC_SEQ_TRANSFORM_S macro behaves identically to CHAOS_PP_VARIADIC_SEQ_TRANSFORM except that it is parametized by the recursion state.

Usage - C99 Specific 

CHAOS_PP_EXPR_S(s)(CHAOS_PP_VARIADIC_SEQ_TRANSFORM_S(s, op, seq))
s

The current recursion state.

op

The operation used to transform each element of seq. This operation must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

op(s, tuple)
seq

The variadic sequence whose elements are to be transformed.

Usage - C99 Specific 

CHAOS_PP_EXPR_S(s)(CHAOS_PP_VARIADIC_SEQ_TRANSFORM_S(s, op, seq, ...))
s

The current recursion state.

op

The operation used to transform each element of seq. This operation must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

op(s, tuple, data†)

† The data pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the auxiliary data, __VA_ARGS__.

seq

The variadic sequence whose elements are to be transformed.

[...]

Auxiliary data to be passed through the algorithm to the operation.

Remarks
