The seq/reverse.h header defines a macro that reverses the elements in a sequence.

Usage

#include <chaos/preprocessor/seq/reverse.h>

Contents

CHAOS_PP_SEQ_REVERSE

The CHAOS_PP_SEQ_REVERSE macro reverses the elements in its argument.

Usage - C99 Specific 

CHAOS_PP_SEQ_REVERSE(seq)
[seq]

The sequence whose elements are to be reversed.

Usage - C/C++ Specific 

CHAOS_PP_SEQ_REVERSE(seq)
seq

The sequence whose elements are to be reversed.

Remarks

This macro expands to a sequence containing the elements of seq in reverse order.

This macro uses sequential iteration. As a result, the algorithm is theoretically capable of processing sequences containing an unrestricted number of elements.

The structure of this algorithm causes approximately Θ(n2) element scans (where n is the number of elements in seq). This scanning may be inefficient when seq contains a large number of elements. In such a situation in C99, it may be more efficient to use CHAOS_PP_SEQ_REVERSE_DUPLEX instead. The structure of the CHAOS_PP_SEQ_REVERSE_DUPLEX algorithm causes approximately Θ(n∙lgn) element scans. On the other hand, though both CHAOS_PP_SEQ_REVERSE and CHAOS_PP_SEQ_REVERSE_DUPLEX invoke approximately Θ(n) macros, the discarded constant multiple is significantly larger for CHAOS_PP_SEQ_REVERSE_DUPLEX. This causes CHAOS_PP_SEQ_REVERSE to be more efficient when n is smaller. Thus, as n increases, at some point (depending on the preprocessor) CHAOS_PP_SEQ_REVERSE_DUPLEX is likely to become more efficient.

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/seq/reverse.h>

CHAOS_PP_SEQ_REVERSE((a)(b)(c)) // (c)(b)(a)

#if CHAOS_PP_VARIADICS

CHAOS_PP_SEQ_REVERSE() //
CHAOS_PP_SEQ_REVERSE((a, b)(c, d)(e, f)) // (e, f)(c, d)(a, b)

#endif

CHAOS_PP_SEQ_REVERSE_ID

Usage

Remarks

CHAOS_PP_SEQ_REVERSE_DUPLEX

The CHAOS_PP_SEQ_REVERSE_DUPLEX macro reverses the elements in its argument.

Usage - C99 Specific 

CHAOS_PP_SEQ_REVERSE_DUPLEX(seq)
[seq]

The sequence whose elements are to be reversed.

Remarks

This macro expands to a sequence containing the elements of seq in reverse order.

The efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Θ, where n is the number of elements in seq.

As with CHAOS_PP_SEQ_REVERSE, the structure of this algorithm causes the invocation of approximately Θ(n) macros (where n is the number of elements in seq). However, the discarded constant multiple is significantly larger. This macro expansion may be inefficient when seq contains a small number of elements. In such a situation, it is more efficient to use CHAOS_PP_SEQ_REVERSE instead. On the other hand, the structure of the CHAOS_PP_SEQ_REVERSE algorithm causes approximately Θ(n2) element scans. The structure of the CHAOS_PP_SEQ_REVERSE_DUPLEX algorithm causes approximately Θ(n∙lgn) element scans. Thus, as n increases, at some point (depending on the preprocessor) CHAOS_PP_SEQ_REVERSE_DUPLEX is likely to become more efficient.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_SEQ_REVERSE_DUPLEX_BYPASS must be used instead.

CHAOS_PP_SEQ_REVERSE_DUPLEX_BYPASS

The CHAOS_PP_SEQ_REVERSE_DUPLEX_BYPASS macro behaves identically to CHAOS_PP_SEQ_REVERSE_DUPLEX except that it is parametized by the bypass recursion state.

Usage - C99 Specific 

CHAOS_PP_SEQ_REVERSE_DUPLEX_BYPASS(s, seq)
s

The current bypass recursion state.

[seq]

The sequence whose elements are to be reversed.

Remarks

If bypass semantics are already in effect, this macro must be used instead of CHAOS_PP_SEQ_REVERSE_DUPLEX.
