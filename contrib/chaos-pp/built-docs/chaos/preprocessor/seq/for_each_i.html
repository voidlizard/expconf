The seq/for_each_i.h header defines an algorithm that repeats a user-specified target for each element in a sequence.

Usage

#include <chaos/preprocessor/seq/for_each_i.h>

Contents

  • CHAOS_PP_SEQ_FOR_EACH_I

CHAOS_PP_SEQ_FOR_EACH_I

The CHAOS_PP_SEQ_FOR_EACH_I macro repeats the target specified by its first argument for each element in its second argument.

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_SEQ_FOR_EACH_I(macro, seq))
macro

The target that is to be repeated for each element in seq. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, i, elem†)

† If seq contains elements that have varying arities, elem mus be .... Otherwise, elem must be the number of parameters that corresponds to the arity of each element in seq.

[seq]

The sequence for which macro is to be repeated for each element.

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_SEQ_FOR_EACH_I(macro, seq, ...))
macro

The target that is to be repeated for each element in seq. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, i, elem†, data‡)

† The elem pseudo-parameter must be the number of parameters that corresponds to the arity of each element in seq.

‡ The data pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the auxiliary data, __VA_ARGS__.

[seq]

The sequence for which macro is to be repeated for each element.

[...]

Auxiliary data to be passed through the algorithm to the target.

Usage - C/C++ Specific 

CHAOS_PP_EXPR(CHAOS_PP_SEQ_FOR_EACH_I(macro, seq, data))
macro

The target that is to be repeated for each element in seq. This target must be a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, i, elem, data)
seq

The sequence for which macro is to be repeated for each element.

data

Auxiliary data to be passed through the algorithm to the target.

Remarks

The CHAOS_PP_SEQ_FOR_EACH_I macro itself expands to a deferred expression in terms of CHAOS_PP_NEXT(s). This deferred expression, when invoked by CHAOS_PP_EXPR, expands to the result of the repeated invocation of the target for each element in seq. This is illustrated by the following pseudo-code (where N is the index of the last element in seq): 

macro(s, 0, seq[0], data) macro(s, 1, seq[1], data) ... macro(s, N, seq[N], data)

The target is invoked with the current recursion state (s), the index of a particular element (i), a particular element (elem), and, if it exists, the auxiliary data (data or __VA_ARGS__). This target may expand to a deferred expression in terms of CHAOS_PP_NEXT(s). This allows the target to recurse on itself if necesssary.

In C99, it is possible to omit the auxiliary data argument altogether. This is different than passing "nothing" as the auxiliary data. 

CHAOS_PP_EXPR(
    CHAOS_PP_SEQ_FOR_EACH_I(MACRO, (a)(b)(c),) // #1
    CHAOS_PP_SEQ_FOR_EACH_I(MACRO, (a)(b)(c))  // #2
)

The first line passes "nothing" as the auxiliary data, which causes MACRO to be invoked as (e.g.)... 

MACRO(s, 0, a,)

The second line omits the auxiliary data, which causes MACRO to be invoked as (e.g.)... 

MACRO(s, 0, a)

The fundamental difference is that, in the first line, the auxiliary data is empty (which might have a valid meaning as data), but, in the second line, there is no auxiliary data. Hence, the algorithm is overloaded on a true-optional parameter.

Invocations of the target are trampolined such that each invocation is invoked with CHAOS_PP_NEXT(s), where s is the original recursion state at the beginning of the algorithm (i.e. CHAOS_PP_STATE()). This maximizes the utility of the recursion backend.

This macro uses sequential iteration. As a result, the efficiency of the algorithm's use of the recursion backend, described in terms of the number of recursion steps required to complete the algorithm and ignoring the complexity of macro, is a constant Θ(1). As such, the algorithm is capable of processing up to CHAOS_PP_LIMIT_MAG + 1 elements.

This macro deduces the current recursion state and invokes CHAOS_PP_SEQ_FOR_EACH_I_S. If the current recursion state is already known, direct use of CHAOS_PP_SEQ_FOR_EACH_I_S is more efficient.

This macro may not be used if bypass semantics are in effect.

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/lambda/ops.h>
#include <chaos/preprocessor/recursion/expr.h>
#include <chaos/preprocessor/seq/for_each_i.h>

#define MACRO(s, i, elem, _) elem[i]

CHAOS_PP_EXPR(CHAOS_PP_SEQ_FOR_EACH_I(
    MACRO, (x)(y)(z), ~
))
// x[0] y[1] z[2]

#if CHAOS_PP_VARIADICS

#define BIN(s, i, x, y) (x[i] + y)

CHAOS_PP_EXPR(CHAOS_PP_SEQ_FOR_EACH_I(
    BIN, (a, b)(c, d)(e, f)
))
// (a[0] + b) (c[1] + d) (e[2] + f)

CHAOS_PP_EXPR(CHAOS_PP_SEQ_FOR_EACH_I(
    CHAOS_PP_ARG(2) CHAOS_PP_LAMBDA([) CHAOS_PP_ARG(1) ],
    (x)(y)(z)
))
// x[0] y[1] z[2]

#endif

CHAOS_PP_SEQ_FOR_EACH_I_ID

The CHAOS_PP_SEQ_FOR_EACH_I_ID macro indirectly expands to CHAOS_PP_SEQ_FOR_EACH_I.

Usage 

CHAOS_PP_SEQ_FOR_EACH_I_ID()

Remarks

This macro effectively allows CHAOS_PP_SEQ_FOR_EACH_I to pass through itself without becoming disabled.

CHAOS_PP_SEQ_FOR_EACH_I_

The CHAOS_PP_SEQ_FOR_EACH_I_ macro expands to a λ expression representing CHAOS_PP_SEQ_FOR_EACH_I.

Usage - C99 Specific 

CHAOS_PP_SEQ_FOR_EACH_I_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_SEQ_FOR_EACH_I_ID)()
