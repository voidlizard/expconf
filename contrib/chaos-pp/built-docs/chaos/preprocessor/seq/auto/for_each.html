The seq/auto/for_each.h header defines an algorithm that repeats a user-specified target for each element in a sequence.

Usage

#include <chaos/preprocessor/seq/auto/for_each.h>

Contents

  • CHAOS_PP_SEQ_AUTO_FOR_EACH

CHAOS_PP_SEQ_AUTO_FOR_EACH

The CHAOS_PP_SEQ_AUTO_FOR_EACH macro repeats the target specified by its first argument for each element in its second argument.

Usage - C99 Specific 

CHAOS_PP_SEQ_AUTO_FOR_EACH(macro, seq)
macro

The target that is to be repeated for each element in seq. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, elem†)

† If seq contains elements that have varying arities, elem mus be .... Otherwise, elem must be the number of parameters that corresponds to the arity of each element in seq.

[seq]

The sequence for which macro is to be repeated for each element.

Usage - C99 Specific 

CHAOS_PP_SEQ_AUTO_FOR_EACH(macro, seq, ...)
macro

The target that is to be repeated for each element in seq. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, elem†, data‡)

† The elem pseudo-parameter must be the number of parameters that corresponds to the arity of each element in seq.

‡ The data pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the auxiliary data, __VA_ARGS__.

[seq]

The sequence for which macro is to be repeated for each element.

[...]

Auxiliary data to be passed through the algorithm to the target.

Remarks

The CHAOS_PP_SEQ_AUTO_FOR_EACH macro is a higher-order thunk that deduces the current higher-order entry point and invokes CHAOS_PP_SEQ_FOR_EACH.

This macro may not be used if bypass semantics are in effect.

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/lambda/ops.h>
#include <chaos/preprocessor/seq/auto/for_each.h>

#if CHAOS_PP_VARIADICS

CHAOS_PP_SEQ_AUTO_FOR_EACH(
    CHAOS_PP_LAMBDA({) CHAOS_PP_ARG(1) CHAOS_PP_LAMBDA(}),
    (a)(b)(c)(d)(e)
)
// { a } { b } { c } { d } { e }

#endif

CHAOS_PP_SEQ_AUTO_FOR_EACH_ID

The CHAOS_PP_SEQ_AUTO_FOR_EACH_ID macro indirectly expands to CHAOS_PP_SEQ_AUTO_FOR_EACH.

Usage - C99 Specific 

CHAOS_PP_SEQ_AUTO_FOR_EACH_ID()

Remarks

This macro effectively allows CHAOS_PP_SEQ_AUTO_FOR_EACH to be passed around without expansion.

CHAOS_PP_SEQ_AUTO_FOR_EACH_

The CHAOS_PP_SEQ_AUTO_FOR_EACH_ macro expands to a λ expression representing CHAOS_PP_SEQ_AUTO_FOR_EACH.

Usage - C99 Specific 

CHAOS_PP_SEQ_AUTO_FOR_EACH_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_SEQ_AUTO_FOR_EACH_ID)()
