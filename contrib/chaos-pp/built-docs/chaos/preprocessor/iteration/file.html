CHAOS_PP_IS_ITERATING

The CHAOS_PP_IS_ITERATING macro is a flag used to determine if a file-iteration is in progress.

Usage 

CHAOS_PP_IS_ITERATING

Remarks

If a file-iteration is in progress, this macro expands to 1. Otherwise, this macro is not defined.

See Also

  • CHAOS_PP_ITERATION_DEPTH

CHAOS_PP_ITERATION_PARAMS

The CHAOS_PP_ITERATION_PARAMS macro is a user-defined named external argument that specifies the parameters of a file-iteration.

Usage 

#define CHAOS_PP_ITERATION_PARAMS (lower)(upper)(file)
lower

The lower bound of the file-iteration. Valid values range from 0 to CHAOS_PP_LIMIT_ITERATION.

upper

The upper bound of the file-iteration. Valid values range from 0 to CHAOS_PP_LIMIT_ITERATION.

file

The quoted path of the file to be iterated.

Usage 

#define CHAOS_PP_ITERATION_PARAMS (lower)(upper)(file)(flags)
lower

The lower bound of the file-iteration. Valid values range from 0 to CHAOS_PP_LIMIT_ITERATION.

upper

The upper bound of the file-iteration. Valid values range from 0 to CHAOS_PP_LIMIT_ITERATION.

file

The quoted path of the file to be iterated.

flags

Auxiliary flags used primarily to distinguish sibling iterations of the same file.

Remarks

This macro is defined by the user to specify the parameters of a file-iteration. At the start of a file-iteration, these parameters are evaluated and stored, and the macro is undefined by the library for subsequent (or nested) reuse.

The lower, upper, and flags parameters are all fully evaluated. This implies that they may contain arithmetic, logical, and bitwise expressions. 

#define CHAOS_PP_ITERATION_PARAMS (S)(S + 10)("file.h")

Note that if upper is less than lower, the file-iteration mechanism will iterate in reverse.

The file path specified by file must be in the q-char-sequence form because the file-iteration mechanism must store this value in __FILE__ with the #line directive so that it can undefine the CHAOS_PP_ITERATION_PARAMS macro. Fortunately, file paths specified in the q-char-sequence form that are not found are searched for as if was they were written in the h-char-sequence form.

See Also

  • CHAOS_PP_ITERATE

CHAOS_PP_ITERATE

The CHAOS_PP_ITERATE macro is used to initiate a file-iteration.

Usage 

??=include CHAOS_PP_ITERATE()

Remarks

The CHAOS_PP_ITERATION_PARAMS macro must be defined prior to initiating a file-iteration.

See Also

  • CHAOS_PP_ITERATION_PARAMS

Sample Code

// file.h
#if !CHAOS_PP_IS_ITERATING

#ifndef FILE_H
#define FILE_H

#include <chaos/preprocessor/facilities/identity.h>
#include <chaos/preprocessor/iteration/file.h>
#include <chaos/preprocessor/repetition/delineate_params.h>
#include <chaos/preprocessor/repetition/enum_params.h>

#define CHAOS_PP_ITERATION_PARAMS (1)(10)("file.h")
??=include CHAOS_PP_ITERATE()

#endif // FILE_H

#else

#define N CHAOS_PP_ITERATION()

template<CHAOS_PP_ENUM_PARAMS(N, class T)>
T0 add(CHAOS_PP_ENUM_PARAMS(N, (T, p))) {
    return CHAOS_PP_DELINEATE_PARAMS(N, CHAOS_PP_IDENTITY(+), p);
}

#undef N

#endif

Usage 

CHAOS_PP_ITERATION_DEPTH

Remarks

CHAOS_PP_RELATIVE_ITERATION

The CHAOS_PP_RELATIVE_ITERATION macro expands to the iteration value of a file-iteration dimension relative to the current dimension.

Usage 

CHAOS_PP_RELATIVE_ITERATION(i)
i

The offset to the dimension whose iteration value is to be retrieved. Valid values range from 0 to CHAOS_PP_ITERATION_DEPTH - 1.

Remarks

The i argument is interpreted as the dimensional offset backwards. Hence, CHAOS_PP_RELATIVE_ITERATION(0) is equivalent to CHAOS_PP_ITERATION(), CHAOS_PP_RELATIVE_ITERATION(1) is equivalent to CHAOS_PP_FRAME_ITERATION(CHAOS_PP_DEC(CHAOS_PP_ITERATION_DEPTH)) (i.e. the iteration value of the next outer iteration), and so on.

This macro may only be used if a file-iteration is in progress.
