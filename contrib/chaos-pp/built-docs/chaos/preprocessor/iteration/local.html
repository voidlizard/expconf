The iteration/local.h header defines the interface to the local-iteration mechanism.

Usage

#include <chaos/preprocessor/iteration/local.h>

Contents

  • CHAOS_PP_LOCAL_ITERATE
  • CHAOS_PP_LOCAL_LIMITS
  • CHAOS_PP_LOCAL_MACRO

CHAOS_PP_LOCAL_LIMITS

The CHAOS_PP_LOCAL_LIMITS macro is a user-defined named external argument that specifies the lower and upper bounds of a local-iteration.

Usage 

#define CHAOS_PP_LOCAL_LIMITS (lower, upper)
lower

The lower bound of the local-iteration. Valid values range from 0 to CHAOS_PP_LIMIT_ITERATION.

upper

The upper bound of the local-iteration. Valid values range from 0 to CHAOS_PP_LIMIT_ITERATION.

Remarks

This macro is defined by the user to specify the boundaries of a local-iteration. It is automatically undefined by the library.

Both lower and upper parameters are fully evaluated. This implies that they may contain arithmetic, logical, and bitwise expressions. 

#define CHAOS_PP_LOCAL_LIMITS (S, S + 10)

Note that if upper is less than lower, the local-iteration mechanism will iterate in reverse.

CHAOS_PP_LOCAL_MACRO

The CHAOS_PP_LOCAL_MACRO macro is a user-defined named external argument that specifies the iterated macro of a local-iteration.

Usage 

#define CHAOS_PP_LOCAL_MACRO(n) rep
n

The current iteration value.

[rep]

The replacement list to be repeated.

Remarks

This macro is defined by the user to specify the iterated macro of a local-iteration. It is automatically undefined by the library.

This macro is repeatedly invoked by the local-iteration mechanism with values in the range specified by CHAOS_PP_LOCAL_LIMITS.

CHAOS_PP_LOCAL_ITERATE

The CHAOS_PP_LOCAL_ITERATE macro is used to initiate a local-iteration.

Usage 

??=include CHAOS_PP_LOCAL_ITERATE()

Remarks

Both the CHAOS_PP_LOCAL_LIMITS macro and the CHAOS_PP_LOCAL_MACRO macro must be defined prior to initiating a local-iteration.

See Also

  • CHAOS_PP_LOCAL_LIMITS
  • CHAOS_PP_LOCAL_MACRO

Sample Code

#include <iostream>

#include <chaos/preprocessor/facilities/identity.h>
#include <chaos/preprocessor/facilities/intercept.h>
#include <chaos/preprocessor/iteration/local.h>
#include <chaos/preprocessor/repetition/enum_params.h>

template<class> char arity(void);

#define ARITY(id) (sizeof(arity<id>()) - 1)

#define CHAOS_PP_LOCAL_LIMITS (0, 5)
#define CHAOS_PP_LOCAL_MACRO(n) \
    template< \
        template< \
            CHAOS_PP_ENUM_PARAMS(n, class CHAOS_PP_INTERCEPT) \
        > class \
    > char (& arity(void))[n + 1]; \
    /**/

??=include CHAOS_PP_LOCAL_ITERATE()

class X { };
template<class> class Y { };
template<class, class> class Z { };

int main(int argc, char* argv[]) {
    std::cout
        << ARITY(X) << ' '
        << ARITY(Y) << ' '
        << ARITY(Z) << '\n';
    return 0;
}

CHAOS_PP_LOCAL_ITERATE_ID

The CHAOS_PP_LOCAL_ITERATE_ID macro indirectly expands to CHAOS_PP_LOCAL_ITERATE.

Usage 

CHAOS_PP_LOCAL_ITERATE_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_LOCAL_ITERATE_

The CHAOS_PP_LOCAL_ITERATE_ macro expands to a λ expression representing CHAOS_PP_LOCAL_ITERATE.

Usage - C99 Specific 

CHAOS_PP_LOCAL_ITERATE_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_LOCAL_ITERATE)
