The algorithm/x/fold_right_aux.h header defines an exponential algorithm that accumulates the elements of a generic data structure from the last element to the first element which can be parametrically resumed.

Usage

#include <chaos/preprocessor/algorithm/x/fold_right_aux.h>

Contents

CHAOS_PP_FOLD_RIGHT_AUX_X

The CHAOS_PP_FOLD_RIGHT_AUX_X macro accumulates the elements of its third argument with the operation specified by its second argument.

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_FOLD_RIGHT_AUX_X(size, op, g, data, ...))
size

The size of the buffer of valid recursion steps maintained by the algorithm.

op

The operation that is to be used to accumulate the elements of g. This operation must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

op(s, elem1, data2, state3)

1 If the type of g is non-variadic, elem must be the number of parameters that corresponds to the number of sub-elements in each element of g. Otherwise, elem must be a single parameter.

2 If the auxiliary data is parenthetic, the data pseudo-parameter must be the number of parameters that corresponds to the number of sub-elements in the auxiliary data. Otherwise, it must be a single parameter.

3 The state pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the initial state, __VA_ARGS__.

g

The generic data structure whose elements are to be accumulated.

[data]

Auxiliary data to be passed through the algorithm to the operation. This argument may be either a parenthetic expression of any arity or a non-parenthetic expression.

[...]

The initial state of the accumulation.

Usage - C/C++ Specific 

CHAOS_PP_EXPR(CHAOS_PP_FOLD_RIGHT_AUX_X(size, op, g, data, state))

op

op(s, elem, data, state)
g

The generic data structure whose elements are to be accumulated.

data

Auxiliary data to be passed through the algorithm to the operation. This argument may be either a unary parenthetic expression or a non-parenthetic expression.

state

The initial state of the accumulation.

Remarks

This macro behaves identically to CHAOS_PP_FOLD_RIGHT_AUX except that the invocations of the operation are not trampolined, the algorithm makes exponential use of the recursion backend, and the algorithm itself yields a trampolined deferred expression whenever the number of recursion steps is exhausted. This allows for parametric resumption.

This algorithm maintains a buffer of size recursion steps available at all times. This guarantees that the operation has at least size recursion steps available whenever it is invoked.

The exponential structure of the X-suffixed algorithms yields a total of α recursion steps, where p is the number of parametric resumptions plus the initial entry, sω is CHAOS_PP_LIMIT_EXPR, s is the current recursion state at the beginning of the algorithm (i.e. CHAOS_PP_STATE()), sδ is the buffer size, and z is the exponential base. Currently, z is fixed at two in the CHAOS_PP_FOLD_RIGHT_X algorithm (though it could easily be increased), which reduces the equation to β. Ignoring the complexity of op (which is invoked once for each element in g), the efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The aysmptotic upper and lower bounds are Θ(2n+1), where n is the number of elements in g. These bounds preserve all low-order terms, which implies that the algorithm can process exactly γ elements.

This macro deduces the current recursion state and invokes CHAOS_PP_FOLD_RIGHT_AUX_X_S. If the current recursion state is already known, direct use of CHAOS_PP_FOLD_RIGHT_AUX_X_S is more efficient.

This macro may not be used if bypass semantics are in effect.
