The algorithm/x/for_each_product.h header defines an exponential algorithm that repeats a user-specified target for each Cartesian product of elements in a sequence of generic data structures which can be parametrically resumed.

Usage

#include <chaos/preprocessor/algorithm/x/for_each_product.h>

Contents

  • CHAOS_PP_FOR_EACH_PRODUCT_X

CHAOS_PP_FOR_EACH_PRODUCT_X

The CHAOS_PP_FOR_EACH_PRODUCT_X macro repeats the target specified by its second argument for each Cartesian product of elements in its third argument.

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_FOR_EACH_PRODUCT_X(size, macro, seq))
macro

The target that is to be repeated for each Cartesian product in seq. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, product)
[seq]

The sequence of generic data structures from which Cartesian products are to be generated.

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_FOR_EACH_PRODUCT_X(size, macro, seq, ...))
macro

The target that is to be repeated for each Cartesian product in seq. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, product, data†)

† The data pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the auxiliary data, __VA_ARGS__.

[seq]

The sequence of generic data structures from which Cartesian products are to be generated.

[...]

Auxiliary data to be passed through the algorithm to the target.

Usage - C/C++ Specific 

CHAOS_PP_EXPR(CHAOS_PP_FOR_EACH_PRODUCT_X(size, macro, seq, data))
macro

The target that is to be repeated for each Cartesian product in seq. This target must be a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, product, data)
seq

The sequence of generic data structures from which Cartesian products are to be generated.

data

Auxiliary data to be passed through the algorithm to the target.

Remarks

This macro behaves identically to CHAOS_PP_FOR_EACH_PRODUCT except that the invocations of the target are not trampolined, the algorithm makes exponential use of the recursion backend, and the algorithm itself yields a trampolined deferred expression whenever the number of recursion steps is exhausted. This allows for parametric resumption.

This algorithm maintains a buffer of size recursion steps available at all times. This guarantees that the target has at least size recursion steps available whenever it is invoked.

This macro deduces the current recursion state and invokes CHAOS_PP_FOR_EACH_PRODUCT_X_S. If the current recursion state is already known, direct use of CHAOS_PP_FOR_EACH_PRODUCT_X_S is more efficient.

This macro may not be used if bypass semantics are in effect.

CHAOS_PP_FOR_EACH_PRODUCT_X_ID

The CHAOS_PP_FOR_EACH_PRODUCT_X_ID macro indirectly expands to CHAOS_PP_FOR_EACH_PRODUCT_X.

Usage 

CHAOS_PP_FOR_EACH_PRODUCT_X_ID()

Remarks

This macro effectively allows CHAOS_PP_FOR_EACH_PRODUCT_X to pass through itself without becoming disabled.
