The algorithm/for_each_product.h header defines an algorithm that repeats a user-specified target for each Cartesian product of elements in a sequence of generic data structures.

Usage

#include <chaos/preprocessor/algorithm/for_each_product.h>

Contents

  • CHAOS_PP_FOR_EACH_PRODUCT

CHAOS_PP_FOR_EACH_PRODUCT

The CHAOS_PP_FOR_EACH_PRODUCT macro repeats the target specified by its first argument for each Cartesian product of elements in its second argument.

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_FOR_EACH_PRODUCT(macro, seq))
macro

The target that is to be repeated for each Cartesian product in seq. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, product)
[seq]

The sequence of generic data structures from which Cartesian products are to be generated.

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_FOR_EACH_PRODUCT(macro, seq, ...))
macro

The target that is to be repeated for each Cartesian product in seq. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, product, data†)

† The data pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the auxiliary data, __VA_ARGS__.

[seq]

The sequence of generic data structures from which Cartesian products are to be generated.

[...]

Auxiliary data to be passed through the algorithm to the target.

Usage - C/C++ Specific 

CHAOS_PP_EXPR(CHAOS_PP_FOR_EACH_PRODUCT(macro, seq, data))
macro

The target that is to be repeated for each Cartesian product in seq. This target must be a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, product, data)
seq

The sequence of generic data structures from which Cartesian products are to be generated.

data

Auxiliary data to be passed through the algorithm to the target.

Remarks

The CHAOS_PP_FOR_EACH_PRODUCT macro itself expands to a deferred expression in terms of CHAOS_PP_NEXT(s). This deferred expression, when invoked by CHAOS_PP_EXPR, expands to the result of the repeated invocation of the target for each Cartesian product in seq.

The target is invoked with the current recursion state (s), a sequence of elements representing a Cartesian product (product), and, if it exists, the auxiliary data (data or __VA_ARGS__). This target may expand to a deferred expression in terms of CHAOS_PP_NEXT(s). This allows the target to recurse on itself if necessary.

In C99, it is possible to omit the auxiliary data argument altogether. This is different than passing "nothing" as the auxiliary data. 

#define A (CHAOS_PP_TUPLE) (a, b, c)
#define B (CHAOS_PP_TUPLE) (x, y, z)

CHAOS_PP_EXPR(
    CHAOS_PP_FOR_EACH_PRODUCT(MACRO, (A)(B),) // #1
    CHAOS_PP_FOR_EACH_PRODUCT(MACRO, (A)(B))  // #2
)

The first line passes "nothing" as the auxiliary data, which causes MACRO to be invoked as (e.g.)... 

MACRO(s, (a)(x),)

The second line omits the auxiliary data, which causes MACRO to be invoked as (e.g.)... 

MACRO(s, (a)(x))

The fundamental difference is that, in the first line, the auxiliary data is empty (which might have a valid meaning as data), but, in the second line, there is no auxiliary data. Hence, the algorithm is overloaded on a true-optional parameter.

Invocations of the target are trampolined such that each invocation is invoked with CHAOS_PP_NEXT(s), where s is the original recursion state at the beginning of the algorithm (i.e. CHAOS_PP_STATE()). This maximizes the utility of the recursion backend.

Ignoring the complexity of macro (which is invoked once for each Cartesian product), the efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Θα, where n is the number of elements in seq and nk is the number of elements in the k-th element of seq. (This formula can also be expressed functionally as Θ(f(n)+1), where f is F, which might be easier to conceptualize.)

Ignoring macro once again, the efficiency of the algorithm's use of the recursion backend can also be described in terms of the total depth of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Θβ, where n is the number of elements in seq and nk is the number of elements in the k-th element of seq.

This macro deduces the current recursion state and invokes CHAOS_PP_FOR_EACH_PRODUCT_S. If the current recursion state is already known, direct use of CHAOS_PP_FOR_EACH_PRODUCT_S is more efficient.

This macro may not be used if bypass semantics are in effect.

Sample Code

#include <chaos/preprocessor/algorithm/for_each_product.h>
#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/lambda/ops.h>
#include <chaos/preprocessor/list/core.h>
#include <chaos/preprocessor/seq/core.h>
#include <chaos/preprocessor/seq/to_tuple.h>
#include <chaos/preprocessor/string/core.h>

#define MACRO(s, product, data) \
    CHAOS_PP_SEQ_TO_TUPLE(product) \
    /**/

#define A (CHAOS_PP_LIST) (a, (b, ...))
#define B (CHAOS_PP_LIST) (x, (y, ...))

CHAOS_PP_EXPR(CHAOS_PP_FOR_EACH_PRODUCT(
    MACRO, (A)(B), ~
))
// (a, x) (a, y) (b, x) (b, y)

#if CHAOS_PP_VARIADICS

CHAOS_PP_EXPR(CHAOS_PP_FOR_EACH_PRODUCT(
    CHAOS_PP_SEQ_TO_TUPLE_(CHAOS_PP_ARG(1)),
    (A)(B)
))
// (a, x) (a, y) (b, x) (b, y)

#endif

#define X (CHAOS_PP_STRING) a b c
#define Y (CHAOS_PP_SEQ) (+)(-)
#define Z (CHAOS_PP_LIST) (?, ...)

CHAOS_PP_EXPR(CHAOS_PP_FOR_EACH_PRODUCT(
    MACRO, (X)(Y)(Z), ~
))
// (a, +, ?) (a, -, ?)
// (b, +, ?) (b, -, ?)
// (c, +, ?) (c, -, ?)

CHAOS_PP_FOR_EACH_PRODUCT_ID

The CHAOS_PP_FOR_EACH_PRODUCT_ID macro indirectly expands to CHAOS_PP_FOR_EACH_PRODUCT.

Usage 

CHAOS_PP_FOR_EACH_PRODUCT_ID()

Remarks

This macro effectively allows CHAOS_PP_FOR_EACH_PRODUCT to pass through itself without becoming disabled.

CHAOS_PP_FOR_EACH_PRODUCT_

The CHAOS_PP_FOR_EACH_PRODUCT_ macro expands to a λ expression representing CHAOS_PP_FOR_EACH_PRODUCT.

Usage - C99 Specific 

CHAOS_PP_FOR_EACH_PRODUCT_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_FOR_EACH_PRODUCT_ID)()
