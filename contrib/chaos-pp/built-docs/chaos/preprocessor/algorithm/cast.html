The algorithm/cast.h header defines an algorithm that converts a generic data structure to another type.

Usage

#include <chaos/preprocessor/algorithm/cast.h>

Contents

  • CHAOS_PP_CAST

CHAOS_PP_CAST

The CHAOS_PP_CAST macro converts its second argument to the type specified by its first argument.

Usage 

CHAOS_PP_CAST(type, g)
type

The type to which g is to be converted.

g

The generic data structure to be converted.

Remarks

This macro expands to a generic data structure of type type containing the elements of g. If g is already of type type, this macro expands to g without performing any conversion.

This macro assumes that g can be converted to type type. The algorithm does not attempt to verify that the resulting data structure is actually a valid data structure of the specified type. For example, the following produces an invalid string: 

CHAOS_PP_CAST(
    CHAOS_PP_STRING, (CHAOS_PP_SEQ) (+)(-)(*)(/)(%)
)
// invalid

The efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper bound of O(n+1), where n is the number of elements in g, occurs when g is not of type type. The asymptotic lower bound of Ω(0) occurs when g is already of type type.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_CAST_BYPASS must be used instead.

Sample Code

#include <chaos/preprocessor/algorithm/cast.h>
#include <chaos/preprocessor/list/core.h>
#include <chaos/preprocessor/string/core.h>

CHAOS_PP_CAST(CHAOS_PP_LIST, (CHAOS_PP_SEQ) (C)(H)(A)(O)(S))
// (CHAOS_PP_LIST) (C, (H, (A, (O, (S, ...)))))

CHAOS_PP_CAST_ID

The CHAOS_PP_CAST_ID macro indirectly expands to CHAOS_PP_CAST.

Usage 

CHAOS_PP_CAST_ID()

Remarks

This macro effectively allows CHAOS_PP_CAST to pass through itself without becoming disabled.

CHAOS_PP_CAST_

The CHAOS_PP_CAST_ macro expands to a λ expression representing CHAOS_PP_CAST.

Usage - C99 Specific 

CHAOS_PP_CAST_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_CAST_ID)()

CHAOS_PP_CAST_BYPASS

The CHAOS_PP_CAST_BYPASS macro behaves identically to CHAOS_PP_CAST except that it is parametized by the bypass recursion state.

Usage 

CHAOS_PP_CAST_BYPASS(s, type, g)
s

The current bypass recursion state.

type

The type to which g is to be converted.

g

The generic data structure to be converted.

Remarks

If bypass semantics are already in effect, this macro must be used instead of CHAOS_PP_CAST.

CHAOS_PP_CAST_BYPASS_ID

The CHAOS_PP_CAST_BYPASS_ID macro indirectly expands to CHAOS_PP_CAST_BYPASS.

Usage 

CHAOS_PP_CAST_BYPASS_ID()

Remarks

This macro effectively allows CHAOS_PP_CAST_BYPASS to pass through itself without becoming disabled.

CHAOS_PP_CAST_BYPASS_

The CHAOS_PP_CAST_BYPASS_ macro expands to a λ expression representing CHAOS_PP_CAST_BYPASS.

Usage - C99 Specific 

CHAOS_PP_CAST_BYPASS_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_CAST_BYPASS_ID)()
