The algorithm/concat.h header defines an algorithm that concatenates the elements of a generic data structure.

Usage

#include <chaos/preprocessor/algorithm/concat.h>

Contents

  • CHAOS_PP_CONCAT

CHAOS_PP_CONCAT

The CHAOS_PP_CONCAT macro concatenates the elements of its argument.

Usage 

CHAOS_PP_CONCAT(g)
g

The generic data structure whose elements are to be concatenated.

Remarks

This macro performs an ordered concatenation of the elements of g beginning with the first element. This is illustrated by the following pseudo-code: 

( ... ((g[0] ## g[1]) ## g[2]) ... ## g[length(g) - 1])

In C99, if g contains non-unary elements, the sub-elements of each element will also be concatenated. For example, 

CHAOS_PP_CONCAT((CHAOS_PP_SEQ) (a, b)(c, d)) // abcd

The efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Θ(n+1), where n is the number of elements in g.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_CONCAT_BYPASS must be used instead.

Sample Code

#include <chaos/preprocessor/algorithm/concat.h>
#include <chaos/preprocessor/list/core.h>

CHAOS_PP_CONCAT(
    (CHAOS_PP_LIST) (C, (H, (A, (O, (S, ...)))))
)
// CHAOS

CHAOS_PP_CONCAT_ID

The CHAOS_PP_CONCAT_ID macro indirectly expands to CHAOS_PP_CONCAT.

Usage 

CHAOS_PP_CONCAT_ID()

Remarks

This macro effectively allows CHAOS_PP_CONCAT to pass through itself without becoming disabled.

CHAOS_PP_CONCAT_

The CHAOS_PP_CONCAT_ macro expands to a λ expression representing CHAOS_PP_CONCAT.

Usage - C99 Specific 

CHAOS_PP_CONCAT_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_CONCAT_ID)()

CHAOS_PP_CONCAT_BYPASS

The CHAOS_PP_CONCAT_BYPASS macro behaves identically to CHAOS_PP_CONCAT except that it is parametized by the bypass recursion state.

Usage 

CHAOS_PP_CONCAT_BYPASS(s, g)
s

The current bypass recursion state.

g

The generic data structure whose elements are to be concatenated.

Remarks

If bypass semantics are already in effect, this macro must be used instead of CHAOS_PP_CONCAT.
