The algorithm/remove.h header defines an algorithm that removes an element from a generic data structure.

Usage

#include <chaos/preprocessor/algorithm/remove.h>

Contents

  • CHAOS_PP_REMOVE

CHAOS_PP_REMOVE

The CHAOS_PP_REMOVE macro removes the element at the index specified by its first argument from its second argument.

Usage 

CHAOS_PP_REMOVE(i, g)
i

The zero-based index of the element to be removed. Valid values range from 0 to CHAOS_PP_SIZE(g) - 1.

g

The non-nil generic data structure from which an element is to be removed.

Remarks

This macro expands to a generic data structure of the same type as g containing all of the elements of g except the i-th element.

The efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Θ(i+1), where i is i.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_REMOVE_BYPASS must be used instead.

Sample Code

#include <chaos/preprocessor/algorithm/remove.h>
#include <chaos/preprocessor/array/core.h>

CHAOS_PP_REMOVE(2, (CHAOS_PP_ARRAY) (5, (0, 1, 2, 3, 4)))
// (CHAOS_PP_ARRAY) (4, (0, 1, 3, 4))

CHAOS_PP_REMOVE_ID

The CHAOS_PP_REMOVE_ID macro indirectly expands to CHAOS_PP_REMOVE.

Usage 

CHAOS_PP_REMOVE_ID()

Remarks

This macro effectively allows CHAOS_PP_REMOVE to pass through itself without becoming disabled.

CHAOS_PP_REMOVE_

The CHAOS_PP_REMOVE_ macro expands to a λ expression representing CHAOS_PP_REMOVE.

Usage - C99 Specific 

CHAOS_PP_REMOVE_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_REMOVE_ID)()

CHAOS_PP_REMOVE_BYPASS

The CHAOS_PP_REMOVE_BYPASS macro behaves identically to CHAOS_PP_REMOVE except that it is parametized by the bypass recursion state.

Usage 

CHAOS_PP_REMOVE_BYPASS(s, i, g)
s

The current bypass recursion state.

i

The zero-based index of the element to be removed. Valid values range from 0 to CHAOS_PP_SIZE(g) - 1.

g

The non-nil generic data structure from which an element is to be removed.

Remarks

If bypass semantics are already in effect, this macro must be used instead of CHAOS_PP_REMOVE.

CHAOS_PP_REMOVE_BYPASS_ID

The CHAOS_PP_REMOVE_BYPASS_ID macro indirectly expands to CHAOS_PP_REMOVE_BYPASS.

Usage 

CHAOS_PP_REMOVE_BYPASS_ID()

Remarks

This macro effectively allows CHAOS_PP_REMOVE_BYPASS to pass through itself without becoming disabled.

CHAOS_PP_REMOVE_BYPASS_

The CHAOS_PP_REMOVE_BYPASS_ macro expands to a λ expression representing CHAOS_PP_REMOVE_BYPASS.

Usage - C99 Specific 

CHAOS_PP_REMOVE_BYPASS_

Remarks
