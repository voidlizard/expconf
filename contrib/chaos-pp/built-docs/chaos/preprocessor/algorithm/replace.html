The algorithm/replace.h header defines an algorithm that replaces an element in a generic data structure.

Usage

#include <chaos/preprocessor/algorithm/replace.h>

Contents

  • CHAOS_PP_REPLACE

CHAOS_PP_REPLACE

The CHAOS_PP_REPLACE macro replaces the element at the index specified by its first argument in its second argument with the element specified by its third argument.

Usage - C99 Specific 

CHAOS_PP_REPLACE(i, g, ...)
i

The zero-based index of the element to be replaced. Valid values range from 0 to CHAOS_PP_SIZE(g) - 1.

g

The non-nil generic data structure in which an element is to be replaced.

[...]

The element used to replace the element at index i in g.

Usage - C/C++ Specific 

CHAOS_PP_REPLACE(i, g, x)
i

The zero-based index of the element to be replaced. Valid values range from 0 to CHAOS_PP_SIZE(g) - 1.

g

The non-nil generic data structure in which an element is to be replaced.

x

The element used to replace the element at index i in g.

Remarks

This macro expands to a generic data structure of the same type as g containing the elements of g except with the i-th element replaced by x (or __VA_ARGS__).

The efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Θ(i+1), where i is i.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_REPLACE_BYPASS must be used instead.

See Also

  • CHAOS_PP_SEQ_REPLACE
  • CHAOS_PP_TUPLE_REPLACE

Sample Code

#include <chaos/preprocessor/algorithm/replace.h>
#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/tuple/core.h>

#if CHAOS_PP_VARIADICS

CHAOS_PP_REPLACE(2, (CHAOS_PP_TUPLE) (a, b, c, d), !)
// (CHAOS_PP_TUPLE) (a, b, !, d)

#endif

CHAOS_PP_REPLACE_ID

The CHAOS_PP_REPLACE_ID macro indirectly expands to CHAOS_PP_REPLACE.

Usage 

CHAOS_PP_REPLACE_ID()

Remarks

This macro effectively allows CHAOS_PP_REPLACE to pass through itself without becoming disabled.

CHAOS_PP_REPLACE_

The CHAOS_PP_REPLACE_ macro expands to a λ expression representing CHAOS_PP_REPLACE.

Usage - C99 Specific 

CHAOS_PP_REPLACE_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_REPLACE_ID)()

CHAOS_PP_REPLACE_BYPASS

The CHAOS_PP_REPLACE_BYPASS macro behaves identically to CHAOS_PP_REPLACE except that it is parametized by the bypass recursion state.

Usage - C99 Specific 

CHAOS_PP_REPLACE_BYPASS(s, i, g, ...)
s

The current bypass recursion state.

i

The zero-based index of the element to be replaced. Valid values range from 0 to CHAOS_PP_SIZE(g) - 1.

g

The non-nil generic data structure in which an element is to be replaced.

[...]

The element used to replace the element at index i in g.

Usage - C/C++ Specific 

CHAOS_PP_REPLACE_BYPASS(s, i, g, x)
s

The current bypass recursion state.

i

The zero-based index of the element to be replaced. Valid values range from 0 to CHAOS_PP_SIZE(g) - 1.

g

The non-nil generic data structure in which an element is to be replaced.

x

The element used to replace the element at index i in g.

Remarks

If bypass semantics are already in effect, this macro must be used instead of CHAOS_PP_REPLACE.

CHAOS_PP_REPLACE_BYPASS_ID

The CHAOS_PP_REPLACE_BYPASS_ID macro indirectly expands to CHAOS_PP_REPLACE_BYPASS.

Usage 

CHAOS_PP_REPLACE_BYPASS_ID()

Remarks

This macro effectively allows CHAOS_PP_REPLACE_BYPASS to pass through itself without becoming disabled.

CHAOS_PP_REPLACE_BYPASS_

Usage - C99 Specific 

CHAOS_PP_REPLACE_BYPASS_
