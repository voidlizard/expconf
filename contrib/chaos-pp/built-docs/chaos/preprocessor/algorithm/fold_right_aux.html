The algorithm/fold_right_aux.h header defines an algorithm that accumulates the elements of a generic data structure from the last element to the first element.

Usage

#include <chaos/preprocessor/algorithm/fold_right_aux.h>

Contents

  • CHAOS_PP_FOLD_RIGHT_AUX

CHAOS_PP_FOLD_RIGHT_AUX

The CHAOS_PP_FOLD_RIGHT_AUX macro accumulates the elements of its second argument with the operation specified by its first argument.

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_FOLD_RIGHT_AUX(op, g, data, ...))
op

The operation that is to be used to accumulate the elements of g. This operation must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

op(s, elem1, data2, state3)

1 If the type of g is non-variadic, elem must be the number of parameters that corresponds to the number of sub-elements in each element of g. Otherwise, elem must be a single parameter.

2 If the auxiliary data is parenthetic, the data pseudo-parameter must be the number of parameters that corresponds to the number of sub-elements in the auxiliary data. Otherwise, it must be a single parameter.

3 The state pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the initial state, __VA_ARGS__.

g

The generic data structure whose elements are to be accumulated.

[data]

Auxiliary data to be passed through the algorithm to the operation. This argument may be either a parenthetic expression of any arity or a non-parenthetic expression.

[...]

The initial state of the accumulation.

Usage - C/C++ Specific 

CHAOS_PP_EXPR(CHAOS_PP_FOLD_RIGHT_AUX(op, g, data, state))
op

The operation that is to be used to accumulate the elements of g. This operation must be a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

op(s, elem, data, state)
g

The generic data structure whose elements are to be accumulated.

data

Auxiliary data to be passed through the algorithm to the operation. This argument may be either a unary parenthetic expression or a non-parenthetic expression.

state

The initial state of the accumulation.

Remarks

The CHAOS_PP_FOLD_RIGHT_AUX macro itself expands to a deferred expression in terms of CHAOS_PP_NEXT(s). This deferred expression, when invoked by CHAOS_PP_EXPR, expands to the result of accumulating the elements of g, from right to left, with the user-specified operation. This is illustrated by the following pseudo-code: 

op(s, g[0], data, op(s, g[1], data, ... op(s, g[length(g) - 1], data, state) ... ))

An invocation of the operation with the current recursion state (s), a particular element (elem), the auxiliary data (data), and the current state (state or __VA_ARGS__) expands to a new state used in the accumulation of subsequent elements (or as the final result of the algorithm if no further elements exist). This operation may also expand to a deferred expression in terms of CHAOS_PP_NEXT(s) that expands to a new state. Expansion to a deferred expression allows the operation to recurse on itself if necessary.

Invocations of the operation are trampolined such that each invocation is invoked with CHAOS_PP_NEXT(CHAOS_PP_NEXT(s)), where s is the original recursion state at the beginning of the algorithm (i.e. CHAOS_PP_STATE()). This maximizes the utility of the recursion backend.

The auxiliary data, data, is treated differently depending on whether it is parenthetic. If data is non-parenthetic, it is passed directly to the operation as a single argument. In C99, if data is a parenthetic expression of any arity, its sub-elements are passed to the operation as individual arguments. In C (prior to C99) and C++, if data is a unary parenthetic expression, its sub-element is passed to the operation as a single argument (a parenthetic expression of any other arity is prohibited).

Ignoring the complexity of op (which is invoked once for each element in g), the efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Θ(n+1), where n is the number of elements in g.

This macro deduces the current recursion state and invokes CHAOS_PP_FOLD_RIGHT_AUX_S. If the current recursion state is already known, direct use of CHAOS_PP_FOLD_RIGHT_AUX_S is more efficient.

This macro may not be used if bypass semantics are in effect.

CHAOS_PP_FOLD_RIGHT_AUX_ID

The CHAOS_PP_FOLD_RIGHT_AUX_ID macro indirectly expands to CHAOS_PP_FOLD_RIGHT_AUX.

Usage 

CHAOS_PP_FOLD_RIGHT_AUX_ID()

Remarks

This macro effectively allows CHAOS_PP_FOLD_RIGHT_AUX to pass through itself without becoming disabled.

CHAOS_PP_FOLD_RIGHT_AUX_

CHAOS_PP_FOLD_RIGHT_AUX_

Remarks
