The algorithm/prepend.h header defines an algorithm that prepends one generic data structure to another.

Usage

#include <chaos/preprocessor/algorithm/prepend.h>

Contents

  • CHAOS_PP_PREPEND

CHAOS_PP_PREPEND

The CHAOS_PP_PREPEND macro prepends its first argument to its second argument.

Usage 

CHAOS_PP_PREPEND(g1, g2)
g1

The generic data structure to be prepended to g2.

g2

The generic data structure to which g1 is to be prepended.

Remarks

This macro prepends the elements in g1 to the head of g2, resulting in a generic data structure of the same type as g2.

The efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Θ(n+1), where n is the number of elements in g1.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_PREPEND_BYPASS must be used instead.

See Also

  • CHAOS_PP_APPEND
  • CHAOS_PP_PUSH_BACK

Sample Code

#include <chaos/preprocessor/algorithm/prepend.h>
#include <chaos/preprocessor/list/core.h>
#include <chaos/preprocessor/seq/core.h>

CHAOS_PP_PREPEND(
    (CHAOS_PP_SEQ) (a)(b)(c),
    (CHAOS_PP_LIST) (x, (y, (z, ...)))
)
// (CHAOS_PP_LIST) (a, (b, (c, (x, (y, (z, ...))))))

CHAOS_PP_PREPEND_ID

The CHAOS_PP_PREPEND_ID macro indirectly expands to CHAOS_PP_PREPEND.

Usage 

CHAOS_PP_PREPEND_ID()

Remarks

This macro effectively allows CHAOS_PP_PREPEND to pass through itself without becoming disabled.

CHAOS_PP_PREPEND_

The CHAOS_PP_PREPEND_ macro expands to a λ expression representing CHAOS_PP_PREPEND.

Usage - C99 Specific 

CHAOS_PP_PREPEND_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_PREPEND_ID)()

CHAOS_PP_PREPEND_BYPASS

The CHAOS_PP_PREPEND_BYPASS macro behaves identically to CHAOS_PP_PREPEND except that it is parametized by the bypass recursion state.

Usage 

CHAOS_PP_PREPEND_BYPASS(s, g1, g2)
s

The current bypass recursion state.

g1

The generic data structure to be prepended to g2.

g2

The generic data structure to which g1 is to be prepended.

Remarks

If bypass semantics are already in effect, this macro must be used instead of CHAOS_PP_PREPEND.

CHAOS_PP_PREPEND_BYPASS_ID

The CHAOS_PP_PREPEND_BYPASS_ID macro indirectly expands to CHAOS_PP_PREPEND_BYPASS.

Usage 

CHAOS_PP_PREPEND_BYPASS_ID()

Remarks

This macro effectively allows CHAOS_PP_PREPEND_BYPASS to pass through itself without becoming disabled.

CHAOS_PP_PREPEND_BYPASS_

The CHAOS_PP_PREPEND_BYPASS_ macro expands to a λ expression representing CHAOS_PP_PREPEND_BYPASS.

Usage - C99 Specific 

CHAOS_PP_PREPEND_BYPASS_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_PREPEND_BYPASS_ID)()
