The algorithm/auto/for_each_i.h header defines an algorithm that repeats a user-specified target for each element in a generic data structure.

Usage

#include <chaos/preprocessor/algorithm/auto/for_each_i.h>

Contents

  • CHAOS_PP_AUTO_FOR_EACH_I

CHAOS_PP_AUTO_FOR_EACH_I

The CHAOS_PP_AUTO_FOR_EACH_I macro repeats the target specified by its first argument for each element in its second argument.

Usage - C99 Specific 

CHAOS_PP_AUTO_FOR_EACH_I(macro, g)
macro

The target that is to be repeated for each element in g. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, i, elem†)

† If the type of g is non-variadic, elem must be ... or the number of parameters that corresponds to the number of sub-elements in each element of g. Otherwise, elem must be ... or a single parameter.

g

The generic data structure for which macro is to be repeated for each element.

Usage - C99 Specific 

CHAOS_PP_AUTO_FOR_EACH_I(macro, g, ...)
macro

The target that is to be repeated for each element in g. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, i, elem†, data‡)

† If the type of g is non-variadic, elem must be the number of parameters that corresponds to the number of sub-elements in each element of g. Otherwise, elem must be a single parameter.

‡ The data pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the auxiliary data, __VA_ARGS__.

g

The generic data structure for which macro is to be repeated for each element.

[...]

Auxiliary data to be passed through the algorithm to the target.

Remarks

The CHAOS_PP_AUTO_FOR_EACH_I macro is a higher-order thunk that deduces the current higher-order entry point and invokes CHAOS_PP_FOR_EACH.

This macro may not be used if bypass semantics are in effect.

Sample Code

#include <chaos/preprocessor/algorithm/auto/for_each_i.h>
#include <chaos/preprocessor/arithmetic/inc.h>
#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/lambda/ops.h>
#include <chaos/preprocessor/repetition/auto/enum.h>
#include <chaos/preprocessor/string/core.h>

#if CHAOS_PP_VARIADICS

#define MACRO(s, i, elem) \
    int elem[CHAOS_PP_INC(i)] = { \
        CHAOS_PP_AUTO_ENUM(CHAOS_PP_INC(i), CHAOS_PP_ARG(1)) \
    }; \
    /**/

CHAOS_PP_AUTO_FOR_EACH_I(
    MACRO,
    (CHAOS_PP_STRING) a b c
)
// int a[1] = { 0 }; int b[2] = { 0, 1 }; int c[3] = { 0, 1, 2 };

#endif

CHAOS_PP_AUTO_FOR_EACH_I_ID

The CHAOS_PP_AUTO_FOR_EACH_I_ID macro indirectly expands to CHAOS_PP_AUTO_FOR_EACH_I.

Usage - C99 Specific 

CHAOS_PP_AUTO_FOR_EACH_I_ID()

Remarks

This macro effectively allows CHAOS_PP_AUTO_FOR_EACH_I to be passed around without expansion.

CHAOS_PP_AUTO_FOR_EACH_I_

The CHAOS_PP_AUTO_FOR_EACH_I_ macro expands to a λ expression representing CHAOS_PP_AUTO_FOR_EACH_I.

Usage - C99 Specific 

CHAOS_PP_AUTO_FOR_EACH_I_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_AUTO_FOR_EACH_I_ID)()
