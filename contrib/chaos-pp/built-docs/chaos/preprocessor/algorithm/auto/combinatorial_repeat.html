The algorithm/auto/combinatorial_repeat.h header defines an algorithm that produces a combinatorial repetition of a dataset.

Usage

#include <chaos/preprocessor/algorithm/auto/combinatorial_repeat.h>

Contents

  • CHAOS_PP_AUTO_COMBINATORIAL_REPEAT

CHAOS_PP_AUTO_COMBINATORIAL_REPEAT

The CHAOS_PP_AUTO_COMBINATORIAL_REPEAT macro generates repetitions for all possible strings (in the mathematical sense) up to the length specified by its first argument of an alphabet specified by its third argument.

Usage - C99 Specific 

CHAOS_PP_AUTO_COMBINATORIAL_REPEAT(count, macro, g)
count

The maximum length of the strings to be generated. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

macro

The target that is to be repeated for each string. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, size, product)
g

The generic data structure containing the alphabet used to construct strings.

Usage - C99 Specific 

CHAOS_PP_AUTO_COMBINATORIAL_REPEAT(count, macro, g, ...)
count

The maximum length of the strings to be generated. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

macro

The target that is to be repeated for each string. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, size, product, data†)

† The data pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the auxiliary data, __VA_ARGS__.

g

The generic data structure containing the alphabet used to construct strings.

[...]

Auxiliary data to be passed through the algorithm to the target.

Remarks

The CHAOS_PP_AUTO_COMBINATORIAL_REPEAT macro is a higher-order thunk that deduces the current higher-order entry point and invokes CHAOS_PP_COMBINATORIAL_REPEAT.

This macro may not be used if bypass semantics are in effect.

CHAOS_PP_AUTO_COMBINATORIAL_REPEAT_ID

The CHAOS_PP_AUTO_COMBINATORIAL_REPEAT_ID macro indirectly expands to CHAOS_PP_AUTO_COMBINATORIAL_REPEAT.

Usage - C99 Specific 

CHAOS_PP_AUTO_COMBINATORIAL_REPEAT_ID()

Remarks

This macro effectively allows CHAOS_PP_AUTO_COMBINATORIAL_REPEAT to be passed around without expansion.

CHAOS_PP_AUTO_COMBINATORIAL_REPEAT_

The CHAOS_PP_AUTO_COMBINATORIAL_REPEAT_ macro expands to a λ expression representing CHAOS_PP_AUTO_COMBINATORIAL_REPEAT.

Usage - C99 Specific 

CHAOS_PP_AUTO_COMBINATORIAL_REPEAT_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_AUTO_COMBINATORIAL_REPEAT_ID)()
