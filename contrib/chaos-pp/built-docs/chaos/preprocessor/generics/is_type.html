The generics/is_type.h header defines a macro that determines if a generic data structure is a certain type.

Usage

#include <chaos/preprocessor/generics/is_type.h>

Contents

  • CHAOS_PP_IS_TYPE

CHAOS_PP_IS_TYPE

The CHAOS_PP_IS_TYPE macro determines if its second argument is of the type specified by its first argument.

Usage 

CHAOS_PP_IS_TYPE(type, g)
type

The type to be tested against.

g

The generic data structure to be tested.

Remarks

If g is of type type, this macro expands to 1. Otherwise, it expands to 0.

In some cases, a primitive form of inheritance exists between types with different names. One such example is the variadic sequence and sequence types. A sequence is a variadic sequence, but not vice versa: 

CHAOS_PP_IS_TYPE(
    CHAOS_PP_SEQ, (CHAOS_PP_VARIADIC_SEQ) (a)(b, c)
)
// 0

CHAOS_PP_IS_TYPE(
    CHAOS_PP_VARIADIC_SEQ, (CHAOS_PP_SEQ) (a)(b)(c)
)
// 1

Sample Code

#include <chaos/preprocessor/generics/is_type.h>
#include <chaos/preprocessor/list/core.h>
#include <chaos/preprocessor/seq/core.h>

CHAOS_PP_IS_TYPE(
    CHAOS_PP_LIST, (CHAOS_PP_SEQ) (a)(b)(c)
)
// 0

CHAOS_PP_IS_TYPE(
    CHAOS_PP_LIST, (CHAOS_PP_LIST) (a, (b, (c, ...)))
)
// 1

CHAOS_PP_IS_TYPE_ID

The CHAOS_PP_IS_TYPE_ID macro indirectly expands to CHAOS_PP_IS_TYPE.

Usage 

CHAOS_PP_IS_TYPE_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_IS_TYPE_

The CHAOS_PP_IS_TYPE_ macro expands to a λ expression representing CHAOS_PP_IS_TYPE.

Usage - C99 Specific 

CHAOS_PP_IS_TYPE_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_IS_TYPE)
