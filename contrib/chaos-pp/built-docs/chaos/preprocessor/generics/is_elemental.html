The generics/is_elemental.h header defines a macro that determines if the type of a generic data structure is elemental.

Usage

#include <chaos/preprocessor/generics/is_elemental.h>

Contents

  • CHAOS_PP_IS_ELEMENTAL

CHAOS_PP_IS_ELEMENTAL

The CHAOS_PP_IS_ELEMENTAL macro determines if the type represented by its argument is elemental.

Usage 

CHAOS_PP_IS_ELEMENTAL(type)
type

The type to be tested.

Remarks

If the type represented by type is elemental, this macro expands to 1. Otherwise, it expands to 0.

The type of a generic data structure is elemental if the elements contained therein are not variadic (i.e. the contained elements are all the same arity). Consequently, in C (prior to C99) and C++ this macro always expands to 1 because only unary elements are supported.

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/generics/is_elemental.h>
#include <chaos/preprocessor/list/core.h>
#include <chaos/preprocessor/seq/variadic/core.h>

CHAOS_PP_IS_ELEMENTAL(CHAOS_PP_LIST) // 1

#if CHAOS_PP_VARIADICS

CHAOS_PP_IS_ELEMENTAL(CHAOS_PP_VARIADIC_SEQ) // 0

#endif

CHAOS_PP_IS_ELEMENTAL_ID

The CHAOS_PP_IS_ELEMENTAL_ID macro indirectly expands to CHAOS_PP_IS_ELEMENTAL.

Usage 

CHAOS_PP_IS_ELEMENTAL_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_IS_ELEMENTAL_

The CHAOS_PP_IS_ELEMENTAL_ macro expands to a λ expression representing CHAOS_PP_IS_ELEMENTAL.

Usage - C99 Specific 

CHAOS_PP_IS_ELEMENTAL_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_IS_ELEMENTAL)
