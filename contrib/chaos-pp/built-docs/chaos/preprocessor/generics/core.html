CHAOS_PP_TAG

The CHAOS_PP_TAG macro is used to compare types.

Usage 

CHAOS_PP_TAG(type)
type

The type of the tag to be produced.

Remarks

This macro produces a tag macro name used by generic algorithms to compare types.

See Also

CHAOS_PP_CONS

The CHAOS_PP_ARRAY_CONS macro appends an element to the head of a generic data structure.

Usage - C99 Specific 

CHAOS_PP_CONS(g, ...)
g

The generic data structure to which __VA_ARGS__ is to be appended.

[...]

The element to be appended to the head of g.

Usage - C/C++ Specific 

CHAOS_PP_CONS(g, x)
g

The generic data structure to which x is to be appended.

x

The element to be appended to the head of g.

Remarks

This macro expands to a generic data structure of the same type as g containing the elements of g with x (or __VA_ARGS__) appended to the head.

See Also

  • CHAOS_PP_ARRAY_CONS
  • CHAOS_PP_LIST_CONS
  • CHAOS_PP_SEQ_CONS
  • CHAOS_PP_STRING_CONS
  • CHAOS_PP_TUPLE_CONS
  • CHAOS_PP_VARIADIC_SEQ_CONS

Sample Code

#include <chaos/preprocessor/generics/core.h>
#include <chaos/preprocessor/list/core.h>

CHAOS_PP_CONS(CHAOS_PP_NIL(CHAOS_PP_LIST), a)
// (CHAOS_PP_LIST) (a, ...)

CHAOS_PP_CONS((CHAOS_PP_LIST) (y, (z, ...)), x)
// (CHAOS_PP_LIST) (x, (y, (z, ...)))

CHAOS_PP_ITEM

The CHAOS_PP_ITEM macro is used to encapsulate elements.

Usage - C99 Specific 

CHAOS_PP_ITEM(type, ...)
type

The type of the generic data structure from which __VA_ARGS__ was extracted.

[...]

The element that was extracted from a generic data structure.

Usage - C/C++ Specific 

CHAOS_PP_ITEM(type, x)
type

The type of the generic data structure from which x was extracted.

x

The element that was extracted from a generic data structure.

Remarks

This macro is used by generic algorithms to encapsulate elements before they are passed to user-defined macros.

CHAOS_PP_HEAD

The CHAOS_PP_HEAD macro extracts the first element from a generic data structure.

Usage 

CHAOS_PP_HEAD(g)
g

The non-nil generic data structure whose first element is to be extracted.

Remarks

This macro expands to the first element contained in g.

See Also

  • CHAOS_PP_ARRAY_HEAD
  • CHAOS_PP_LIST_HEAD
  • CHAOS_PP_SEQ_HEAD
  • CHAOS_PP_STRING_HEAD
  • CHAOS_PP_TAIL
  • CHAOS_PP_TUPLE_HEAD
  • CHAOS_PP_VARIADIC_SEQ_HEAD

Sample Code

#include <chaos/preprocessor/generics/core.h>
#include <chaos/preprocessor/list/core.h>
#include <chaos/preprocessor/string/core.h>

CHAOS_PP_HEAD((CHAOS_PP_LIST) (1, (2, (3, ...)))) // 1
CHAOS_PP_HEAD((CHAOS_PP_STRING) 1 2 3 4 5)        // 1

CHAOS_PP_TAIL

The CHAOS_PP_TAIL macro removes the first element from a generic data structure.

Usage 

CHAOS_PP_TAIL(g)
g

The non-nil generic data structure whose first element is to be removed.

Remarks

This macro expands to a generic data structure of the same type as g containing all the elements of g except the first.

See Also

  • CHAOS_PP_ARRAY_TAIL
  • CHAOS_PP_HEAD
  • CHAOS_PP_LIST_TAIL
  • CHAOS_PP_SEQ_TAIL
  • CHAOS_PP_STRING_TAIL
  • CHAOS_PP_TUPLE_TAIL
  • CHAOS_PP_VARIADIC_SEQ_TAIL

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/generics/core.h>
#include <chaos/preprocessor/seq/core.h>
#include <chaos/preprocessor/tuple/core.h>

CHAOS_PP_TAIL((CHAOS_PP_SEQ) (X)(Y)(Z)) // (CHAOS_PP_SEQ) (Y)(Z)

#if CHAOS_PP_VARIADICS

CHAOS_PP_TAIL((CHAOS_PP_TUPLE) (a, b, c, d, e))
// (CHAOS_PP_TUPLE) (b, c, d, e)

#endif

CHAOS_PP_IS_CONS

The CHAOS_PP_IS_CONS macro is a predicate that tests whether a generic data structure is non-nil.

Usage 

CHAOS_PP_IS_CONS(g)
g

The generic data structure to be tested.

Remarks

If g is non-nil, this macro expands to 1. Otherwise, it expands to 0.

See Also

  • CHAOS_PP_ARRAY_IS_CONS
  • CHAOS_PP_IS_NIL
  • CHAOS_PP_LIST_IS_CONS
  • CHAOS_PP_SEQ_IS_CONS
  • CHAOS_PP_STRING_IS_CONS
  • CHAOS_PP_TUPLE_IS_CONS
  • CHAOS_PP_VARIADIC_SEQ_IS_CONS

Sample Code

#include <chaos/preprocessor/generics/core.h>
#include <chaos/preprocessor/list/core.h>

CHAOS_PP_IS_CONS((CHAOS_PP_LIST) ...)                // 0
CHAOS_PP_IS_CONS((CHAOS_PP_LIST) (a, (b, (c, ...)))) // 1
