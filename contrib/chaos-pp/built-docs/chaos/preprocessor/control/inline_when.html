The control/inline_when.h header defines a macro that expands to an expression if a condition is 1.

Usage

#include <chaos/preprocessor/control/inline_when.h>

Contents

  • CHAOS_PP_INLINE_WHEN

CHAOS_PP_INLINE_WHEN

The CHAOS_PP_INLINE_WHEN macro expands to its second argument if its first argument is 1.

Usage - C99 Specific 

CHAOS_PP_INLINE_WHEN(bit)(...)
bit

The condition that determines whether the macro should expand to __VA_ARGS__ or nothing. Valid values are 0 and 1.

[...]

The resulting expression if bit is 1.

Usage - C/C++ Specific 

CHAOS_PP_INLINE_WHEN(bit)(x)
bit

The condition that determines whether the macro should expand to x or nothing. Valid values are 0 and 1.

x

The resulting expression if bit is 1.

Remarks

If bit expands to 0, this macro expands to nothing. Otherwise, it expands to x or __VA_ARGS__.

This macro does not perform a Boolean conversion on bit. If this conversion is desirable, CHAOS_PP_WHEN may be used instead.

This macro is lazy. If bit is 0, x (or __VA_ARGS__) is not evaluated.

Sample Code

#include <chaos/preprocessor/control/inline_when.h>

#define VALID() 1
#define INVALID() VALID(!) // too many arguments

CHAOS_PP_INLINE_WHEN(1)(VALID())   // 1
CHAOS_PP_INLINE_WHEN(0)(INVALID()) //

CHAOS_PP_INLINE_WHEN_ID

The CHAOS_PP_INLINE_WHEN_ID macro indirectly expands to CHAOS_PP_INLINE_WHEN.

Usage 

CHAOS_PP_INLINE_WHEN_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_INLINE_WHEN_

The CHAOS_PP_INLINE_WHEN_ macro expands to a λ expression representing CHAOS_PP_INLINE_WHEN.

Usage - C99 Specific 

CHAOS_PP_INLINE_WHEN_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_INLINE_WHEN)
