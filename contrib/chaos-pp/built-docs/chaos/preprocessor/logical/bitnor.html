CHAOS_PP_BITNOR

The CHAOS_PP_BITNOR macro expands to the logical NOR of its operands.

Usage 

CHAOS_PP_BITNOR(x)(y)
x

The left operand. Valid values are 0 and 1.

y

The right operand. Valid values are 0 and 1.

Remarks

If x and y are both 0, this macro expands to 1. Otherwise, it expands to 0.

This macro does not perform Boolean conversions on its operands. If this conversion is desirable, CHAOS_PP_NOR may be used instead.

This macro is lazy. If x is 1, y is not evaluated.

See Also

  • CHAOS_PP_NOR

Sample Code

#include <chaos/preprocessor/debug/failure.h>
#include <chaos/preprocessor/logical/bitnor.h>

CHAOS_PP_BITNOR(1)(1) // 0
CHAOS_PP_BITNOR(0)(0) // 1

CHAOS_PP_BITNOR(1)(CHAOS_PP_FAILURE()) // 0

CHAOS_PP_BITNOR_ID

The CHAOS_PP_BITNOR_ID macro indirectly expands to CHAOS_PP_BITNOR.

Usage 

CHAOS_PP_BITNOR_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_BITNOR_

The CHAOS_PP_BITNOR_ macro expands to a λ expression representing CHAOS_PP_BITNOR.

Usage - C99 Specific 

CHAOS_PP_BITNOR_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_BITNOR)
