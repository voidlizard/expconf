CHAOS_PP_BITAND

The CHAOS_PP_BITAND macro expands to the logical AND of its operands.

Usage 

CHAOS_PP_BITAND(x)(y)
x

The left operand. Valid values are 0 and 1.

y

The right operand. Valid values are 0 and 1.

Remarks

If x and y are both 1, this macro expands to 1. Otherwise, it expands to 0.

This macro does not perform Boolean conversions on its operands. If this conversion is desirable, CHAOS_PP_AND may be used instead.

This macro is lazy. If x is 0, y is not evaluated.

See Also

  • CHAOS_PP_AND

Sample Code

#include <chaos/preprocessor/debug/failure.h>
#include <chaos/preprocessor/logical/bitand.h>

CHAOS_PP_BITAND(1)(1) // 1
CHAOS_PP_BITAND(1)(0) // 0

CHAOS_PP_BITAND(0)(CHAOS_PP_FAILURE()) // 0

CHAOS_PP_BITAND_ID

The CHAOS_PP_BITAND_ID macro indirectly expands to CHAOS_PP_BITAND.

Usage 

CHAOS_PP_BITAND_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_BITAND_

The CHAOS_PP_BITAND_ macro expands to a λ expression representing CHAOS_PP_BITAND.

Usage - C99 Specific 

CHAOS_PP_BITAND_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_BITAND)
