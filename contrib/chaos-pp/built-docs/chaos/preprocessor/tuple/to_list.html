CHAOS_PP_TUPLE_TO_LIST

The CHAOS_PP_TUPLE_TO_LIST macro converts a tuple to a list.

Usage 

CHAOS_PP_TUPLE_TO_LIST(size, tuple)
size

The size of the tuple to be converted. Valid values range from 0 to CHAOS_PP_LIMIT_TUPLE.

tuple

The tuple to be converted.

Remarks

This macro expands to a list containing the elements of tuple.

Sample Code

#include <chaos/preprocessor/tuple/to_list.h>

#define DATA() (x, y, z)

CHAOS_PP_TUPLE_TO_LIST(3, DATA()) // (x, (y, (z, ...)))

CHAOS_PP_TUPLE_TO_LIST_ID

The CHAOS_PP_TUPLE_TO_LIST_ID macro indirectly expands to CHAOS_PP_TUPLE_TO_LIST.

Remarks

This macro effectively allows CHAOS_PP_TUPLE_TO_LIST to pass through itself without becoming disabled.

CHAOS_PP_TUPLE_TO_LIST_ALT

The CHAOS_PP_TUPLE_TO_LIST_ALT macro converts a tuple to a list.

Usage - C99 Specific 

CHAOS_PP_TUPLE_TO_LIST_ALT(tuple)
[tuple]

The tuple to be converted.

Remarks

This macro expands to a list containing the elements of tuple.

The efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Θ, where n is the number of elements in tuple and tω is CHAOS_PP_LIMIT_TUPLE.

This macro operates on the nil tuple representation used by the core tuple operations defined in <chaos/preprocessor/tuple/core.h>. Specifically, () is a one-element tuple, not a zero-element tuple.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_TUPLE_TO_LIST_ALT_BYPASS must be used instead.

The CHAOS_PP_TUPLE_TO_LIST_ALT_BYPASS macro behaves identically to CHAOS_PP_TUPLE_TO_LIST_ALT except that it is parametized by the bypass recursion state.
