CHAOS_PP_VARIADIC_ELEM

The CHAOS_PP_VARIADIC_ELEM macro extracts an element of a variadic list of arguments.

Usage - C99 Specific 

CHAOS_PP_VARIADIC_ELEM(i, ...)
i

The zero-based index of the element to extract. Valid values range from 0 to CHAOS_PP_LIMIT_TUPLE - 1.

[...]

The variadic list of arguments from which an element is to be extracted.

Remarks

This macro expands to the i-th argument within __VA_ARGS__. If the i-th element doesn't exist, this macro expands to nothing.

Sample Code

#include <chaos/preprocessor/tuple/elem.h>

#if CHAOS_PP_VARIADICS

CHAOS_PP_VARIADIC_ELEM(0, a, b, c, d, e) // a
CHAOS_PP_VARIADIC_ELEM(2, x, y, z)       // z

#endif

CHAOS_PP_VARIADIC_ELEM_ID

The CHAOS_PP_VARIADIC_ELEM_ID macro indirectly expands to CHAOS_PP_VARIADIC_ELEM.

Usage 

CHAOS_PP_VARIADIC_ELEM_ID()

Remarks

This macro effectively allows CHAOS_PP_VARIADIC_ELEM to pass through itself without becoming disabled.

CHAOS_PP_TUPLE_ELEM

The CHAOS_PP_TUPLE_ELEM macro extracts an element from a tuple.

Usage - C99 Specific 

CHAOS_PP_TUPLE_ELEM(size, i, tuple)
size

This argument exists only for compatibility. It is otherwise ignored.

i

The zero-based index of the element to be extracted. Valid values range from 0 to CHAOS_PP_LIMIT_TUPLE - 1.

tuple

The tuple from which an element is to be extracted.

Usage - C/C++ Specific 

CHAOS_PP_TUPLE_ELEM(size, i, tuple)
size

The size of the tuple from which an element is to be extracted. Valid values range from 0 to CHAOS_PP_LIMIT_TUPLE.

i

The zero-based index of the element to be extracted. Valid values range from 0 to CHAOS_PP_LIMIT_TUPLE - 1.

tuple

The tuple from which an element is to be extracted.

Remarks

This macro expands to the i-th element of tuple. If the i-th element doesn't exist, this macro expands to nothing.

CHAOS_PP_TUPLE_ELEM_ALT

The CHAOS_PP_TUPLE_ELEM_ALT macro extracts an element from a tuple.

Usage - C99 Specific 

CHAOS_PP_TUPLE_ELEM_ALT(i, tuple)
i

The zero-based index of the element to be extracted. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

tuple

The tuple from which an element is to be extracted.

Remarks

This macro extracts the i-th element of tuple. Consequently, tuple must contain at least i + 1 elements.

The efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Θ, where n is i and tω is CHAOS_PP_LIMIT_TUPLE.

This macro operates on the nil tuple representation used by the core tuple operations defined in <chaos/preprocessor/tuple/core.h>. Specifically, () is a one-element tuple, not a zero-element tuple.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_TUPLE_ELEM_BYPASS must be used instead.

CHAOS_PP_TUPLE_ELEM_ALT_BYPASS

The CHAOS_PP_TUPLE_ELEM_ALT_BYPASS macro behaves identically to CHAOS_PP_TUPLE_ELEM_ALT except that it is parametized by the bypass recursion state.

Usage - C99 Specific 

CHAOS_PP_TUPLE_ELEM_ALT_BYPASS(s, i, tuple)
s

The current bypass recursion state.

i

The zero-based index of the element to be extracted. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

tuple

The tuple from which an element is to be extracted.

Remarks

If bypass semantics are already in effect, this macro must be used instead of CHAOS_PP_TUPLE_ELEM_ALT.
