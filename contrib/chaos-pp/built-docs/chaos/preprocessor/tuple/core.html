CHAOS_PP_TUPLE

The CHAOS_PP_TUPLE macro designates the name of the tuple data structure.

Usage - C99 Specific 

CHAOS_PP_TUPLE

Remarks

This macro expands to CHAOS_PP_TUPLE. In other words, the identifier CHAOS_PP_TUPLE is defined as itself.

CHAOS_PP_TUPLE_NIL

The CHAOS_PP_TUPLE_NIL macro expands to a nil tuple.

Usage - C99 Specific 

CHAOS_PP_TUPLE_NIL()

Remarks

This macro expands to nothing.

Note that a nil tuple is an empty sequence of preprocessing tokens. In particular, when a tuple is used a generic data structure, nullary parentheses (e.g. ()) are considered to be a unary tuple. (This differs from the tuple-specific macros which treat () as a zero-element tuple.)

See Also

  • CHAOS_PP_ARRAY_NIL
  • CHAOS_PP_LIST_NIL
  • CHAOS_PP_NIL
  • CHAOS_PP_SEQ_NIL
  • CHAOS_PP_STRING_NIL
  • CHAOS_PP_VARIADIC_SEQ_NIL

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/tuple/core.h>

#if CHAOS_PP_VARIADICS

CHAOS_PP_TUPLE_NIL()                         //
CHAOS_PP_TUPLE_CONS(CHAOS_PP_TUPLE_NIL(), +) // (+)

#endif

CHAOS_PP_TUPLE_CONS

The CHAOS_PP_TUPLE_CONS macro appends an element to the head of a tuple.

Usage - C99 Specific 

CHAOS_PP_TUPLE_CONS(tuple, x)
[tuple]

The tuple to which x is to be appended.

[x]

The element to be appended to the head of tuple.

Remarks

This macro expands to a tuple containing the elements of tuple with x appended to the head.

See Also

  • CHAOS_PP_ARRAY_CONS
  • CHAOS_PP_CONS
  • CHAOS_PP_LIST_CONS
  • CHAOS_PP_SEQ_CONS
  • CHAOS_PP_STRING_CONS
  • CHAOS_PP_VARIADIC_SEQ_CONS

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/tuple/core.h>

#if CHAOS_PP_VARIADICS

CHAOS_PP_TUPLE_CONS(CHAOS_PP_TUPLE_NIL(), a) // (a)
CHAOS_PP_TUPLE_CONS(, a)                     // (a)
CHAOS_PP_TUPLE_CONS((), a)                   // (, a)

#endif

CHAOS_PP_TUPLE_ITEM

The CHAOS_PP_TUPLE_ITEM macro is an identity macro used to encapsulate elements.

Usage - C99 Specific 

CHAOS_PP_TUPLE_ITEM(x)
[x]

The element of a tuple to be encapsulated.

Remarks

This macro is used by generic algorithms to encapsulate elements before they are passed to user-defined macros. Elements in a tuple cannot be variadic. Therefore, this macro is a simple identity macro.

CHAOS_PP_TUPLE_IS_CONS

The CHAOS_PP_TUPLE_IS_CONS macro is a predicate that tests whether a tuple is non-nil.

Usage - C99 Specific 

CHAOS_PP_TUPLE_IS_CONS(tuple)
[tuple]

The tuple to be tested.

Remarks

If tuple is non-nil, this macro expands to 1. Otherwise, it expands to 0.

See Also
