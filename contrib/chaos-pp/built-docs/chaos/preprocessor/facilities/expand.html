The facilities/expand.h header defines a macro used to cause a singular delayed expansion.

Usage

#include <chaos/preprocessor/facilities/expand.h>

Contents

  • CHAOS_PP_EXPAND

CHAOS_PP_EXPAND

The CHAOS_PP_EXPAND macro effectively causes its argument to expand twice.

Usage - C99 Specific 

CHAOS_PP_EXPAND(...)
[...]

The argument to be expanded.

Usage - C/C++ Specific 

CHAOS_PP_EXPAND(x)
x

The argument to be expanded.

Remarks

This macro expands to its argument.

The argument x (or __VA_ARGS__) is scanned for macro replacement once on entry to this macro and again during the rescanning of this macro's replacement list.

The result of the use of this macro in a context that causes the expansion of a library primitive during the secondary expansion is undefined. 

#define ARGS (text)

CHAOS_PP_EXPAND(CHAOS_PP_WSTRINGIZE ARGS) // undefined

Secondary expansion of library primitives such as the above may or may not work as desired depending on whether the primitive internally uses CHAOS_PP_EXPAND.

Sample Code

#include <chaos/preprocessor/facilities/expand.h>

#define MACRO(a, b) a + b
#define ARGS (1, 2)

MACRO ARGS                  // MACRO (1 + 2)
CHAOS_PP_EXPAND(MACRO ARGS) // 1 + 2

CHAOS_PP_EXPAND_ID

The CHAOS_PP_EXPAND_ID macro indirectly expands to CHAOS_PP_EXPAND.

Usage 

CHAOS_PP_EXPAND_ID()

Remarks

This macro effectively allows CHAOS_PP_EXPAND to pass through itself without becoming disabled.

CHAOS_PP_EXPAND_

The CHAOS_PP_EXPAND_ macro expands to a λ expression representing CHAOS_PP_EXPAND.

Usage - C99 Specific 

CHAOS_PP_EXPAND_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_EXPAND_ID)()
