The facilities/abbr.h header defines a macro used to abbreviate interface macros.

Usage

#include <chaos/preprocessor/facilities/abbr.h>

Contents

  • CHAOS_PP_ABBR

CHAOS_PP_ABBR

The CHAOS_PP_ABBR macro is used to abbreviate library interface macros.

Usage 

CHAOS_PP_ABBR(id)
id

The unprefixed name of a library primitive.

Remarks

This macro prefixes id with CHAOS_PP_. It is intended to be is passed as an argument to another macro such that it can be used as an abbreviation of the prefix CHAOS_PP_. For example, 

#define INTERFACE(arg) IMPLEMENTATION(CHAOS_PP_ABBR, arg)
#define IMPLEMENTATION(_, arg) _(UNBOX)(arg)

INTERFACE(...)    //
INTERFACE((const) // const

The argument id is not allowed to expand prior to prefixing.

See Also

  • CHAOS_PP_ALTERNATE
  • CHAOS_PP_BIND
  • CHAOS_PP_EXTERN
  • CHAOS_PP_PREFIX

Sample Code

#include <chaos/preprocessor/cat.h>
#include <chaos/preprocessor/facilities/abbr.h>

#define MACRO(a, b, c, d) MACRO_I(CHAOS_PP_ABBR, a, b, c, d)
#define MACRO_I(_, a, b, c, d) \
    _(CAT)( \
        _(PRIMITIVE_CAT)(a, b), \
        _(PRIMITIVE_CAT)(c, d) \
    ) \
    /**/

MACRO(1, 2, 3, 4) // 1234

CHAOS_PP_ABBR_ID

The CHAOS_PP_ABBR_ID macro indirectly expands to CHAOS_PP_ABBR.

Usage 

CHAOS_PP_ABBR_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_ABBR_

The CHAOS_PP_ABBR_ macro expands to a λ expression representing CHAOS_PP_ABBR.

Usage - C99 Specific 

CHAOS_PP_ABBR_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_ABBR)
