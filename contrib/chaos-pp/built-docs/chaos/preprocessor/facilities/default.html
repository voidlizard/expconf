The facilities/default.h header defines macros used to apply default values for true-optional parameters.

Usage

#include <chaos/preprocessor/facilities/default.h>

Contents

  • CHAOS_PP_DEFAULT
  • CHAOS_PP_DEFAULT_AT

CHAOS_PP_DEFAULT

The CHAOS_PP_DEFAULT macro is used to apply a default argument value if a true-optional argument doesn't exist.

Usage - C99 Specific 

CHAOS_PP_DEFAULT(def, ...)
[def]

The default value to be used if no true-optional argument exists in __VA_ARGS__.

[...]

The argument that possibly contains a true-optional argument.

Remarks

If __VA_ARGS__ contains more than a single argument, this macro expands to the second argument. Otherwise, it expands to def.

See Also

  • CHAOS_PP_DEFAULT_AT

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/facilities/default.h>
#include <chaos/preprocessor/facilities/optional.h>

#define MACRO(...) \
    MACRO_I( \
        CHAOS_PP_NON_OPTIONAL(__VA_ARGS__), \
        CHAOS_PP_DEFAULT(Chaos, __VA_ARGS__) \
    ) \
    /**/
#define MACRO_I(a, b) b Preprocessor [a]

MACRO(+)        // Chaos Preprocessor [+]
MACRO(-, Boost) // Boost Preprocessor [-]

CHAOS_PP_DEFAULT_ID

The CHAOS_PP_DEFAULT_ID macro indirectly expands to CHAOS_PP_DEFAULT.

Usage - C99 Specific 

CHAOS_PP_DEFAULT_ID()

Remarks

This macro effectively allows CHAOS_PP_DEFAULT to pass through itself without becoming disabled.

CHAOS_PP_DEFAULT_

The CHAOS_PP_DEFAULT_ macro expands to a λ expression representing CHAOS_PP_DEFAULT.

Usage - C99 Specific 

CHAOS_PP_DEFAULT_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_DEFAULT_ID)()

CHAOS_PP_DEFAULT_AT

The CHAOS_PP_DEFAULT_AT macro is used to apply a default argument value for true-optional parameters other than just the first.

Usage - C99 Specific 

CHAOS_PP_DEFAULT_AT(i, def, ...)
i

The index of the true-optional argument possibly contained in __VA_ARGS__.

[def]

The default value to be used if the i-th true-optional argument does not exist.

[...]

The argument that possibly contains true-optional arguments.

Remarks

If an i-th true-optional argument exists, this macro expands to that argument. Otherwise, it expands to def.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_DEFAULT_AT_BYPASS must be used instead.

See Also

  • CHAOS_PP_DEFAULT

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/facilities/default.h>
#include <chaos/preprocessor/facilities/optional.h>

#if CHAOS_PP_VARIADICS

// NAME(first, last => Mensonides, middle => /**/)

#define NAME(...) \
    NAME_I( \
        CHAOS_PP_NON_OPTIONAL(__VA_ARGS__), \
        CHAOS_PP_DEFAULT_AT(0, Mensonides, __VA_ARGS__), \
        CHAOS_PP_DEFAULT_AT(1,, __VA_ARGS__) \
    ) \
    /**/
#define NAME_I(first, last, middle) first middle last

NAME(Paul)                    // Paul Mensonides
NAME(Paul, Mensonides, Jelle) // Paul Jelle Mensonides

#endif

CHAOS_PP_DEFAULT_AT_BYPASS

The CHAOS_PP_DEFAULT_AT_BYPASS macro behaves identically to CHAOS_PP_DEFAULT_AT except that it is parametized by the bypass recursion state.

Usage - C99 Specific 

CHAOS_PP_DEFAULT_AT_BYPASS(s, i, def, ...)
s

The current bypass recursion state.

i

The index of the true-optional argument possibly contained in __VA_ARGS__.

[def]

The default value to be used if the i-th true-optional argument does not exist.

[...]

The argument that possibly contains true-optional arguments.

Remarks

If bypass semantics are already in effect, this macro must be used instead of CHAOS_PP_DEFAULT_AT.
