The facilities/intercept.h header defines a macro that intercepts the concatenation of a numeric value.

Usage

#include <chaos/preprocessor/facilities/intercept.h>

Contents

  • CHAOS_PP_INTERCEPT

CHAOS_PP_INTERCEPT

The CHAOS_PP_INTERCEPT macro expands to an entity that expands to nothing when a number is concatenated to it.

Usage 

CHAOS_PP_INTERCEPT

Remarks

This macro is used to intercept concatenations of numbers ranging from 0 to CHAOS_PP_LIMIT_MAG. When a number in this range is concatenated to the result of this macro, a new macro name is produced which expands to nothing.

This macro is primarily useful in situations where the library automatically concatenates numbers to user-specified arguments.

Sample Code

#include <chaos/preprocessor/facilities/intercept.h>
#include <chaos/preprocessor/repetition/enum_params.h>

CHAOS_PP_ENUM_PARAMS(3, class T)
// class T0, class T1, class T2

CHAOS_PP_ENUM_PARAMS(3, class CHAOS_PP_INTERCEPT)
// class, class, class

CHAOS_PP_ENUM_PARAMS(3, (class T, = U))
// class T0 = U0, class T1 = U1, class T2 = U2

CHAOS_PP_ENUM_PARAMS(3, (class T, = int CHAOS_PP_INTERCEPT))
// class T0 = int, class T1 = int, class T2 = int

CHAOS_PP_INTERCEPT_ID

The CHAOS_PP_INTERCEPT_ID macro indirectly expands to CHAOS_PP_INTERCEPT.

Usage 

CHAOS_PP_INTERCEPT_ID()

Remarks

This macro effectively allows CHAOS_PP_INTERCEPT to be passed around without expansion.

CHAOS_PP_INTERCEPT_

The CHAOS_PP_INTERCEPT_ macro expands to a λ expression representing CHAOS_PP_INTERCEPT.

Usage - C99 Specific 

CHAOS_PP_INTERCEPT_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_INTERCEPT_ID)()
