The facilities/binary.h header defines macros that extract elements from a binary tuple.

Usage

#include <chaos/preprocessor/facilities/binary.h>

Contents

  • CHAOS_PP_BINARY
  • CHAOS_PP_BINARY_CTOR

CHAOS_PP_BINARY

The CHAOS_PP_BINARY macro extracts an element from a binary tuple.

Usage - C99 Specific 

CHAOS_PP_BINARY(i)(a, ...)
i

The index of the element to be extracted. Valid values are 0 and 1.

[a]

The first element of the tuple.

[...]

The second element of the tuple.

Usage - C/C++ Specific 

CHAOS_PP_BINARY(i)(a, b)
i

The index of the element to be extracted. Valid values are 0 and 1.

a

The first element of the tuple.

b

The second element of the tuple.

Remarks

If i is 0, this macro expands to a. Otherwise, it expands to b or __VA_ARGS__.

This macro is lazy. If i is 0, b (or __VA_ARGS__) is not evaluated. Likewise, if i is 1, a is not evaluated.

Sample Code

#include <chaos/preprocessor/facilities/binary.h>

CHAOS_PP_BINARY(0)(x, y) // x
CHAOS_PP_BINARY(1)(x, y) // y

CHAOS_PP_BINARY_ID

The CHAOS_PP_BINARY_ID macro indirectly expands to CHAOS_PP_BINARY.

Usage 

CHAOS_PP_BINARY_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_BINARY_

The CHAOS_PP_BINARY_ macro expands to a λ expression representing CHAOS_PP_BINARY.

Usage - C99 Specific 

CHAOS_PP_BINARY_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_BINARY)

CHAOS_PP_BINARY_CTOR

The CHAOS_PP_BINARY_CTOR macro extracts an element from a binary tuple.

Usage 

CHAOS_PP_BINARY_CTOR(i, tuple)
i

The index of the element to be extracted. Valid values are 0 and 1.

tuple

The binary tuple from which an element is to be extracted.

Remarks

This macro operates similarly to CHAOS_PP_BINARY. Unlike CHAOS_PP_BINARY, however, the tuple is "constructed" at the expense of lazy evaluation.

In C99, the tuple argument need not be strictly binary. However, the macro behaves as if it is binary yielding the first argument or the arguments after the first depending on i. 

CHAOS_PP_BINARY_CTOR(0, (a, b, c)) // a
CHAOS_PP_BINARY_CTOR(1, (a, b, c)) // b, c

The CHAOS_PP_BINARY_CTOR_ID macro indirectly expands to CHAOS_PP_BINARY_CTOR.

Usage 

CHAOS_PP_BINARY_CTOR_ID()

Remarks

This macro effectively allows CHAOS_PP_BINARY_CTOR to pass through itself without becoming disabled.
