The facilities/bind.h header defines macros used to selectively qualify unprefixed macro names.

Usage

#include <chaos/preprocessor/facilities/bind.h>

CHAOS_PP_BIND

The CHAOS_PP_BIND macro is used to introduce a context that allows unprefixed macros to qualify themselves.

Usage - C99 Specific 

CHAOS_PP_BIND(, ...)
[...]

Any number of preprocessing tokens.

Remarks

This macro is used as a binding context on unprefixed macros that utilize CHAOS_PP_PREFIX.

This macro employs automatic recursion up to a nested binding depth of CHAOS_PP_LIMIT_BIND.

See Also

  • CHAOS_PP_ABBR
  • CHAOS_PP_ALTERNATE
  • CHAOS_PP_EXTERN
  • CHAOS_PP_PREFIX

Sample Code

#include <chaos/preprocessor/cat.h>
#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/facilities/bind.h>

#if CHAOS_PP_VARIADICS

#define CAT CHAOS_PP_PREFIX(CAT)

CAT(x, CAT(y, z)) // CAT(x, CAT(y, z))

CHAOS_PP_BIND(, CAT(x, CAT(y, z))) // xyz

#endif

CHAOS_PP_BIND_ID

The CHAOS_PP_BIND_ID macro indirectly expands to CHAOS_PP_BIND.

Usage - C99 Specific 

CHAOS_PP_BIND_ID()

Remarks

This macro effectively allows CHAOS_PP_BIND to be passed around without expansion.

CHAOS_PP_BIND_

The CHAOS_PP_BIND_ macro expands to a λ expression representing CHAOS_PP_BIND.

Usage - C99 Specific 

CHAOS_PP_BIND_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_BIND_ID)()

CHAOS_PP_PREFIX

The CHAOS_PP_PREFIX macro is used to define unprefixed names that are isolated from core language interference.

Usage - C99 Specific 

CHAOS_PP_PREFIX(id)
id

The unprefixed name of a library primitive.

Remarks

This macro is used to define unprefixed variants of library primitives that are incapable of interferring with the core language.

Normal usage consists of starting with a prefixed name and defining an object-like macro of the form: 

#define id CHAOS_PP_PREFIX(id)

For example, 

#define STRINGIZE CHAOS_PP_PREFIX(STRINGIZE)

This defines an object-like macro that expands to CHAOS_PP_STRINGIZE if (and only if) it is expanded in a context introduced by CHAOS_PP_BIND. Otherwise, it expands to plain STRINGIZE.

This effectively renders unprefixed macro names from conflicting with anything but other unprefixed macro names.

See Also

  • CHAOS_PP_ABBR
  • CHAOS_PP_ALTERNATE
  • CHAOS_PP_BIND
  • CHAOS_PP_EXTERN

Sample Code

#include <chaos/preprocessor/cat.h>
#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/stringize.h>

#if CHAOS_PP_VARIADICS

CAT(L, STRINGIZE(text)) // CAT(L, STRINGIZE(text))

CHAOS_PP_BIND(, CAT(L, STRINGIZE(text))) // L"text

#endif

CHAOS_PP_EXTERN

The CHAOS_PP_EXTERN macro is used to bypass the context introduced via CHAOS_PP_BIND.

Usage - C99 Specific 

CHAOS_PP_EXTERN(, id)
id

The unprefixed name of a library primitive.

Remarks

This macro is used to override the context introduced via CHAOS_PP_BIND.

See Also

  • CHAOS_PP_ABBR
  • CHAOS_PP_ALTERNATE
  • CHAOS_PP_BIND
  • CHAOS_PP_PREFIX

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/facilities/bind.h>
#include <chaos/preprocessor/stringize.h>

#if CHAOS_PP_VARIADICS

#define STRINGIZE CHAOS_PP_PREFIX(STRINGIZE)

CHAOS_PP_BIND(,
    STRINGIZE(text)
    CHAOS_PP_EXTERN(, STRINGIZE)(text)
)
// "text" "STRINGIZE(text)"

#endif

CHAOS_PP_ALTERNATE

The CHAOS_PP_ALTERNATE macro works similarly to CHAOS_PP_PREFIX, but allows the use of an alternate mapping.

Usage - C99 Specific 

CHAOS_PP_ALTERNATE(a, b)
a

The unprefixed name of a macro.

b

The the prefixed name of a macro.

Remarks

This macro behaves similarly to CHAOS_PP_PREFIX except that an arbitrary mapping other than CHAOS_PP_-prefixing can be applied.

See Also

  • CHAOS_PP_ABBR
  • CHAOS_PP_BIND
  • CHAOS_PP_EXTERN
  • CHAOS_PP_PREFIX

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/facilities/bind.h>

#if CHAOS_PP_VARIADICS

#define LIB_MACRO(x) x

#define MACRO CHAOS_PP_ALTERNATE(MACRO, LIB_MACRO)

MACRO(1) // MACRO(1)

CHAOS_PP_BIND(, MACRO(1)) // 1

#endif
