The facilities/identity.h header defines a macro that creates an invokable entity.

Usage

#include <chaos/preprocessor/facilities/identity.h>

Contents

  • CHAOS_PP_IDENTITY

CHAOS_PP_IDENTITY

The CHAOS_PP_IDENTITY macro constructs an invokable entity from its argument.

Usage - C99 Specific 

CHAOS_PP_IDENTITY(...)
[...]

The result produced when the result of this macro is invoked.

Usage - C/C++ Specific 

CHAOS_PP_IDENTITY(x)
x

The result produced when the result of this macro is invoked.

Remarks

This macro expands to a nullary invokable entity that, when invoked with (), expands to x or __VA_ARGS__.

This macro is often used as a complement to CHAOS_PP_EMPTY to simulate empty arguments in C (prior to C99) and C++.

Note that, despite its name, CHAOS_PP_IDENTITY is not an "identity" function in the pure sense (i.e. λx.x in the lambda calculus).

See Also

  • CHAOS_PP_EMPTY
  • CHAOS_PP_UNBOX

Sample Code

#include <chaos/preprocessor/facilities/empty.h>
#include <chaos/preprocessor/facilities/identity.h>

#define A(cv) cv()

A(CHAOS_PP_EMPTY)           //
A(CHAOS_PP_IDENTITY(const)) // const

CHAOS_PP_IDENTITY_ID

The CHAOS_PP_IDENTITY_ID macro indirectly expands to CHAOS_PP_IDENTITY.

Usage 

CHAOS_PP_IDENTITY_ID()

Remarks

This macro effectively allows CHAOS_PP_IDENTITY to pass through itself without becoming disabled.

CHAOS_PP_IDENTITY_

The CHAOS_PP_IDENTITY_ macro expands to a λ expression representing CHAOS_PP_IDENTITY.

Usage - C99 Specific 

CHAOS_PP_IDENTITY_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_IDENTITY_ID)()
