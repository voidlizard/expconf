CHAOS_PP_DEFER

The CHAOS_PP_DEFER macro facilitates the deferral of a macro invocation through a single scan.

Usage 

CHAOS_PP_DEFER(macro)
macro

The macro name of a macro invocation to be deferred.

Remarks

This macro prevents macro from expanding during the first scan against arguments that exist following the invocation of CHAOS_PP_DEFER. For example, 

#define A() +
CHAOS_PP_DEFER(A)() // A ()

A subsequent scan is required in order to invoke A. As in, 

#define A() +
CHAOS_PP_EXPAND(CHAOS_PP_DEFER(A)()) // +

Sample Code

#include <chaos/preprocessor/recursion/basic.h>

#define ID(x) x

#define A() CHAOS_PP_DEFER(B)()
#define B() CHAOS_PP_DEFER(A)()

A() ID(A() ID(A() ID(A()))) // B() A() B() A()

CHAOS_PP_DEFER_ID

Usage 

CHAOS_PP_DEFER_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_DEFER_

The CHAOS_PP_DEFER_ macro expands to a λ expression representing CHAOS_PP_DEFER.

Usage - C99 Specific 

CHAOS_PP_DEFER_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_DEFER)

CHAOS_PP_REFER

The CHAOS_PP_REFER macro is an identity macro.

Usage 

CHAOS_PP_REFER(macro)
macro

The macro nameof a macro invocation.

Remarks

This macro is signature compatible with CHAOS_PP_DEFER, but it does not defer the invocation of macro against arguments that exist following the invocation of CHAOS_PP_REFER.

See Also

  • CHAOS_PP_DEFER

Sample Code

#include <chaos/preprocessor/recursion/basic.h>

#define A(mode) mode(B)(+)
#define B(x) x

A(CHAOS_PP_DEFER) // B(+)
A(CHAOS_PP_REFER) // +

CHAOS_PP_OBSTRUCT

The CHAOS_PP_OBSTRUCT macro facilitates the deferral of a macro invocation through two scans.

Usage - C99 Specific 

CHAOS_PP_OBSTRUCT(macro)
[macro]

The macro name of a macro invocation to be obstructed.

Usage - C/C++ Specific 

CHAOS_PP_OBSTRUCT()

Remarks

This macro is used as an obstruction that disappears after two scans. The behavior is similar to CHAOS_PP_DEFER except that the resulting expression is deferred through two scans instead of one.

Normal usage consists of inserting an invocation of this macro between the macro to be obstructed and the arguments to that macro. 

#define X(a, b) a + b
CHAOS_PP_EXPAND(X CHAOS_PP_OBSTRUCT()(1, 2)) // X(1, 2)

This usage is the normal usage because an invocation of CHAOS_PP_OBSTRUCT is often used as an argument to another macro, ultimately producing a shorthand single-scan deferral: 

#define X(a, b) Y(CHAOS_PP_OBSTRUCT(), a, b)
#define Y(_, a, b) a _() b _()

// vs.

#define Z(a, b) CHAOS_PP_DEFER(a)() CHAOS_PP_DEFER(b)()

#define P() +
#define M() -

X(P, M) // P() M()
Z(P, M) // P() M()

In C99, this macro may take the macro name as an argument. This yields syntax similar to CHAOS_PP_DEFER. 

#define X(...) __VA_ARGS__
CHAOS_PP_EXPAND(CHAOS_PP_OBSTRUCT(X)(!)) // X(!)

Sample Code

#include <chaos/preprocessor/arithmetic/dec.h>
#include <chaos/preprocessor/control/iif.h>
#include <chaos/preprocessor/logical/bool.h>
#include <chaos/preprocessor/recursion/basic.h>
#include <chaos/preprocessor/recursion/expr.h>

#define WHILE(pred, op, state) \
    WHILE_S(CHAOS_PP_STATE(), pred, op, state) \
    /**/
#define WHILE_S(s, pred, op, state) \
    WHILE_I( \
        CHAOS_PP_OBSTRUCT(), CHAOS_PP_NEXT(s), \
        pred, op, state \
    ) \
    /**/
#define WHILE_INDIRECT() WHILE_I
#define WHILE_I(_, s, pred, op, state) \
    CHAOS_PP_IIF _(pred _(s, state))( \
        CHAOS_PP_EXPR_S(s) _(WHILE_INDIRECT _()( \
            CHAOS_PP_OBSTRUCT _(), CHAOS_PP_NEXT(s), \
            pred, op, op _(s, state) \
        )), \
        state \
    ) \
    /**/

#define PRED(s, x) CHAOS_PP_BOOL(x)
#define OP(s, x) CHAOS_PP_DEC(x)

CHAOS_PP_EXPR(WHILE(
    PRED, OP, 100
))
// 0
