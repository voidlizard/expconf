The recursion/delay.h header defines a macro that delays the invocation of a macro.

Usage

#include <chaos/preprocessor/recursion/delay.h>

Contents

  • CHAOS_PP_DELAY

CHAOS_PP_DELAY

The CHAOS_PP_DELAY macro delays the expansion of a macro through an arbitrary number of scans.

Usage 

CHAOS_PP_DELAY(count, macro)
count

The number of scans through which the invocation of macro should delayed. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

macro

The macro whose invocation is to be delayed.

Remarks

This macro delays the invocation of macro through count scans.

Normal usage includes the invocation of this macro followed by the argument list to macro. For example, 

#define MACRO(a, b, c) // ...
CHAOS_PP_DELAY(5, MACRO)(~, ~, ~)

Sample Code

#include <chaos/preprocessor/recursion/delay.h>
#include <chaos/preprocessor/punctuation/comma.h>
#include <chaos/preprocessor/punctuation/paren.h>

#define A(x) B(x)
#define B(x) C(x)
#define C(x) x

A(
    CHAOS_PP_DELAY(2, CHAOS_PP_RPAREN)()
    CHAOS_PP_DELAY(2, CHAOS_PP_COMMA)()
    CHAOS_PP_DELAY(2, CHAOS_PP_LPAREN)()
)
// ) , (

CHAOS_PP_DELAY_ID

The CHAOS_PP_DELAY_ID macro indirectly expands to CHAOS_PP_DELAY.

Usage 

CHAOS_PP_DELAY_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_DELAY_

The CHAOS_PP_DELAY_ macro expands to a λ expression representing CHAOS_PP_DELAY.

Usage - C99 Specific 

CHAOS_PP_DELAY_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_DELAY)
