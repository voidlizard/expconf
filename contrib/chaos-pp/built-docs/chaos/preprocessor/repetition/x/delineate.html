The repetition/x/delineate.h header defines an exponential algorithm that delineates a user-specified target a specified number of times which can be parametrically resumed.

Usage

#include <chaos/preprocessor/repetition/x/delineate.h>

Contents

  • CHAOS_PP_DELINEATE_X

CHAOS_PP_DELINEATE_X

The CHAOS_PP_DELINEATE_X macro delineates its fourth argument the number of times specified by its second argument.

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_DELINEATE_X(size, count, sep, macro))
size

The size of the buffer of valid recursion steps maintained by the algorithm.

count

The number of times that macro is to be repeated. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

sep

The delineator to be situated between each repetition of macro. This delineator must be a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

sep()
macro

The target that is to be repeated. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, n)

Usage - C99 Specific

Remarks

This macro behaves identically to CHAOS_PP_DELINEATE except that the invocations of the target are not trampolined, the algorithm makes exponential use of the recursion backend, and the algorithm itself yields a trampolined deferred expression whenever the number of recursion steps is exhausted. This allows for parametric resumption

This algorithm maintains a buffer of size recursion steps available at all times. This guarantees that the target has at least size recursion steps available whenever it is invoked.

The exponential structure of the X-suffixed algorithms yields a total of α recursion steps, where p is the number of parametric resumptions plus the initial entry, sω is CHAOS_PP_LIMIT_EXPR, s is the current recursion state at the beginning of the algorithm (i.e. CHAOS_PP_STATE()), sδ is the buffer size, and z is the exponential base. Currently, z is fixed at two in the CHAOS_PP_DELINEATE_X algorithm (though it could easily be increased), which reduces the equation to β. Ignoring the complexity of macro (which is invoked count times), the efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Θ(n+1), where n is count. These bounds preserve all low-order terms, which implies that the algorithm can generate exactly γ repetitions, where m is CHAOS_PP_LIMIT_MAG.

This macro deduces the current recursion state and invokes CHAOS_PP_DELINEATE_X_S. If the current recursion state is already known, direct use of CHAOS_PP_DELINEATE_X_S is more efficient.

This macro may not be used if bypass semantics are in effect.

CHAOS_PP_DELINEATE_X_ID

The CHAOS_PP_DELINEATE_X_ID macro indirectly expands to CHAOS_PP_DELINEATE_X.

Usage 

CHAOS_PP_DELINEATE_X_ID()

Remarks

This macro effectively allows CHAOS_PP_DELINEATE_X to pass through itself without becoming disabled.

CHAOS_PP_DELINEATE_X_

The CHAOS_PP_DELINEATE_X_ macro expands to a λ expression representing CHAOS_PP_DELINEATE_X.

Usage - C99 Specific 

CHAOS_PP_DELINEATE_X_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_DELINEATE_X_ID)()
