The repetition/delineate_shifted_trailing_params.h header defines an algorithm that delineates a sequence of preprocessing tokens a specified number of times.

Usage

#include <chaos/preprocessor/repetition/delineate_shifted_trailing_params.h>

Contents

CHAOS_PP_DELINEATE_SHIFTED_TRAILING_PARAMS

The CHAOS_PP_DELINEATE_SHIFTED_TRAILING_PARAMS macro delineates its third argument one less time than the number specified by its first argument.

Usage - C99 Specific 

CHAOS_PP_DELINEATE_SHIFTED_TRAILING_PARAMS(count, sep, ...)
count

One greater than the number of times that __VA_ARGS__ is to be repeated. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

sep

The delineator to be situated between each repetition of __VA_ARGS__. This delineator must be a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

sep()
[...]

The sequence of preprocessing tokens to be repeated.

Usage - C/C++ Specific 

CHAOS_PP_DELINEATE_SHIFTED_TRAILING_PARAMS(count, sep, param)
count

One greater than the number of times that param is to be repeated. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

sep

The delineator to be situated between each repetition of param. This delineator must be a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

sep()
param

The sequence of preprocessing tokens to be repeated.

Remarks

This macro behaves identically to CHAOS_PP_DELINEATE_SHIFTED_PARAMS except that it inserts a leading delineator if any repetitions exist.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_DELINEATE_SHIFTED_TRAILING_PARAMS_BYPASS must be used instead.

CHAOS_PP_DELINEATE_SHIFTED_TRAILING_PARAMS_ID

The CHAOS_PP_DELINEATE_SHIFTED_TRAILING_PARAMS_ID macro indirectly expands to CHAOS_PP_DELINEATE_SHIFTED_TRAILING_PARAMS.

Remarks

This macro effectively allows CHAOS_PP_DELINEATE_SHIFTED_TRAILING_PARAMS to pass through itself without becoming disabled.
