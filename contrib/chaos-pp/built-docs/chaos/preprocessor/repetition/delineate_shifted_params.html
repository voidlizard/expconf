The repetition/delineate_shifted_params.h header defines an algorithm that delineates a sequence of preprocessing tokens a specified number of times.

Usage

#include <chaos/preprocessor/repetition/delineate_shifted_params.h>

Contents

  • CHAOS_PP_DELINEATE_SHIFTED_PARAMS

CHAOS_PP_DELINEATE_SHIFTED_PARAMS

The CHAOS_PP_DELINEATE_SHIFTED_PARAMS macro delineates its third argument one less time than the number specified by its first argument.

Usage - C99 Specific 

CHAOS_PP_DELINEATE_SHIFTED_PARAMS(count, sep, ...)
count

One greater than the number of times that __VA_ARGS__ is to be repeated. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

sep

The delineator to be situated between each repetition of __VA_ARGS__. This delineator must be a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

sep()
[...]

The sequence of preprocessing tokens to be repeated.

Usage - C/C++ Specific 

CHAOS_PP_DELINEATE_SHIFTED_PARAMS(count, sep, param)
count

One greater than the number of times that param is to be repeated. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

sep

The delineator to be situated between each repetition of param. This delineator must be a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

sep()
param

The sequence of preprocessing tokens to be repeated.

Remarks

This macro expands to the result of delineating param or __VA_ARGS__ with sep count - 1 times such that the rightmost preprocessing token of each repetition is concatenated to the position of the repetition. This is illustrated by the following pseudo-code (where N is count - 1): 

param ## 1 sep() param ## 2 sep() ... param ## N

The rightmost preprocessing token of param or __VA_ARGS__ must be either pasteable to a number or be a binary parenthetic expression such as 

(a, b)

If param or __VA_ARGS__ is a binary parenthetic expression, this macro extracts the two elements and concatenates the position of the current repetition to each of them. This is illustrated by the following pseudo-code (where N is again count - 1): 

a ## 1 b ## 1 sep() a ## 2 b ## 2 sep() ... a ## N b ## N

In C99, the requirement that a parenthetic expression must be binary is relaxed. It can be any arity provided that there are at least two elements. 

(a, ...)

In that circumstance, the current repetition number is concatenated to both the first of the two elements and to the all the rest of the elements as a single unit.

This secondary mechanism is intended to make it easy to define parameter lists, such as the following: 

T1 p1, T2 p2, ... T3 p3

The efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Î˜(n), where n is count.

Note that CHAOS_PP_DELINEATE_SHIFTED_PARAMS is functionally equivalent to CHAOS_PP_ENUM_SHIFTED_PARAMS when sep is CHAOS_PP_COMMA.

This macro differs from CHAOS_PP_DELINEATE_PARAMS simply by omitting the first repetition which can be useful in many contexts.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_DELINEATE_SHIFTED_PARAMS_BYPASS must be used instead.

Sample Code

#include <chaos/preprocessor/facilities/intercept.h>
#include <chaos/preprocessor/punctuation/comma.h>
#include <chaos/preprocessor/repetition/delineate_shifted_params.h>

CHAOS_PP_DELINEATE_SHIFTED_PARAMS(
    3, CHAOS_PP_COMMA, class T
)
// class T1, class T2

CHAOS_PP_DELINEATE_SHIFTED_PARAMS(
    3, CHAOS_PP_COMMA, (class T, = U)
)
// class T1 = U1, class T2 = U2

CHAOS_PP_DELINEATE_SHIFTED_PARAMS(
    3, CHAOS_PP_COMMA, (class T, = void CHAOS_PP_INTERCEPT)
)
// class T1 = void, class T2 = void

CHAOS_PP_DELINEATE_SHIFTED_PARAMS_ID

The CHAOS_PP_DELINEATE_SHIFTED_PARAMS_ID macro indirectly expands to CHAOS_PP_DELINEATE_SHIFTED_PARAMS.

Usage 

CHAOS_PP_DELINEATE_SHIFTED_PARAMS_ID()

Remarks

This macro effectively allows CHAOS_PP_DELINEATE_SHIFTED_PARAMS to pass through itself without becoming disabled.
