The ustringize.h header defines stringizing macros for producing Unicode string literals.

Usage

#include <chaos/preprocessor/ustringize.h>

Contents

  • CHAOS_PP_PRIMITIVE_USTRINGIZE
  • CHAOS_PP_USTRINGIZE

CHAOS_PP_USTRINGIZE

The CHAOS_PP_USTRINGIZE macro stringizes its argument after it has been expanded.

Usage - C++0x Specific 

CHAOS_PP_USTRINGIZE(n)(...)
n

The type of encoding of the resulting string literal. Valid values are 8, 16, and 32.

[...]

The argument to be stringized.

Remarks

This macro allows its argument to expand on input. This macro's sibling, CHAOS_PP_PRIMITIVE_USTRINGIZE, does not provide this delay.

If n is 8, the resulting string literal is encoded in UTF-8 such as in u8"text". If n is 16, the resulting string literal is encoded in UTF-16 such as in u"text". If n is 32, the resulting string literal is encoded in UTF-32 such as in U"text".

Both leading and trailing whitespace are removed from the argument when it is stringized, and internal whitespace separations are each condensed to a single space character. 

CHAOS_PP_USTRINGIZE(8)( abc )  // u8"abc"
CHAOS_PP_USTRINGIZE(32)(1   2) // U"1 2"

If __VA_ARGS__ expands to a placemarker, this macro expands to an empty wide string literal. 

CHAOS_PP_USTRINGIZE(16)() // u""

Sample Code

#include <chaos/preprocessor/ustringize.h>

#define A text

CHAOS_PP_USTRINGIZE(32)(A) // U"text"

CHAOS_PP_USTRINGIZE_ID

The CHAOS_PP_USTRINGIZE_ID macro indirectly expands to CHAOS_PP_USTRINGIZE.

Usage - C++0x Specific 

CHAOS_PP_USTRINGIZE_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_USTRINGIZE_

The CHAOS_PP_USTRINGIZE_ macro expands to a λ expression representing CHAOS_PP_USTRINGIZE.

Usage - C++0x Specific 

CHAOS_PP_USTRINGIZE_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_USTRINGIZE)

CHAOS_PP_PRIMITIVE_USTRINGIZE

The CHAOS_PP_PRIMITIVE_USTRINGIZE macro stringizes its argument before it is expanded.

Usage - C++0x Specific 

CHAOS_PP_PRIMITIVE_USTRINGIZE(n)(...)
n

The type of encoding of the resulting string literal. Valid values are 8, 16, and 32.

[...]

The argument to be stringized.

Remarks

This macro behaves identically to CHAOS_PP_USTRINGIZE except that it does not allow its argument to expand before it is wide-stringized.
