CHAOS_PP_LAMBDA

The CHAOS_PP_LAMBDA macro is used to create a λ binding.

Usage - C99 Specific 

CHAOS_PP_LAMBDA(...)
[...]

The argument on which a λ binding is to be created.

Remarks

This macro produces a λ expression containing __VA_ARGS__.

The primary purpose of this macro is to bind certain elements of a λ expression such that the parsing mechanism can parse past those elements. Technically, just about any sequence of preprocessing tokens is a λ expression. Because of the limitations of the preprocessor, however, the λ mechanism cannot parse past certain types of preprocessing tokens. For example, in the following code: 

CHAOS_PP_INVOKE(+ _1, x)

The λ mechanism cannot reach the _1, and consequently cannot replace it. This macro may be used in such situations to bind preprocessing tokens. 

CHAOS_PP_INVOKE(CHAOS_PP_LAMBDA(+) _1, x) // + x

Another purpose of this macro is to create λ expressions that contain open commas. One of the rules of a λ expression is that it must be passable as a single argument, so an expression such as 

a, b

cannot be a λ expression. This macro allows for the creation of such expressions. 

CHAOS_PP_LAMBDA(a, b)

Finally, the last purpose of this macro is to define λ bindings on macro names. For an arbitrary macro MACRO, this is commonly accomplished in one of two ways. The first is... 

#define MACRO_ CHAOS_PP_LAMBDA(MACRO)

The second is... 

#define MACRO_ CHAOS_PP_LAMBDA(MACRO_ID)()

The second form requires an indirection macro, MACRO_ID, that expands to MACRO when invoked with nullary parentheses. This indirection acts as a buffer that allows the resulting λ expression to pass through MACRO without the identifier MACRO becoming disabled.

Sample Code

#include <chaos/preprocessor/arithmetic/dec.h>
#include <chaos/preprocessor/arithmetic/inc.h>
#include <chaos/preprocessor/cat.h>
#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/control/while.h>
#include <chaos/preprocessor/lambda/invoke.h>
#include <chaos/preprocessor/lambda/ops.h>
#include <chaos/preprocessor/logical/bool.h>
#include <chaos/preprocessor/recursion/expr.h>
#include <chaos/preprocessor/tuple/elem.h>

#if CHAOS_PP_VARIADICS

??=include CHAOS_PP_PLACEHOLDERS(1)

CHAOS_PP_EXPR(CHAOS_PP_INVOKE(
    CHAOS_PP_CAT_(class T, _0) CHAOS_PP_LAMBDA(=) _1,
    x, y
))
// class Tx = y

#define ADD(x, y) \
    CHAOS_PP_VARIADIC_ELEM( \
        0, \
        CHAOS_PP_EXPR(CHAOS_PP_WHILE( \
            CHAOS_PP_BOOL_(_2), \
            CHAOS_PP_LAMBDA( \
                CHAOS_PP_INC_(_1), CHAOS_PP_DEC_(_2) \
            ), \
            x, y \
        )) \
    ) \
    /**/

ADD(5, 6) // 11

#define MACRO(a, b) { a, b }
#define MACRO_ID() MACRO
#define MACRO_ CHAOS_PP_LAMBDA(MACRO_ID)()

CHAOS_PP_EXPR(CHAOS_PP_INVOKE(
    MACRO_(_0, _1),
    p, q \
))
// { p, q }

??=include CHAOS_PP_PLACEHOLDERS(0)

#endif

CHAOS_PP_ESCAPE

The CHAOS_PP_ESCAPE macro produces a λ expression whose contents are not parsed by the λ mechanism.

Usage - C99 Specific 

CHAOS_PP_ESCAPE(...)
[...]

The preprocessing tokens to be escaped.

Remarks

This macro expands to a λ expression that prevents __VA_ARGS__ from being parsed by the λ mechanism.

Sample Code

#include <chaos/preprocessor/cat.h>
#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/lambda/invoke.h>
#include <chaos/preprocessor/lambda/ops.h>
#include <chaos/preprocessor/recursion/expr.h>

#if CHAOS_PP_VARIADICS

??=include CHAOS_PP_PLACEHOLDERS(1)

CHAOS_PP_EXPR(CHAOS_PP_INVOKE(
    CHAOS_PP_EXPR(CHAOS_PP_INVOKE(
        _0,
        CHAOS_PP_ESCAPE(
            CHAOS_PP_CAT_(class T, _0) CHAOS_PP_LAMBDA(=) _1
        )
    )),
    x, int
))
// class Tx = int

??=include CHAOS_PP_PLACEHOLDERS(0)

#endif

CHAOS_PP_OVERRIDE

The CHAOS_PP_OVERRIDE macro provides a means for the λ mechanism to parse tuples that contain more than CHAOS_PP_LIMIT_LAMBDA elements.

Usage - C99 Specific 

CHAOS_PP_OVERRIDE(...)
[...]

The elements of a tuple.

Remarks

The λ mechanism stores internal data in tuple elements beyond CHAOS_PP_LIMIT_LAMBDA elements. This imposes a restriction on λ expressions such that they cannot contain tuples containing more than CHAOS_PP_LIMIT_LAMBDA elements. This macro overrides that restriction by explicitly telling the λ mechanism that a tuple is to be parsed as a tuple rather than as internal data.

This macro is a more explicit form of the following λ expression: 

(CHAOS_PP_LAMBDA(__VA_ARGS__))

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/lambda/invoke.h>
#include <chaos/preprocessor/lambda/ops.h>
#include <chaos/preprocessor/recursion/expr.h>

#if CHAOS_PP_VARIADICS

CHAOS_PP_EXPR(CHAOS_PP_INVOKE(
    CHAOS_PP_OVERRIDE(a, CHAOS_PP_ARG(0), CHAOS_PP_ARG(1)),
    b, c
))
// (a, b, c)

#endif

CHAOS_PP_SELF

The CHAOS_PP_SELF macro expands to the λ expression currently being parsed.

Usage - C99 Specific 

CHAOS_PP_SELF()

Remarks

This macro expands to the λ expression currently being parsed. This allows a λ expression to recurse on itself.

Sample Code

#include <chaos/preprocessor/arithmetic/dec.h>
#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/lambda/invoke.h>
#include <chaos/preprocessor/lambda/ops.h>
#include <chaos/preprocessor/punctuation/comma_if.h>
#include <chaos/preprocessor/recursion/expr.h>

#if CHAOS_PP_VARIADICS

??=include CHAOS_PP_PLACEHOLDERS(1)

CHAOS_PP_EXPR(CHAOS_PP_INVOKE(
    CHAOS_PP_WHEN_(_0)(
        CHAOS_PP_EXPR_(CHAOS_PP_INVOKE_(
            CHAOS_PP_SELF(), CHAOS_PP_DEC_(_0)
        ))
        CHAOS_PP_COMMA_IF_(CHAOS_PP_DEC_(_0))
        CHAOS_PP_CAT_(class T, CHAOS_PP_DEC_(_0))
    ),
    3
))
// class T0, class T2, class T3

??=include CHAOS_PP_PLACEHOLDERS(0)

#endif

CHAOS_PP_ARG

The CHAOS_PP_ARG macro produces a reference to a λ placeholder.

Usage - C99 Specific 

CHAOS_PP_ARG(n)
n

The index of the λ placeholder to be referenced. Valid values range from 0 to CHAOS_PP_LIMIT_PLACEHOLDERS - 1.

Remarks

This macro expands to a reference to the n-th λ placeholder.

This macro is an longhand form of the direct use of the λ placeholders (e.g. _0, _1, etc.) which are more convenient to use, but cannot always be defined.

CHAOS_PP_PLACEHOLDERS

The CHAOS_PP_PLACEHOLDERS macro is used to define and undefine the λ placeholders.

Usage 

??=include CHAOS_PP_PLACEHOLDERS(bit)
bit

The value that determines whether the λ placeholders should be defined or undefined. Valid values are 0 and 1.

Remarks

In C99, if bit expands to 1, the λ placeholders (e.g. _0, _1, etc.) are defined by the library. Otherwise, they are undefined.

In C (prior to C99) and C++, the CHAOS_PP_PLACEHOLDERS macro expands to CHAOS_PP_NULL(). Thus, using it has no effect on translation.

It is important that the definitions of the λ placeholders only exist temporarily because they are unprefixed macro definitions that will interfere with any other use of those identifiers. Therefore, for every use of CHAOS_PP_PLACEHOLDERS that enables the λ placeholders, there should be a matching use of CHAOS_PP_PLACEHOLDERS that disables them. The library cannot enforce this directly, but the CHAOS_PP_WITH_PLACEHOLDERS mechanism may be used to enforce it, or the longhand CHAOS_PP_ARG form may be used instead.

Names that begin with an underscore are reserved for the implementation in file scope (C) or the global namespace (C++). Consequently, if a standard header is included while the λ placeholders are defined, the result is undefined behavior. As a general rule, no "normal" headers should be included while the λ placeholders are defined.

See Also

  • CHAOS_PP_ARG
  • CHAOS_PP_INCLUDE
  • CHAOS_PP_IS_WITH_PLACEHOLDERS
  • CHAOS_PP_WITH_PLACEHOLDERS

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/lambda/invoke.h>
#include <chaos/preprocessor/lambda/ops.h>
#include <chaos/preprocessor/recursion/expr.h>

#define MACRO(x, y) x + y

#if CHAOS_PP_VARIADICS

CHAOS_PP_EXPR(CHAOS_PP_INVOKE(
    CHAOS_PP_LAMBDA(MACRO)(
        CHAOS_PP_ARG(0), CHAOS_PP_ARG(1)
    ),
    1, 2
))
// 1 + 2

??=include CHAOS_PP_PLACEHOLDERS(1)

CHAOS_PP_EXPR(CHAOS_PP_INVOKE(
    CHAOS_PP_LAMBDA(MACRO)(_0, _1),
    1, 2
))
// 1 + 2

??=include CHAOS_PP_PLACEHOLDERS(0)

#endif

CHAOS_PP_INCLUDE

The CHAOS_PP_INCLUDE macro is a user-defined named external argument that specifies the file that should be included by CHAOS_PP_WITH_PLACEHOLDERS.

Usage - C99 Specific 

#define CHAOS_PP_INCLUDE file
file

The quoted path of the file to be included.

Remarks

This macro is defined by the user prior to the use of CHAOS_PP_WITH_PLACEHOLDERS. When CHAOS_PP_WITH_PLACEHOLDERS is used, the file specified by file is included with the λ placeholders defined. Upon completion of the inclusion, the λ placeholders are undefined.

This macro is automatically undefined by the library.

CHAOS_PP_WITH_PLACEHOLDERS

The CHAOS_PP_WITH_PLACEHOLDERS macro is used to include a file with the λ placeholders defined only for that file.

Usage - C99 Specific 

??=include CHAOS_PP_WITH_PLACEHOLDERS()

Remarks

This macro defines the λ placeholders (e.g. _0, _1, _2, etc.), includes the file specified by the named external argument CHAOS_PP_INCLUDE, and then undefines the λ placeholders. This functionality can be used to enforce the guideline that every definition of the unsafe λ placeholders is followed by a corresponding undefinition.

See Also

  • CHAOS_PP_ARG
  • CHAOS_PP_INCLUDE
  • CHAOS_PP_IS_WITH_PLACEHOLDERS
  • CHAOS_PP_PLACEHOLDERS

Sample Code

// file.h
#if !CHAOS_PP_IS_WITH_PLACEHOLDERS

#ifndef FILE_H
#define FILE_H

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/extended/variadic_cat.h>
#include <chaos/preprocessor/lambda/invoke.h>
#include <chaos/preprocessor/lambda/ops.h>

#if CHAOS_PP_VARIADICS

#define CHAOS_PP_INCLUDE "file.h"
??=include CHAOS_PP_WITH_PLACEHOLDERS()

#endif

#endif // FILE_H

#else

CHAOS_PP_EXPR(CHAOS_PP_INVOKE(
    CHAOS_PP_VARIADIC_CAT_(_0, _1, _2, _3, _4, _5),
    a, b, c, d, e
))
// abcde

#endif
