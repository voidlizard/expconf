The lambda/route.h header defines a macro used to abstract the difference between the invocation of a macro and a λ expression as well as explicitly route the point of invocation.

Usage

#include <chaos/preprocessor/lambda/route.h>

Contents

  • CHAOS_PP_ROUTE
  • CHAOS_PP_WAYPOINT

CHAOS_PP_WAYPOINT

The CHAOS_PP_WAYPOINT macro provides a context used to cause the expansion of invocations routed with CHAOS_PP_ROUTE.

Usage - C99 Specific 

CHAOS_PP_WAYPOINT(...)
[...]

The argument whose routed invocations are to be expanded.

Remarks

This macro is used to specify the point at which certain invocations are to expand.

This macro may not be used when bypass semantics are in effect.

See Also

  • CHAOS_PP_PLANAR

CHAOS_PP_WAYPOINT_ID

The CHAOS_PP_WAYPOINT_ID macro indirectly expands to CHAOS_PP_WAYPOINT.

Usage - C99 Specific 

CHAOS_PP_WAYPOINT_ID()

Remarks

This macro effectively allows CHAOS_PP_WAYPOINT to pass through itself without becoming disabled.

CHAOS_PP_WAYPOINT_

The CHAOS_PP_WAYPOINT_ macro expands to a λ expression representing CHAOS_PP_WAYPOINT.

Usage - C99 Specific 

CHAOS_PP_WAYPOINT_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_WAYPOINT_ID)()

CHAOS_PP_ROUTE

The CHAOS_PP_ROUTE macro is used to delay the invocation of a macro or λ expression until the context introduced by CHAOS_PP_WAYPOINT is present.

Usage - C99 Specific 

CHAOS_PP_ROUTE(x)()(s, x, ...)
x

The macro or λ expression to be invoked. If this argument is a macro, it must be a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, args†)

† The args pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in __VA_ARGS__ are intermediate expansion.

s

The recursion state in terms of which the resulting deferred expression is to be generated.

[...]

The intermediate argument or arguments that are to comprise the trailing arguments passed to x.

Remarks

This macro expands to an active expression that, when introduced into the context created by CHAOS_PP_WAYPOINT expands to a deferred expression in terms of s that contains an invocation of x. It also allows this deferred expression, when invoked, to expand to a deferred expression in terms of CHAOS_PP_NEXT(s).

Normal usage consists of invoking CHAOS_PP_ROUTE without the trailing invocations, passing that result around, and finishing the invocation at another time. This avoids repeatedly detecting the difference between a macro name and a λ expression. 

#define MACRO(s, x) \
    CHAOS_PP_WAYPOINT(MACRO_I(s, x, CHAOS_PP_ROUTE(x))) \
    /**/
#define MACRO_I(s, x, _x) _x()(s, x, ~)

This macro is similar in function to CHAOS_PP_TRAMPOLINE, but allows for an arbitrary number of scans over the resulting expression before expansion. The CHAOS_PP_TRAMPOLINE macro is much trickier to use than CHAOS_PP_ROUTE to accomplish the same overall effect, but the use of CHAOS_PP_ROUTE is significantly more expensive because it is active.

This macro may not be used when bypass semantics are in effect.

Sample Code

#include <chaos/preprocessor/arithmetic/dec.h>
#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/control/when.h>
#include <chaos/preprocessor/lambda/ops.h>
#include <chaos/preprocessor/lambda/route.h>
#include <chaos/preprocessor/recursion/basic.h>
#include <chaos/preprocessor/recursion/expr.h>

#if CHAOS_PP_VARIADICS

#define REPEAT(count, macro, ...) \
    REPEAT_S(CHAOS_PP_STATE(), count, macro, __VA_ARGS__) \
    /**/
#define REPEAT_S(s, count, macro, ...) \
    CHAOS_PP_WAYPOINT(CHAOS_PP_EXPR_S(s)(REPEAT_I( \
        CHAOS_PP_OBSTRUCT(), CHAOS_PP_NEXT(s), CHAOS_PP_NEXT(s), \
        count, macro, CHAOS_PP_ROUTE(macro), __VA_ARGS__ \
    ))) \
    /**/
#define REPEAT_INDIRECT() REPEAT_I
#define REPEAT_I(_, s, o, count, macro, _m, ...) \
    CHAOS_PP_WHEN _(count)( \
        CHAOS_PP_EXPR_S(s) _(REPEAT_INDIRECT _()( \
            CHAOS_PP_OBSTRUCT _(), CHAOS_PP_NEXT(s), o, \
            CHAOS_PP_DEC(count), macro, _m, __VA_ARGS__ \
        )) \
        _m()(o, macro, CHAOS_PP_DEC(count), __VA_ARGS__) \
    ) \
    /**/

#define MACRO(s, n, _) CHAOS_PP_EXPR_S(s)(n)

CHAOS_PP_EXPR(REPEAT(5, MACRO,)) // 0 1 2 3 4

??=include CHAOS_PP_PLACEHOLDERS(1)

CHAOS_PP_EXPR(REPEAT(
    5,
    CHAOS_PP_EXPR_S_(_0)(_1),
))
// 0 1 2 3 4

??=include CHAOS_PP_PLACEHOLDERS(0)

#endif
